# Authority Clarity Operating Law

Status: DOCUMENTATION ONLY -- PRINCIPLE ANCHOR

## Core Operating Law

Power must kneel to responsibility before it moves.

## Missing Human Governance Principle

Humans are missing governance that respects vulnerability.

Systems must be built from the vulnerable subject outward, not from capability
outward.

Non-action can be a successful protected outcome.

## Authority Clarity Identification Rule

Authority Clarity identifies where power is about to move while responsibility,
authority, permission, refusal, mode, or risk remains unresolved.

## Living Control Pattern

* For every strength, name its shadow.
* For every shadow, build a gate.
* For every gate, write a test.
* For every test, require proof.
* For every proof, keep an audit trail.

## Repo Traceability

The system can find this principle in:

* documents / policy intent
* schema constraints
* validator checks
* tests
* CI proof
* future runtime gates
* audit logs

## Protected Non-Action Examples

Protected non-action can include:

* the system did not act
* the trade did not fire
* the child was not forced
* the caregiver was not overridden
* the broker was not touched
* the machine stopped because authority was unclear
