## Layer Roles

NI-AI explains and contextualizes inputs, meaning, intent, coherence, and
possible interpretation. NI-AI does not authorize execution.

SCI frames the structural context before authority movement. SCI helps place
inputs inside the relevant context, but it does not grant permission by
itself.

MPC / MCG controls mode, state, discipline, and process routing. MPC / MCG
identifies whether movement is coherent, bounded, and properly routed before
authority can be treated as available.

SAI determines the authorized state level. SAI language describes state
authorization boundaries; it is not live execution permission by itself.

EchoAuth enforces permission and authentication boundaries. EchoAuth is the
permission-enforcement layer that keeps authority, policy, invariants,
runtime envelope, action, resource, payload, and time bounded before any
execution surface can be considered.

Execution remains bounded and cannot exceed the authorized mode. A bounded
execution context may only exist within separately authorized scope and must
not expand because doctrine, journal notes, glossary terms, or this bridge
