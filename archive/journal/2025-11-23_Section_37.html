• secure memory partition

• fuse-based key storage

• hardware root-of-trust

• prohibited-driver lockout

→ device auto-locks

→ parent is alerted

• AES-256 encryption

• local-only decoding

⭐ 37.3 Data Categories & Protection Rules


EchoAuth categorizes data into five classes, each with strict rules:


Class 1 — Safety Data


(elopement risk, danger signals)


Protection:

• stored temporarily

• never exported

• deleted after window expires


Class 2 — Sensory Data


(light, noise, overload patterns)


Protection:

• local-only

• no cloud transmission unless parent chooses


Class 3 — Emotional Data


(anxiety, fear, shutdown indicators)


Protection:

• anonymized by default

• cannot be used by schools

• available only to caregiver


Class 4 — Medical Micro-Data


(fever, fatigue, hydration risk)


Protection:

• encrypted with medical-only key

• export allowed only to approved clinician


Class 5 — Preference Data (NI-based)


(comfort tones, stimming meaning, transitions)


Protection:

• never shared outside the home/school network

• parent has full editing power
