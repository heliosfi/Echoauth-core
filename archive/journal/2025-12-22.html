Purpose: Details the audit logging process, supporting AU family controls.

Text-Based Representation (Step-by-Step Flow):

1 Event Trigger: Any security-relevant action (e.g., authentication attempt, drift detection, access request) in P1-P7.

2 Log Capture: P8 Layer collects timestamp, user/session ID (pseudonymous), event type, outcome, risk score.

3 Enrichment: Adds context (e.g., IP, device type if allowed; RMS details).

4 Encryption: AES-256 applied before write.

5 Storage: Written to D3 Immutable Audit Log Store (write-once, read-many).

6 Monitoring: Real-time scans for anomalies; alerts to Admin Console.

7 Review/Export: Weekly manual reviews; exports to agency via secure API (e.g., SIEM integration).

8 Retention: 6-12 months per policy (AU-11); auto-purge of old logs.
