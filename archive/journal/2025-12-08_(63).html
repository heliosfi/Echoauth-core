6.0 API Overview

• Style: REST-ish over HTTPS, JSON in / JSON out

• Base URL (example): https://ni-ai-core.yourdomain.com/v1

• Auth: API key or JWT in Authorization: Bearer <token>

• Guarantee:

Every call flows through ORI → NI → Coherence → Execution → Safety.

No endpoint bypasses safety.

6.2 Endpoint Summary


Endpoint Method Purpose

/session/init POST Start / refresh ORI session

/interpret POST NI-only: parse + classify intent

/coherence/check POST Run packet through Coherence Engine

/execute POST Execute a known intent with safety

/pipeline POST Full chain: interpret → coherence → execute

/session/reset POST Clear memory for a given session/user

/health GET Liveness + safety mode check


You can implement all of these using the bootstrap_system() + handle_request() skeleton we just built.

6.3 Endpoint Details


6.3.1 POST /session/init


Purpose:

Bind ORI to a user/role + create or resume a session.


Request:


{

  "identity": { ... IdentityContext ... },

  "session_id": "optional-existing-session-id"

}


Response:


{

  "session_id": "sess-abc123",

  "ori_signature": "ori-xyz789",

  "status": "initialized"

}


Backend hook:

• Calls ori.initialize(traits=identity.traits) once if new user

• Calls ori.sync_session(...) each time


⸻


6.3.2 POST /interpret


Purpose:

NI-only mode. Good for labs that want to inspect intent without running actions.


Request:


{

  "input": { ... InputPacket ... }

}


Response:


{

  "parsed": { "tokens": [...], "structure": {...} },

  "intent": "ANSWER_QUESTION",

  "confidence": 0.92,

  "identity": {

    "ori_signature": "ori-xyz789",

    "user_id": "child-123"

  },

  "session": {

    "session_id": "sess-abc123"

  }

}


Backend hook:

• Maps 1:1 to ni.interpret(raw_input)

• No execution, no side effects beyond normal memory logging.


⸻


6.3.3 POST /coherence/check


Purpose:

Let integrators feed a packet (usually from /interpret) and see if it passes UMCE checks.


Request:


{

  "packet": {

    "parsed": {...},

    "intent": "ANSWER_QUESTION",

    "identity": {...},

    "session": {...}

  }

}


Response:


{

  "coherence": {

    "narrative_ok": true,

    "temporal_ok": true,

    "logic_ok": true,

    "notes": []

  },

  "status": "ok"

}


Backend hook:

• Calls coherence.validate(packet)


⸻


6.3.4 POST /execute


Purpose:

Run a specific intent through the Execution Engine with safety.

Used when the caller already knows the intent.


Request:


{

  "intent": "ANSWER_QUESTION",

  "context": {

    "parsed": {...},

    "identity": {...},

    "session": {...}

  }

}


Response:

ExecutionResult JSON as defined above.


Backend hook:

• Validates intent vs safety rules: exec_engine.execute(intent, context)


⸻


6.3.5 POST /pipeline


Purpose (most important):

One-shot: interpret → coherence → execute → safety wrap.

This is the main endpoint for EchoAuth, ASD tools, robotics shells, etc.


Request:


{

  "input": { ... InputPacket ... },

  "options": {

    "require_coherence": true,

    "max_latency_ms": 800,

    "mode": "assistive"   // e.g. "assistive", "explanatory", "debug"

  }

}


Response:

Full ExecutionResult.


Backend flow (conceptual):


packet = ni.interpret(input["text"])

checks = coherence.validate(packet)


if options["require_coherence"] and not all(checks.values()):

    return {

        "status": "refused",

        "reason": "coherence_check_failed",

        "coherence": checks

    }


result = exec_engine.execute(packet["intent"], context=packet)

return result



⸻


6.3.6 POST /session/reset


Purpose:

Allow parent/teacher/clinician or system admin to reset memory for a session/user.


Request:


{

  "session_id": "sess-abc123",

  "user_id": "child-123",

  "mode": "soft"     // "soft" = keep identity, clear recent history; "hard" = full reset

}


Response:


{

  "status": "ok",

  "mode": "soft",

  "cleared": true

}



⸻


6.3.7 GET /health


Purpose:

Simple health + safety-mode probe for hospital IT / school admins.


Response:


{

  "status": "ok",

  "uptime_sec": 123456,

  "version": "ni-ai-core-0.1.0",

  "safety_mode": "strict",     // strict / normal / maintenance

  "allowed_intents": ["ANSWER_QUESTION", "EXPLAIN", "SUMMARIZE"]

}
