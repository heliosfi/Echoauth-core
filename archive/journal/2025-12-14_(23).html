STAGE 1 — SYSTEM DEFINITION


NI-AI + EchoAuth + MCG (One-page, unified definition)


System One-Liner (use everywhere)


NI-AI is a read-only explanation layer, EchoAuth is the authorization layer, and MCG is the governance spine that enforces safety and restraint—together they support non-verbal children through caregiver-mediated, non-inferential interaction.


What each part is


MCG (Governance Spine)

• Final authority for “allow / deny / pause”

• Observes context and safety state without inferring intent or emotion

• Enforces “non-action is success” when uncertain


EchoAuth (Authorization + Roles)

• Verifies who is allowed to act (caregiver/parent/staff/admin)

• Default state is deny execution

• Any failure collapses back to observation + logging


NI-AI (Explanation Layer)

• Provides transparency: what the system saw, what rule applied, why it paused/denied

• Does not decide, diagnose, predict, or control

• Produces audit-friendly explanations for caregivers and reviewers


How they connect (pipeline)

1. MCG observes system state + context (where/when/transitions)

2. EchoAuth checks authorization (role + permission)

3. MCG decides allow / pause / deny (fail-closed)

4. NI-AI explains the outcome (read-only)

5. Audit log records the state + decision + reason


Non-verbal guarantee (must stay explicit)

• No speech required

• Silence is valid

• Non-participation is not failure

• The system stabilizes environment + caregiver response, not child behavior


What the system does NOT do (funding-safe boundary)

• No medical diagnosis

• No intent/emotion inference as truth

• No autonomous actions affecting the child

• No behavioral scoring for punishment or compliance


✅ Stage 1 Status: Defined, funder-safe, reviewer-legible.
