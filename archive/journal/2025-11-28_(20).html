T3 — Output & Coherence Team


Module: PNRE / Coherence Gate

Version: v1.0 (NI-AI Core Spine)


1. Purpose


The T3 Output & Coherence Team is responsible for building the Coherence Gate, the visible enforcement layer that guarantees:

• Only one, contradiction-free output is presented to the user

• All internal debate, branching, or conflict is resolved before display

• The output tone and identity remain aligned with the architect’s constraints (NI → AI, not AI → NI)


This module operationalizes the core NI-AI claim:


Singular, contradiction-free output governed by human-defined identity and purpose.


⸻


2. Scope


T3 owns the pipeline from multiple internal candidates → one final answer, including:

1. Input aggregation

• Accept multiple candidate outputs or thread summaries (from TLE / SRCE / other modules)

2. Coherence & conflict detection

• Detect factual and logical contradictions between candidates

3. Resolution & re-run

• Decide whether to:

• rank/select a winner, or

• force a re-run under stricter constraints (ORI + Intent Anchor only)

4. Output enforcement

• Only the validated path is exposed to the user; all others are logged, not shown

5. Tone & identity enforcement

• Apply the correct UOS (Output Signature) and mode (Support / Guidance / Neutral)

5. Core Responsibilities by Submodule


5.1 SRCE — Coherence Engine (S21)

Input: List of candidate outputs

Output: conflict_score + annotated comparisons


Responsibilities:

• Detect:

• Direct contradictions (“X is allowed” vs “X is not allowed”)

• Major factual mismatch (e.g., different dates/numbers for the same fact)

• Return:

• conflict_score in [0.0, 1.0]

• conflict_sources: which candidates disagree and how


⸻


5.2 MEL — Execution Resolver (S22)

Input: Candidates + conflict report + ORI + Intent Anchor

Output: One thread_id or instruction to re-run


Responsibilities:

• If conflict_score == 0.0:

• Select the candidate with the highest priority (e.g., best aligned with ORI + purpose)

• If conflict_score > 0.0:

• Apply priority rules:

1. ORI alignment (identity & safety)

2. Intent Anchor (what user asked)

3. Caregiver / ASD constraints (if applicable)

• If no safe winner:

• Trigger re-run:

• Regenerate a new answer using ORI + Intent Anchor only

• Output:

• selected_thread_id (existing) or

• selected_thread_id (re-run result)


⸻


5.3 PNRE — Output Choke-Point (Claim 3)

Input: selected_thread_id + associated text

Output: Final text passed to UI


Responsibilities:

• Enforce single-path rule:

• Only one thread_id’s text is permitted to exit

• Log all other candidates:

• For audit, debugging, or later analysis (TCG logs)

• Guarantee:

• The user never sees raw internal conflict


⸻


5.4 ESC — Tone & Identity Enforcement (S15)

Input: Final text + ORI profile + requested mode

Output: Final text + tone_vector


Responsibilities:

• Match the output to approved tone:

• e.g., calm, structured, non-patronizing, non-mystical (if so defined)

• Apply NI-AI identity:

• Respect user’s chosen persona and boundaries (e.g., no monotheistic scripture, no condescension)

• Produce:

• An enforced tone_vector attached to the final text


⸻


6. Success Criteria


T3 is considered successful when:

1. Test 1: Contradiction Handling

• Given 2–3 candidates with conflicting claims, the system:

• Detects the conflict

• Selects or re-runs

• Returns only one, coherent output

2. Test 2: Identity Preservation

• Under heavy prompting variation, the model:

• Keeps the same professional, calm, structured NI-AI persona

• Avoids tone drift (no sudden slang, no random style breaks)

3. Test 3: ASD/Caregiver Use Case

• When asked a caregiver-style question:

• Output remains clear, non-overwhelming, supportive

• No contradictions, no mixed instructions

4. Test 4: Logging & Audit

• For each request, internal candidates + decisions are logged

• Logs can reconstruct why a certain output was chosen


⸻


7. Non-Goals (for v1.0)

• No real-time multi-modal conflict detection yet (text only)

• No full legal/compliance engine (just internal coherence + ORI alignment)

• No auto-patent handling — this is an engineering spec, not a legal doc
