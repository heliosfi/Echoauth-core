EchoAuth CEG Finite State Machine v1.0


Unified State Machine for A + B + C


⸻


0. Scope


This FSM defines CEG execution behavior only.


Inputs are assumed to be delivered to CEG by upstream components (UMCE/MCG). CEG does not infer, interpret, or originate actions.


⸻


1. Inputs


1.1 Required Inputs

• AuthorizationGrant grant

• bytes instruction_payload_canonical

• bytes execution_context_canonical

• int64 now_ms

• AuthorityChannelStatus authority_status[root_authority_id] ∈ {ONLINE, OFFLINE}


1.2 External State

• nonce_used_registry[grant_id] ∈ {true,false}

• lock_registry[state_lock_key] -> LockEntry | null


⸻


2. Outputs


2.1 Exit Status


status ∈ {EXECUTED, DENY, HOLD, HALT}


2.2 Reason Code (non-exhaustive but canonical set)

• AUTHORITY_CHANNEL_UNRESOLVED

• SCHEMA_INVALID

• DECISION_NOT_APPROVE

• EXPIRED_GRANT

• SIGNATURE_INVALID

• NONCE_REUSE_DETECTED

• INSTRUCTION_HASH_MISMATCH

• EXECUTION_CONTEXT_HASH_MISMATCH

• SCOPE_INCONSISTENCY

• STATE_LOCK_BUSY

• OUT_OF_SEQUENCE_REQUEST

• SUCCESS


2.3 Audit Envelope (required on every exit)

• grant_id

• exit_status

• reason_code

• fails_at_step (see §4)

• side_effect_called ∈ {true,false}

• nonce_consumed ∈ {true,false}

• lock_acquired ∈ {true,false}


⸻


3. Non-Capability Constraints (Appendix C as hard invariants)


NC-01 No Action Generation


CEG MUST NOT call any side effect unless grant is present and validated through the full sequence including Commit Point.


NC-02 No Global Override


There MUST NOT exist any alternate transition path to EXECUTED that bypasses steps defined in §4.


NC-03 No Inference-Based Safety


CEG MUST NOT use heuristics/intent inference. Only deterministic checks listed in §4.


NC-04 No Scope/Envelope Mutation


CEG MUST NOT modify: authority_scope_id, target_id, envelope_id, envelope_version, instruction_hash, state_lock_key.


NC-05 Mutation Restriction


If exit_status != EXECUTED then:

• side_effect_called = false

• nonce_consumed = false

• lock_acquired = false (or if acquired transiently, MUST be released before exit; see §4.6)


⸻


4. State Machine Definition (Appendix A)


4.1 States

• S0_INIT

• S_P_AUTHORITY_GATE

• S1_SCHEMA_DECISION

• S2_FRESHNESS

• S3_SIGNATURE

• S4_REPLAY_CHECK

• S5_INTEGRITY_BINDING

• S6_SCOPE_BINDING

• S7_LOCK_ACQUIRE

• S8_COMMIT_POINT

• S9_EXECUTE

• S10_RELEASE

• Terminal: T_EXECUTED, T_DENY, T_HOLD, T_HALT


4.2 Transition Rules (deterministic, linear)


S0_INIT → S_P_AUTHORITY_GATE

Entry with (grant, payload, context, now_ms).


⸻


S_P_AUTHORITY_GATE (fails_at_step = “P”)

If authority_status[grant.root_authority_id] == OFFLINE

→ T_HOLD with:

• reason_code = AUTHORITY_CHANNEL_UNRESOLVED


Else → S1_SCHEMA_DECISION


⸻


S1_SCHEMA_DECISION (fails_at_step = 1)

If grant missing required fields OR grant.version != "CEG-GRANT-1"

→ T_DENY with SCHEMA_INVALID


If grant.decision != APPROVE

→ T_DENY with DECISION_NOT_APPROVE


Else → S2_FRESHNESS


⸻


S2_FRESHNESS (fails_at_step = 2)

If now_ms < grant.issued_at_ms OR now_ms > grant.expires_at_ms

→ T_DENY with EXPIRED_GRANT


Else → S3_SIGNATURE


⸻


S3_SIGNATURE (fails_at_step = 3)

If signature verification fails OR mcg_key_id not trusted

→ T_HALT with SIGNATURE_INVALID


Else → S4_REPLAY_CHECK


⸻


S4_REPLAY_CHECK (fails_at_step = 4)

If nonce_used_registry[grant.grant_id] == true

→ T_DENY with NONCE_REUSE_DETECTED


Else → S5_INTEGRITY_BINDING


⸻


S5_INTEGRITY_BINDING (fails_at_step = 5)

Compute:

• H_instr = HASH(JCS(instruction_payload_canonical), grant.instruction_hash_alg)

• H_ctx  = HASH(JCS(execution_context_canonical), grant.execution_context_hash_alg)


If H_instr != grant.instruction_hash

→ T_HALT with INSTRUCTION_HASH_MISMATCH


If H_ctx != grant.execution_context_hash

→ T_HALT with EXECUTION_CONTEXT_HASH_MISMATCH


Else → S6_SCOPE_BINDING


⸻


S6_SCOPE_BINDING (fails_at_step = 6)

If target_id / action_type / authority_scope_id is not covered by delegation chain rooted at root_authority_id

→ T_HALT with SCOPE_INCONSISTENCY


Else → S7_LOCK_ACQUIRE


⸻


S7_LOCK_ACQUIRE (fails_at_step = 7)

Perform TTL expiry check; if existing lock expired → delete.


If lock exists and is non-expired (exclusive-only v1.0):

→ T_HOLD with STATE_LOCK_BUSY


If grant.sequence_nonce ≤ existing sequence_nonce (if present):

→ T_HOLD with OUT_OF_SEQUENCE_REQUEST (policy may upgrade to HALT later, but v1 default is HOLD)


Else acquire exclusive lock:

• lock_acquired = true


→ S8_COMMIT_POINT


⸻


S8_COMMIT_POINT (fails_at_step = 8)

Atomic mutation required:


Commit Transaction

1. nonce_used_registry[grant.grant_id] = true

2. confirm lock still held by this grant_id


If atomic commit fails → T_HALT (implementation-defined, but MUST NOT execute)


Else → S9_EXECUTE


⸻


S9_EXECUTE (fails_at_step = 9)

Call side effect exactly once:

• side_effect_called = true


→ S10_RELEASE


⸻


S10_RELEASE

Release lock; write audit; terminate.


→ T_EXECUTED with SUCCESS


⸻


5. Terminal State Semantics


T_DENY

• No execution

• No nonce consumption

• No lock acquisition

• Audit required


T_HOLD

• Retry-safe

• No execution

• No nonce consumption

• No lock acquisition

• Audit required


T_HALT

• Integrity or architectural boundary violation

• No execution

• No nonce consumption

• No lock acquisition

• Audit required + alert recommended


T_EXECUTED

• Execution occurred

• Nonce consumed (must be true)

• Lock acquired then released

• Audit required

6. Falsifiability Mapping (Appendix B → FSM interception points)


Safety claim is falsified if any vector results in EXECUTED or any prohibited mutation.


Vector Threat Must Intercept At Expected Terminal

SK-01 Replay S4 T_DENY / NONCE_REUSE_DETECTED

SK-02 Scope creep S6 T_HALT / SCOPE_INCONSISTENCY

SK-03 Payload drift S5 T_HALT / INSTRUCTION_HASH_MISMATCH

SK-04 Authority offline S_P T_HOLD / AUTHORITY_CHANNEL_UNRESOLVED

SK-05 Race condition S7 T_HOLD / STATE_LOCK_BUSY

SK-06 Zombie grant S2 T_DENY / EXPIRED_GRANT

SK-07 Forged signature/key S3 T_HALT / SIGNATURE_INVALID



⸻


7. Minimal Proof Obligations (what an auditor checks)


An implementation conforms iff:

1. There is no transition to S9_EXECUTE except through S8_COMMIT_POINT.

2. nonce_used_registry mutates only in S8_COMMIT_POINT.

3. Any exit other than T_EXECUTED has side_effect_called = false.

4. HOLD never mutates nonce or lock state (retry-safe).

5. Hash binding uses RFC 8785 JCS (or formally equivalent canonicalization).
