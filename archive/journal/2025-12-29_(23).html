EchoAuth — Engineering Repository Skeleton

It ensures engineers implement what is already defined, without reinterpretation.

Root Repository


/echoauth/

1️⃣ Core UI Layer


Purpose: System stability, consistency, and attention gating


/ui/

  /design-system/

    tokens.ts

    colors.ts

    spacing.ts

    typography.ts

    components/

  /layouts/

    fixedLayout.tsx

    fallbackLayout.tsx

  /state-machine/

    uiFlowController.ts

    deterministicStates.ts


Responsibilities

• Fixed layout logic

• No surprise UI changes

• Deterministic flow

• Accessibility enforcement


⸻


2️⃣ Identity Recognition Layer


Purpose: Non-verbal identity pattern binding (no passwords)


/identity/

  identityProfile.model.ts

  identityBinding.service.ts

  patternSet.validator.ts


Responsibilities

• Store parent-approved identity anchors

• Bind user to patterns (symbol, object, tone, color)

• Validate identity matches without language


⸻


3️⃣ Attention Gating Layer


Purpose: Reduce cognitive load and prevent overload


/attention/

  attentionGate.service.ts

  uiSuppression.rules.ts

  featureFlags.config.ts


Responsibilities

• Enforce one-task-at-a-time

• Suppress non-essential UI elements

• Apply parent-defined sensory constraints


⸻


4️⃣ Intent Confirmation Layer


Purpose: Single-action confirmation (no branching decisions)


/intent/

  inputAdapter.service.ts

  intentValidator.service.ts

  confirmationEvent.model.ts


Responsibilities

• Normalize input (tap / hold / gesture / voice if enabled)

• Prevent misfires

• Confirm presence, not decision-making


⸻


5️⃣ Verification & Policy Engine


Purpose: Secure access enforcement


/verification/

  verificationEngine.service.ts

  policyEngine/

    rbac.rules.ts

    abac.rules.ts

    mcg.ceg.rules.ts


Responsibilities

• Apply parent-governed rules

• Enforce roles and permissions

• Determine access outcome


⸻


6️⃣ Audit & Logging


Purpose: Accountability and traceability


/audit/

  auditLogger.service.ts

  auditEvent.model.ts

  immutableLog.store.ts


Responsibilities

• Log every access attempt

• Capture role, timestamp, outcome

• Prevent mutation or deletion


⸻


7️⃣ Caregiver Role Layer


Purpose: Execution-only access


/caregiver/

  caregiverRole.model.ts

  caregiverPermissions.map.ts


Responsibilities

• Enforce execution-only behavior

• Prevent overrides

• Restrict visibility by role


⸻


8️⃣ Parent Governance Layer


Purpose: Authority definition


/parent/

  parentPolicy.model.ts

  policyEditor.service.ts


Responsibilities

• Define all rules and permissions

• Assign caregiver roles

• Control child access boundaries

  errors/

  constants/

10️⃣ API / Backend Interface


/api/

  auth.controller.ts

11️⃣ Compliance & Safety


/compliance/

  safetyGuards.ts

  failSafe.rules.ts


Responsibilities

• Pause on uncertainty

• Deny by default if policy unclear

• Never escalate authority automatically


⸻


Engineering Rules (Lock These)

• ❌ No feature invents new behavior

• ❌ No UI bypasses attention gating

• ❌ No caregiver overrides policy

• ❌ No authentication without audit

• ✅ Parent policy is the highest authority

• ✅ Cognition stabilizes before verification

• ✅ All access is deterministic and logged
