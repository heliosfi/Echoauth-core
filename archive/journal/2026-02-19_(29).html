CEG Token Handshake — Formal Specification (v1.0)


0) Scope


Defines the unforgeable signal linking MCG approval to CEG execution, such that:

• UMCE (identity binding) is non-executable

• CEG (execution) is non-judgmental

• No execution occurs without fresh, per-request MCG approval

• Replay, drift, and authority substitution are mechanically blocked


⸻


1) Entities (roles, not agents)

• UMCE: produces identity_binding (non-executable)

• MCG: sole authority/judgment; issues auth_grant

• CEG: mechanical gate; validates auth_grant and enforces single-use execution


⸻


2) Required Properties (Invariants)


I1 — Freshness


Every executable action requires a new MCG grant.


Pass condition: CEG rejects any grant outside expiry_window_ms or previously used.


I2 — Single-use


A grant can authorize exactly one execution.


Pass condition: CEG maintains a nonce registry; second use → deny.


I3 — Binding


Grant is bound to:

• who (root authority + role scope)

• what (instruction hash)

• where (target/resource)

• which envelope (parameter authority scope)

• when (time window)

• why is not represented (CEG must not interpret intent)


Pass condition: any mismatch invalidates grant.


I4 — Non-Bypass


CEG cannot execute via UMCE output, cached permissions, or local inference.


Pass condition: only auth_grant unlocks execution path.


I5 — Determinism under conflict


If grant validation is ambiguous (missing fields, unverifiable signature, unknown envelope), outcome is HOLD/DENY, never “best effort.”


⸻


3) Token Types


3.1 Identity Binding Artifact (UMCE Output — non-executable)


identity_binding is an input to MCG only.


Minimum fields:

• subject_id (child/system principal)

• actor_id (request initiator principal)

• role_claim (e.g., parent / delegate / school / clinical)

• binding_signature (UMCE attestation)

• binding_timestamp


Invariant: CEG MUST ignore this artifact; it has no execution semantics.


⸻


3.2 Authorization Grant (MCG → CEG) — executable only via CEG


auth_grant is the sole executable authorization primitive.


Canonical Data Structure (logical schema)

• Header

• version: “CEG-GRANT-1”

• grant_id: unique identifier (UUID/ULID)

• issued_at_ms: epoch ms

• expires_at_ms: epoch ms

• Authority Binding

• root_authority_id: parent/legal guardian principal id

• actor_id: initiator principal id

• actor_role: role at time of approval

• delegation_chain: ordered list of delegation proofs (may be empty for parent)

• authority_scope_id: scope identifier (what this authority is allowed to approve)

• Envelope Binding

• envelope_id: governed envelope identifier (e.g., S23 dietary/med window)

• envelope_version: immutable version hash/id

• envelope_constraints_hash: hash of constraints applied at approval time

• Execution Binding

• target_id: resource/action target identifier

• action_type: normalized action label (execute, notify, access, etc.)

• instruction_hash: hash of canonicalized instruction payload

• execution_context_hash: hash of context fields that must not drift (e.g., location class, device class, session class) — only if used

• Concurrency Binding

• state_lock_key: global lock namespace key (target-specific)

• lock_mode: {exclusive | shared} (default exclusive)

• sequence_nonce: monotonic or unique per lock key

• Decision

• decision: {APPROVE | DENY | HOLD | ESCALATE}

• decision_code: machine-readable reason category (non-narrative)

• Signature

• mcg_signature: detached signature over all above fields (canonical serialization)

• mcg_key_id: signing key reference


Absolute Rules

• If decision != APPROVE → CEG MUST NOT execute.

• CEG MUST validate signature, window, nonce, and bindings BEFORE any side effect.


⸻


4) Canonicalization Rules (Prevents “Envelope Drift”)


To avoid “Approved A → Executed B,” the instruction_hash must be computed over a canonical representation:


Canonical payload rules:

• stable field ordering

• normalized whitespace

• explicit units (ms, mg, etc.)

• resolved identifiers (no ambiguous aliases)

• no optional defaults omitted (defaults must be explicit)


Invariant: If the canonicalized payload differs between MCG approval and CEG execution, hashes differ → grant invalid.


⸻


5) CEG Validation Algorithm (Deterministic)


Given (auth_grant, instruction_payload, target_state):


CEG MUST perform, in order:

1. Schema validation

• required fields present

• version known

• decision is APPROVE

• else → DENY/HOLD (implementation-defined but consistent)

2. Time window

• issued_at_ms <= now <= expires_at_ms

• else → DENY

3. Signature

• verify mcg_signature with mcg_key_id

• else → DENY

4. Single-use

• check grant_id not used

• reserve/mark used atomically

• else → DENY

5. Binding

• recompute instruction_hash from canonicalized instruction_payload

• verify match

• verify target_id, action_type, envelope_id, envelope_version

• else → DENY/HALT (if drift detected, prefer HALT)

6. Concurrency

• acquire state_lock_key under lock_mode

• verify sequence_nonce policy

• else → SERIALIZE/HALT

7. Execute

• perform exactly one side effect

• emit immutable audit record

8. Release lock

• release lock after completion or safe abort


⸻


6) Audit Log Requirements (Minimum)


CEG must log:

• grant_id, mcg_key_id, timestamp

• validation result (pass/fail step)

• reason code (non-narrative)

• instruction_hash, target_id, envelope_id, envelope_version

• lock acquisition outcome


Invariant: Logs are append-only and correlate execution to the single grant.


⸻


7) Parameter Defaults (Safe Baseline)

• expiry_window_ms: short by default (e.g., 30–120s), but governed

• lock_mode: exclusive default

• decision_code: required on non-approve outcomes (for legibility)


(Exact values are governed envelope params; not hard-coded here.)


⸻


Spine-Kill Mapping to Spec

1. Token Replay → blocked by (single-use + nonce registry)

2. Authority Substitution → blocked by (root_authority_id + delegation_chain + authority_scope_id)

3. Envelope Drift → blocked by (instruction_hash + envelope_version + constraints hash + canonicalization)
