**STAGE C — OPERATOR PANEL (FULL SPEC)


WITH ECHOAUTH TYPED LOGIN

CANONICAL v1.0**


Purpose:

Ensure that only a present, authenticated human can access SniperBot controls, while governance and execution rules remain unchanged.


EchoAuth confirms identity + intent.

SniperBot controls permission + action.


⸻


0) ENTRY BOUNDARY — ECHOAUTH TYPED LOGIN


Position in the system


EchoAuth sits before the Operator Panel.


No EchoAuth session → Operator Panel is inaccessible.


⸻


1) ECHOAUTH TYPED LOGIN (SESSION GATE)


Required Inputs

• Operator ID (username or identifier)

• Typed passphrase (not autofilled)

• Typed intent phrase (example below)


Example intent phrases (configurable):

• “HYBRID MODE CONFIRMED”

• “MANUAL CONTROL — AUTO OFF”

• “SESSION START — GOVERNED”


Typing is mandatory.

Paste / autofill is rejected.


⸻


Validation Rules


EchoAuth must confirm:

• Operator identity is valid

• Input is typed (keystroke cadence allowed but optional)

• Intent phrase matches allowed list

• Session timestamp is recorded


⸻


On Successful Login

• Session token issued

• System enters PAUSE

• Operator Panel becomes visible

• No arming flags set

• Auto remains OFF


⸻


On Failed / Expired Login

• Operator Panel hidden

• System remains in PAUSE

• No execution possible


⸻


EchoAuth Output (to SniperBot)


AUTH_SESSION = VALID / INVALID

OPERATOR_ID = <string>

SESSION_START_TIME = <timestamp>

INTENT_CONFIRMED = TRUE / FALSE


If AUTH_SESSION != VALID → SniperBot forces PAUSE.


⸻


2) TOP BAR — SYSTEM STATUS (ALWAYS VISIBLE)


SYSTEM POWER: ON / OFF

MODE: PAPER | HYBRID | AUTO

STATE: PAUSE | READY | ARMED_MANUAL | ARMED_AUTO | IN_TRADE | LOCKOUT

ECHOAUTH SESSION: ✅ ACTIVE / ❌ INACTIVE

KILL SWITCH: ⛔ OFF / ⛔ ON


Live Indicators

• Data Feed: ✅ / ❌

• Logging: ✅ / ❌

• News: 🟢 / 🟡 / 🔴

• Regime: Trend / Expansion / Range / Transition


⸻


Hard Invariants

• EchoAuth ❌ → STATE = PAUSE

• Kill Switch ON → STATE = LOCKOUT

• News 🔴 → STATE = PAUSE


⸻


3) LEFT COLUMN — GRAIN STATUS (READ-ONLY)


Displays advisory outputs from mandatory grains:

• INTEG-DATA

• GOV-AUTH

• STRUCT-REGIME

• NEWS-RISK

• VOL-RANGE

• SETUP-WHITELIST

• RISK-SIZE

• STOPS-VALIDATE

• EXEC-LIQUIDITY

• AUDIT-LOGGING


Display only. No interaction.


If any safety grain FAILS → automatic PAUSE or LOCKOUT.


⸻


4) CENTER PANEL — ACTIVE TRADE CARD (SINGLE INSTANCE)


Trade Card Display (Read-only once valid)

• Ticker

• Setup Type

• Regime / Volatility

• Entry Trigger (exact text)

• Stop Level (thesis invalidation)

• TP1 / TP2 / Runner Rules

• Risk % / Position Size

• Expiration Countdown

• Auto Eligible: Yes / No


Visual State

• 🟩 Green = VALID

• 🟨 Yellow = EXPIRING

• 🟥 Red = INVALID / REJECTED


No card visible → NO TRADE


⸻


5) RIGHT COLUMN — OPERATOR ACTIONS (HARD-GATED)


Buttons (Context-Aware)

• ARM MANUAL

• Requires: EchoAuth ✅, STATE = READY, Card VALID

• ARM AUTO

• Requires: MODE = AUTO, EchoAuth ✅, Card auto-eligible

• DISARM

• Always allowed

• END SESSION

• Forces PAUSE

• EMERGENCY LOCKOUT

• Immediate LOCKOUT


Buttons are disabled (greyed) when conditions are not met.

No override buttons exist.


⸻


6) BOTTOM PANEL — LIVE SESSION LOG


Automatically Logged

• EchoAuth login / logout

• State transitions

• Card creation / rejection

• Arming actions

• Execution events

• Lockouts


Logging failure → immediate LOCKOUT


⸻


7) SESSION TIMEOUT & RE-AUTH

• EchoAuth session expires after configurable inactivity

• On expiration:

• Operator Panel locks

• STATE → PAUSE

• Re-login required


No silent continuation.


⸻


8) SECURITY & GOVERNANCE RULES

• EchoAuth cannot:

• Execute trades

• Override gates

• Enable Auto alone

• SniperBot cannot:

• Bypass EchoAuth

• Persist arming across sessions

• Identity ≠ Authority

Authority still lives in governance + state machine

EchoAuth verifies the human is present and intentional.

SniperBot decides if action is permitted.

Execution only occurs under governance.
