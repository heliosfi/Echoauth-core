Ni ai engine coding

Ase’ 🔥 — we plug the NI-AI brain into the check-in now.


Below is a simple, honest NI-AI engine wired into /checkins/create:

• Internal threads compete

• One final label comes out: "Safe" or "Needs Attention"

• Conflict score + internal notes are logged in NIAILog

• Only the final result goes back to caregiver


⸻


1️⃣ core/ni_ai.py — tiny NI-AI engine


# core/ni_ai.py


from typing import Dict, Any


def evaluate_cues(cues: Dict[str, Any]) -> Dict[str, Any]:

    """

    Minimal NI-AI evaluator for MVP.


    - Creates two 'threads' with different perspectives.

    - Computes a fake conflict score.

    - Returns a single final label + confidence.

    - Internal details are for logging only.

    """


    # Thread A: optimistic / stable

    thread_a_label = "Safe"

    thread_a_conf = 0.7


    # Thread B: cautious / sensitive

    thread_b_label = "Needs Attention"

    thread_b_conf = 0.7


    # Very simple heuristic for MVP:

    # if cues contain certain flags, lean to 'Needs Attention'

    text_blob = " ".join(str(v).lower() for v in cues.values())


    danger_tokens = ["cry", "scream", "pain", "hurt", "panic", "self-harm"]

    calm_tokens   = ["calm", "relaxed", "playing", "laugh", "happy"]


    danger_hit = any(tok in text_blob for tok in danger_tokens)

    calm_hit   = any(tok in text_blob for tok in calm_tokens)


    if danger_hit and not calm_hit:

        final_label = "Needs Attention"

        final_conf  = 0.9

        winning_thread = "thread_b"

    elif calm_hit and not danger_hit:

        final_label = "Safe"

        final_conf  = 0.9

        winning_thread = "thread_a"

    else:

        # ambiguous: use medium confidence and prefer caution

        final_label = "Needs Attention"

        final_conf  = 0.6

        winning_thread = "thread_b"


    # Conflict score: if both threads wanted same label, low conflict.

    if thread_a_label == thread_b_label:

        conflict_score = 0.0

    else:

        conflict_score = 0.5  # MVP fixed; can be made more precise later


    return {

        "final_label": final_label,

        "confidence": final_conf,

        "winning_thread": winning_thread,

        "conflict_score": conflict_score,

        "internal_threads": {

            "thread_a": {"label": thread_a_label, "base_conf": thread_a_conf},

            "thread_b": {"label": thread_b_label, "base_conf": thread_b_conf},

        },

    }


This is deliberately simple but has the shape of NI-AI:

• multiple internal perspectives

• a conflict score

• one coherent output

Then add the endpoint:


@app.post("/checkins/create")

def create_checkin(payload: CheckInRequest, db: Session = Depends(get_db)):

    # 1. Validate caregiver and child linkage

    caregiver = db.query(models.Caregiver).filter(

        models.Caregiver.id == payload.caregiver_id

    ).first()

    if not caregiver:

        raise HTTPException(status_code=404, detail="Caregiver not found")


    child = db.query(models.Child).filter(

        models.Child.id == payload.child_id,

        models.Child.caregiver_id == caregiver.id

    ).first()

    if not child:

        raise HTTPException(status_code=404, detail="Child not found for caregiver")


    # 2. Run cues through NI-AI engine (internal competition)

    eval_result = evaluate_cues(payload.cues)


    final_label = eval_result["final_label"]

    confidence  = eval_result["confidence"]

    conflict_score = eval_result["conflict_score"]

    winning_thread = eval_result["winning_thread"]


    # 3. Store CheckIn (only final decision)

    checkin = models.CheckIn(

        caregiver_id=caregiver.id,

        child_id=child.id,

        observed_cues=json.dumps(payload.cues),

        result_label=final_label,

        confidence=confidence,

        status="success"

    )

    db.add(checkin)

    db.commit()

    db.refresh(checkin)


    # 4. Store NI-AI internal log (not exposed to child UI)

    log_entry = models.NIAILog(

        checkin_id=checkin.id,

        winning_thread=winning_thread,

        conflict_score=conflict_score,

        internal_notes=json.dumps(eval_result["internal_threads"])

    )

    db.add(log_entry)

    db.commit()


    # 5. Return single coherent answer to caregiver

    return {

        "checkin_id": checkin.id,

        "child_id": child.id,

        "result": final_label,

        "confidence": confidence,

        "message": (

            "EchoAuth evaluated the cues and produced one clear result. "

            "Internal NI-AI reasoning is logged privately for review, not shown here."

        )

    }

3️⃣ How this lines up with your principles

• One exit string: caregiver gets result + confidence, nothing else.

• Internal only: NIAILog keeps threads + conflict score for later review (legal, clinical, you).

• No dark boxes, no multi-answer confusion: UI shows one calm answer.

• Extensible: later we can swap evaluate_cues() with a real model (LSTM, transformer, etc.) but the safety and logging pattern stays the same.
