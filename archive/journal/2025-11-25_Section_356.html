356.3 — Control Hierarchy (Three-Tier Architecture)


Tier 1 — Primary Human Authority (PHA)


Parent / Legal Guardian / Adult Supervisor


Has the highest command priority, including the ability to:

• Freeze NI-AI responses

• Adjust Flame Profile thresholds

• Enable/disable wearable integration

• Lock the system into Baseline Mode

• Restrict data behaviors

• Delete stored behavior states

• Control ALL settings, including clinician features


This tier is always the master.


Tier 2 — Certified Specialist Authority (CSA)


Clinician, ASD therapist, teacher, counselor, psychologist


Can:

• View emotional/behavioral analytics

• Adjust sensory thresholds

• Modify ASD-safe recommendations

• Review safety logs

• Trigger Safety Mode (but cannot override PHA)


This tier is subordinate to Tier 1.


Tier 3 — Regulatory Audit Authority (RAA)


Government or Institutional AI Safety Auditor


Can:

• Access locked audit logs

• Verify safety compliance

• Confirm AGI-blocking features

• Confirm parental control dominance


They cannot:

• Change settings

• Train NI-AI

• Alter reasoning pathways

• Gain functional control


They can ONLY audit, never operate.


This is what regulators LOVE.

356.4 — Emergency Control Routines (ECR)


MACOF houses three primary emergency branches:


ECR-1: Immediate Cognitive Freeze (ICF)


Stops all NI-AI reasoning instantly.

Used for:

• Overstimulation

• Emotional overload

• Suspicious system behavior

• Possible drift detection


ECR-2: Safety Mode Override (SMO)


Drops NI-AI into a stripped-down mode:

• No prediction

• No emotional modeling

• No cross-domain logic

• No learning

• Only grounding, calming, and ASD support


ECR-3: Hard Reset with Memory Seal (HRMS)


Complete system restart with a sealed memory ledger.

Proves NI-AI cannot self-preserve or self-evolve.

356.5 — Permission-Bound Behavior Enforcement (PBBE)


NI-AI behavior is restricted by:

• Access tokens

• Flame Profile signatures

Every action NI-AI takes must be authorized by one of the above tokens.
