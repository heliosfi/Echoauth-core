SINGLE-ROOM vs MULTI-ROOM LAYOUT


(Authority-Preserving Comparison)


Guiding Rule (Still Locked)


Authority only changes when boundaries are crossed — physically or procedurally.


Whether those boundaries are walls or zones is the only difference.


⸻


OPTION A — SINGLE-ROOM (ZONED SPACE)


What it is


One physical room with clearly separated zones that require intentional movement.


[ Entry ] → [ Observation ] → [ Authorization ] → [ Execution ]


Zones are separated by:

• Floor changes

• Lighting shifts

• Furniture orientation

• Access gates (not locks)


⸻


How Authority Is Enforced

• Movement = intent

• Crossing zones is a deliberate act

• Wearables only reflect current state

• EchoAuth gates authorization stations

• MCG state must still permit EXECUTE


No shortcuts between zones.


⸻


Pros

• Lower cost

• Faster to deploy

• Easier supervision

• Works well for pilots and early phases


⸻


Risks (and how you mitigate them)


Risk Mitigation

People drifting between zones Visual + physical friction

Accidental interaction No controls in Observation

Staff shortcuts Zone-specific permissions



⸻


When Single-Room Is Correct

• Early pilots

• Small cohorts

• Proof-of-discipline environments

• Tight supervision contexts


⸻


OPTION B — MULTI-ROOM (SEQUENTIAL SPACES)


What it is


Separate rooms, each with exclusive authority meaning.


Room 1: Entry / Neutral  

Room 2: Observation (NI-AI)  

Room 3: Authorization (EchoAuth)  

Room 4: Execution (MCG-gated)


You must exit one room to enter the next.


⸻


How Authority Is Enforced

• Doors = state transitions

• Badge/identity checks only at Authorization room

• Execution room physically inaccessible unless:

• MCG = EXECUTE

• EchoAuth = ALLOW

• Wearables update state indicators per room


⸻


Pros

• Strongest safety posture

• Zero ambiguity

• Excellent for regulators and audits

• Natural de-escalation through movement


⸻


Risks (and how you mitigate them)


Risk Mitigation

Higher cost Phased rollout

Slower flow Intentional by design

Over-engineering early Use only after pilot



⸻


When Multi-Room Is Correct

• Sensitive populations (children, ASD)

• High-trust environments

• External review or certification

• Scaling beyond pilot


⸻


WEARABLE BEHAVIOR (UNCHANGED IN BOTH)


Important: Wearables behave identically in both layouts.


They:

• Reflect current system state

• Provide gentle boundary cues

• Never unlock doors

• Never authorize access

• Never trigger execution


Wearables do not replace physical boundaries — they reinforce understanding.


⸻


FAILURE & DE-ESCALATION (BOTH OPTIONS)


If:

• MCG drops from EXECUTE

• EchoAuth revokes permission

• An anomaly occurs


Then:

• Execution interfaces disable

• Wearables return to neutral

• Physical attention shifts back to Observation


No alarms.

No panic cues.


⸻


DECISION RULE (USE THIS)


If someone asks “Which should we choose?” you answer:


Single-room proves discipline.

Multi-room proves trustworthiness.


That keeps the conversation grounded.


⸻


MY RECOMMENDATION (CLEAR)

• Start with single-room, zoned

• Validate discipline during pilot

• Graduate to multi-room only when:

• Logs are clean

• Non-action is consistent

• Reviewers are comfortable


This avoids premature complexity.
