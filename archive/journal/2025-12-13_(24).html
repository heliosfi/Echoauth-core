1-PAGE TECHNICAL DIAGRAM NARRATIVE


EchoAuth × NI-AI — Safety-First, Non-Verbal Infrastructure


Diagram Reading Guide (How to Read This System)


This system is intentionally layered so that no single component can cause action. Authority is separated, execution is denied by default, and non-verbal interaction is preserved without interpretation.


The diagram should be read left to right, then top to bottom.


⸻


Layer 1 — Environment & Presence (MCG)


MCG (Motion / Context Governance) is the foundation layer.

• Manages physical presence, time, and transitions

• Observes where and when, not why

• Does not infer intent, emotion, or mental state

• Treats stillness, silence, and non-participation as valid states


Key principle:


Presence is observed, not interpreted.


This layer ensures non-verbal individuals are never required to explain or perform for the system to remain safe.


⸻


Layer 2 — Authorization & Boundaries (EchoAuth)


EchoAuth sits above MCG and enforces authority discipline.

• Default state is deny execution

• Action requires:

• Correct system state

• Explicit authorization

• Human oversight

• Staff cannot bypass boundaries

• Failure collapses capability back to observation


Key principle:


Nothing happens just because something is detected.


This layer prevents escalation driven by uncertainty or misinterpretation.


⸻


Layer 3 — Explanation & Transparency (NI-AI)


NI-AI is a read-only explanation layer.

• Explains what the system sees and why it did not act

• Provides transparency for caregivers, auditors, and reviewers

• Does not:

• Predict behavior

• Score individuals

• Control execution


Key principle:


AI explains decisions; it does not make them.


This preserves trust without introducing opaque logic.


⸻


Optional Layer — Wearables (Passive Only)


Wearables, when present:

• Reflect system state

• Provide predictable orientation cues

• Collect minimal, non-diagnostic signals

• Have no authority, no triggers, no inference pathways


The system functions safely with or without wearables.


Key principle:


Wearables inform awareness; they never drive outcomes.


⸻


Execution Boundary (Hard Stop)


Between authorization and any execution pathway is a hard boundary:

• Execution is simulated or stubbed in pilots

• Real-world action requires separate approval

• Non-action is logged as a successful outcome


Key principle:


The safest action is often no action.


⸻


Audit & Logging (Always On)


Every state transition, authorization, denial, and pause is:

• Logged

• Time-stamped

• Reviewable

• Auditable


This enables accountability without surveillance.


⸻


Non-Verbal Guarantee (Explicit)


Across all layers:

• No speech required

• No verbal confirmation expected

• No interpretation of silence

• No pressure to respond


Silence is a valid signal.

Non-participation is not failure.


⸻


Diagram One-Line Summary (Use This Caption)


A safety-first system that supports non-verbal interaction by enforcing restraint, separating authority, and treating non-action as success.
