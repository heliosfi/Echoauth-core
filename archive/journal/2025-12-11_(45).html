🔬 Layer 0 — Bio-Perceptual Layer (BPL)


Purpose:

Turn non-verbal cues into a small set of numbers that the 3-layer governance stack can read.


Inputs (examples):

• Movement: still / fidgety / agitated

• Posture: upright / slumped / turned away

• Gaze: engaged / avoiding / scanning

• Voice: calm / shaky / loud / flat (if voice is available)

• Rhythm: sudden pauses / repetitive motions

• Environment: noise level / brightness / crowding


Output: a simple state bundle:


bio_state = {

  "arousal": 0.0–1.0,      # low → high activation

  "stress": 0.0–1.0,       # low → high tension

  "engagement": 0.0–1.0,   # checked out → tuned in

  "overload": 0.0–1.0      # comfortable → overwhelmed

}


No magic. Just 4–5 numbers.


⸻


🔁 How BPL feeds the 3 Layers


Navigator reads bio_state to answer


“Where is this heading?”


• High arousal + rising overload → “toward meltdown / shutdown”

• Low arousal + low engagement → “toward disengagement / drift”

• Medium arousal + stable engagement → “toward steady focus”


Guardian reads bio_state to answer


“What must NOT happen?”


• stress > 0.7 → no new demands

• overload > 0.8 → no sudden transitions

• engagement < 0.2 + stress high → avoid forcing interaction


Architect uses bio_state + the two above to answer


“What do we do now + next?”


• If overload high → lower stimulation, give break

• If engagement low, stress low → simplify + re-engage gently

• If arousal < 0.2 (under-stim), stress low → maybe add positive stimulation
