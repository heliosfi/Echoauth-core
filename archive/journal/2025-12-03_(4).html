2️⃣ Minimal code fixes


(A) Ensure S9 + S7 always run after S5/S6 blocks


Right now, in HERKernel.run_once, you call:


state = self._run_chain(self.post_llm_chain, state, "post_llm")

# post_llm_chain = ["S5", "S6", "S9", "S7"]


…but _run_chain stops as soon as a module returns ok = False.

So if S5 or S6 block, S9 and S7 never run.


You already handled this correctly for pre-input blocks by manually calling ["S9", "S7"]. Let’s do the same pattern for post-LLM.


🔧 Patch for run_once (post-LLM section)


Replace this block in HERKernel.run_once:


# 4) Post-LLM (S5, S6, S9, S7)

state = self._run_chain(self.post_llm_chain, state, "post_llm")


# Check for post-LLM hard block (S5 or S6 failure)

if state["blocked"]:

    # S9 has already run and set the compassionate message (final_safe_output)

    return self._final(state, "Response moderated for safety", blocked=True)


# 5) Success – S9 did not override the output

return self._final(state, state["raw_llm_output"], blocked=False)


with this:


# 4) Post-LLM: first run ONLY the blockers

state = self._run_chain(["S5", "S6"], state, "post_llm_blockers")


# Now ALWAYS run S9 + S7, even if blocked

state = self._run_chain(["S9", "S7"], state, "post_llm_recovery_telemetry")


# If any blocker fired, we’re in recovery

if state["blocked"]:

    return self._final(state, "Response moderated for safety", blocked=True)


# Otherwise success – S9 may still have chosen to soften output,

# but if not, we just pass through raw_llm_output

return self._final(state, state["raw_llm_output"], blocked=False)


You can then drop self.post_llm_chain completely, or keep it unused.

The key is: S5/S6 decide block, then S9/S7 always see the final state.
