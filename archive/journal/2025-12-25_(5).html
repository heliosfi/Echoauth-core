STAGE 2 — v1.0 IMPLEMENTATION ORDER (BUILD SEQUENCE)


Goal: build SniperBot in the safest order, where each step is testable alone and nothing can drift.


Each stage ends with a pass condition. If it doesn’t pass, you don’t advance.


⸻


PHASE 0 — Chamber Foundation (No Trading Yet)


Step 0.1 — State Machine Core


Build: Page 16 FSM

• States: PAUSE, READY, ARMED_MANUAL, ARMED_AUTO, IN_TRADE, LOCKOUT

• Transitions + downgrade rules


✅ Pass when:

• You can simulate: PAUSE→READY, READY→PAUSE, ANY→LOCKOUT


Grains involved: none yet (skeleton only)


⸻


Step 0.2 — Safety Spine (Mandatory First Grains)


Implement:

1. GRAIN-INTEG-DATA-v1

2. GRAIN-AUDIT-LOGGING-v1


✅ Pass when:

• If data fails → LOCKOUT always

• If logging fails → LOCKOUT always

• Every state transition logs


This prevents building a “silent bot.”

PHASE 1 — No-Trade Intelligence (Paper Mode Engine)


Step 1.1 — Governance Gate


Implement:

3) GRAIN-GOV-AUTH-v1 (arming + kill switch)


✅ Pass when:

• Without arming → system never leaves READY into execution states

• Kill switch works instantly

Step 1.2 — Context Classifiers


Implement:

4) GRAIN-STRUCT-REGIME-v1

5) GRAIN-NEWS-RISK-v1

6) GRAIN-VOL-RANGE-v1


✅ Pass when:

• Regime not Trend/Expansion → PAUSE

• News Red → PAUSE

• Volatility unknown → PAUSE


Now the chamber knows when not to care.

Step 1.3 — Setup Detection (Whitelist Only)


Implement:

7) GRAIN-SETUP-WHITELIST-v1


✅ Pass when:

• TradeCard object + creation pipeline (Page 18)

At this point you have a full Paper Mode system.
