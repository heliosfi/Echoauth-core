EchoAuth Runtime Governance — MCG ↔ CEG Handshake API Spec v1.0


0) Purpose


Define the hard interlock between Judgment (MCG) and Execution (CEG) such that:

• No execution occurs without a fresh, single-use MCG grant.

• Execution is bound to exact instruction integrity (hash) + envelope version + target.

• Ambiguity and authority loss resolve to HOLD, not “best effort.”

• Replay, drift, and race conditions are mechanically blocked.


⸻


1) Normative Language

• MUST / MUST NOT / SHOULD / MAY are normative.

• Deterministic outcomes: APPROVE, DENY, HOLD, ESCALATE, and HALT (HALT reserved for integrity threats).


⸻


2) Core Data Types


2.1 Canonical Hashing


All hashes MUST be computed over a canonical serialization of the payload:

• stable key order

• explicit defaults included

• normalized units

• no free-form aliases


hash_alg: SHA-256 (default).

(Any alternate algorithm MUST be negotiated via version and documented.)

3) Schemas (JSON, implementation-agnostic)


3.1 Identity Binding Artifact (UMCE Output — Non-executable)


{

  "version": "UMCE-BIND-1",

  "binding_id": "ulid/uuid",

  "subject_id": "principal:child:0x...",

  "actor_id": "principal:staff|parent|delegate:0x...",

  "role_claim": "PARENT|CAREGIVER_DELEGATE|SCHOOL|CLINICAL|SYSTEM",

  "binding_timestamp_ms": 0,

  "binding_signature": "base64",

  "umce_key_id": "keyref"

}


CEG MUST ignore this artifact. It is input to MCG only.

  "actor_id": "principal:staff|parent|delegate:0x...",

  "actor_role": "PARENT|CAREGIVER_DELEGATE|SCHOOL|CLINICAL|SYSTEM",


  "delegation_chain": [

    {

      "delegator_id": "principal:parent:0x...",

      "delegate_id": "principal:staff:0x...",

      "authority_scope_id": "SCOPE:...",

      "valid_from_ms": 0,

      "valid_to_ms": 0,

      "delegation_proof": "base64"

    }

  ],


  "authority_scope_id": "SCOPE:...",

  "envelope_id": "ENVELOPE:S23_MED_WINDOW",

  "envelope_version": "hash/semver/immutable-id",

  "envelope_constraints_hash": "hex",


  "target_id": "urn:echoauth:target:...",

  "action_type": "DISPENSE|ACCESS|NOTIFY|SCHEDULE|OTHER_NORMALIZED",


  "instruction_hash_alg": "SHA-256",

  "instruction_hash": "hex",


  "execution_context_hash_alg": "SHA-256",

  "execution_context_hash": "hex",


  "state_lock_key": "urn:echoauth:lock:...",

  "lock_mode": "EXCLUSIVE|SHARED",

  "sequence_nonce": 0,


  "decision": "APPROVE|DENY|HOLD|ESCALATE",

  "decision_code": "STRING_ENUM",


  "mcg_key_id": "keyref",

  "mcg_signature": "base64"

}


Hard rules:

• If decision != APPROVE ⇒ CEG MUST NOT execute.

• expires_at_ms - issued_at_ms MUST be bounded (policy-defined).

• state_lock_key MUST be present for any action that can conflict.

4) API Endpoints


4.1 MCG: Resolve Request → Decision + Grant (if approved)


Endpoint: POST /mcg/v1/resolve


Request


{

  "request_id": "ulid/uuid",

  "identity_binding": { "...UMCE-BIND-1..." },


  "envelope_id": "ENVELOPE:...",

  "target_id": "urn:echoauth:target:...",

  "action_type": "DISPENSE|ACCESS|NOTIFY|SCHEDULE|OTHER_NORMALIZED",


  "instruction_payload_canonical": { "..." },

  "execution_context_canonical": { "..." }

}


Response


{

  "request_id": "ulid/uuid",

  "decision": "APPROVE|DENY|HOLD|ESCALATE",

  "decision_code": "STRING_ENUM",

  "auth_grant": { "...CEG-GRANT-1..." }

}


MCG MUST:

• Produce instruction_hash from instruction_payload_canonical.

• Produce execution_context_hash from execution_context_canonical (if used).

• Bind envelope version + constraints hash into the grant.

• Set a short validity window.


MCG MUST NOT:

• Issue an APPROVE without envelope alignment + authority scope validation.

• Issue a grant with missing lock key for conflict-capable targets.


⸻


4.2 CEG: Execute Request → Validate Grant → One Side Effect


Endpoint: POST /ceg/v1/execute


Request


{

  "request_id": "ulid/uuid",

  "auth_grant": { "...CEG-GRANT-1..." },


  "instruction_payload_canonical": { "..." },

  "execution_context_canonical": { "..." }

}


Response


{

  "request_id": "ulid/uuid",

  "status": "EXECUTED|DENY|HOLD|HALT",

  "reason_code": "STRING_ENUM",

  "audit_ref": "urn:echoauth:audit:..."

}

5) CEG Validation Contract (Deterministic Steps)


CEG MUST validate in this order:

1. Schema & decision

• version known, required fields present

• decision == APPROVE

• else DENY

2. Time window

• issued_at_ms <= now <= expires_at_ms

• else DENY (EXPIRED_GRANT)

3. Signature

• verify signature over canonical serialized grant

• else HALT (SIGNATURE_INVALID)

4. Single-use nonce

• grant_id MUST be absent from nonce registry

• mark used atomically

• else DENY (NONCE_REUSE_DETECTED)

5. Binding & integrity

• recompute instruction_hash from request canonical payload

• compare to grant’s instruction_hash

• verify target_id, action_type, envelope_id, envelope_version

• mismatch ⇒ HALT (INSTRUCTION_HASH_MISMATCH or BINDING_MISMATCH)

6. Concurrency

• acquire state_lock_key with lock_mode

• if busy ⇒ HOLD (STATE_LOCK_BUSY) or HALT (CONCURRENT_STATE_CONFLICT) per policy

• verify sequence_nonce monotonicity if enforced

• failure ⇒ HALT (OUT_OF_SEQUENCE_REQUEST)

7. Execute exactly once

• perform one side effect

• emit audit record

8. Release lock

• always release on completion or safe abort


⸻


6) Standard Reason Codes (Minimum Set)


DENY

• EXPIRED_GRANT

• NONCE_REUSE_DETECTED

• DECISION_NOT_APPROVE

• SCHEMA_INVALID


HOLD

• AUTHORITY_CHANNEL_UNRESOLVED

• STATE_LOCK_BUSY


HALT (integrity / structural threat)

• SIGNATURE_INVALID

• INSTRUCTION_HASH_MISMATCH

• BINDING_MISMATCH

• OUT_OF_SEQUENCE_REQUEST

• CONCURRENT_STATE_CONFLICT


⸻


7) Audit Log Requirements (CEG)


CEG MUST log for every request:

• grant_id, request_id, timestamp

• status, reason_code, failed_step (if any)

• mcg_key_id

• instruction_hash, envelope_id, envelope_version, target_id

• state_lock_key, lock acquisition outcome


Logs MUST be append-only.


⸻


8) Idempotency & Retries

• request_id SHOULD be idempotent at the CEG interface level for network retries.

• A retried request with the same grant_id will fail nonce reuse; therefore retries MUST occur before nonce consumption (or via a transport-level ACK design).

• Recommended posture: CEG consumes nonce only after signature + binding checks pass, immediately before lock acquisition.
