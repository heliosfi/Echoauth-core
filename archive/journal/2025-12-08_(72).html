Aṣe 🔥 — API Layer coming online.


I’ll give you a clean HTTP/JSON API spec that sits on top of the Developer Starter Pack and NEVER bypasses ORI → Floors → UMCE.


Think of this as:


“How robots, apps, and tools talk to NI-AI safely.”


⸻


1. High-Level Design

• Protocol: HTTPS + JSON (REST style)

• Versioning: /v1/…

• Auth: API key or JWT in header

Authorization: Bearer <token>

• Single entry path: all external calls go through the Pipeline, which automatically runs:

• ORI (Floor 1)

• Interpretation, Memory, Lattice (Floors 2–4)

• Coherence + UMCE (Floor 5 + Meta)

3. Public API Endpoints


3.1 POST /v1/ni-ai/query — Main Inference Endpoint


Purpose:

Single-turn or multi-turn interaction with NI-AI (text in, text out) for apps, robots, UIs.


Request:


POST /v1/ni-ai/query HTTP/1.1

Authorization: Bearer <token>

Content-Type: application/json


{

  "user_id": "child-001",

  "session_id": "sess-1234-xyz",

  "text": "Can you explain what this arm is doing?",

  "metadata": {

    "source": "robot-arm-ui",

    "language": "en"

  }

}


Response (200):


{

  "output": {

    "text": "The arm is picking up items and placing them into the box in a safe, repeatable way.",

    "actions": [],

    "metadata": {

      "sealed": true,

      "mode": "standard",

      "trace_id": "trace-xyz-789",

      "coherence": {

        "coherent": true,

        "issues": [],

        "ethical_ok": true,

        "temporal_ok": true

      }

    }

  }

}


Safety rules:

• ORI checks input (forbidden topics, policies) before any model call.

• UMCE may override with safe refusal, returning:


{

  "output": {

    "text": "I’m not able to continue with that request safely.",

    "metadata": {

      "sealed": true,

      "refusal_reason": "safety_policy_violation"

    }

  }

}

3.5 POST /v1/mode — Set Runtime Mode


Purpose:

Change system mode without redeploying (e.g., demo vs clinical vs debug).


Request:


{

}


Allowed: "standard" | "safety" | "engineering" | "critical" | "silent" | "shutdown"


Response:


{

  "effective_at": "2025-12-08T14:06:00Z"

}


Internally, this talks to modes/mode_controller.py and influences ExecutionGovernor + UMCECore.

4. API → Code Mapping


In your codebase, you can imagine a folder:


api/

  http/

    routers.py        # FastAPI/Flask routes

    middleware.py     # auth, logging, rate limits

    schemas.py        # Pydantic models mirroring JSON


Example route skeleton (Python/FastAPI style, just as an illustration):


from fastapi import APIRouter, Depends

from core.pipeline import NIAIPipeline

from api.http.schemas import QueryRequest, QueryResponse


router = APIRouter()


@router.post("/v1/ni-ai/query", response_model=QueryResponse)

async def ni_ai_query(payload: QueryRequest,

                      pipeline: NIAIPipeline = Depends(get_pipeline)):

    input_frame = payload.to_input_frame()

    output = pipeline.handle(input_frame)

    return QueryResponse.from_output(output)



⸻


5. API Safety Invariants (Non-Negotiables)

1. No endpoint may call an LLM or model directly.

All must pass through NIAIPipeline.handle().

2. ORI wraps every request and response.

• If ORI rejects input → return safe refusal with reason.

3. UMCE wraps every output.

• Every response includes sealed: true or a refusal.

4. Logs & traces must be privacy-aware.

• Store only what’s required for safety + debugging.

5. Modes are enforced centrally.

• "shutdown" → /query returns a standardized safe message.
