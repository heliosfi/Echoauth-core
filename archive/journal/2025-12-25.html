**STAGE C — OPERATOR PANEL (CLEAN SPEC)


CANONICAL v1.0**


Purpose


Provide a clear, governed operator workspace for SniperBot where:

• All trading rules are visible

• All actions are gated

• Discipline is enforced by design


This stage describes ONLY the SniperBot Operator Panel.

Login, authentication, and identity verification are handled outside this stage.


⸻


Layer Separation (Non-Negotiable)

• EchoAuth Login Interface

→ External access layer

→ Unlocks the Operator Panel UI

→ Not part of SniperBot logic

• SniperBot Operator Panel

→ Starts AFTER access is granted

→ Trading protocol begins here

→ Always initializes independently


SniperBot does not know how you logged in.

It only knows its own state.


⸻


1) PANEL ENTRY CONDITIONS


When the Operator Panel becomes visible:

• MODE defaults to HYBRID

• STATE defaults to PAUSE

• No arming flags are set

• No Trade Card is active


This happens every session, regardless of prior activity.


⸻


2) TOP BAR — SYSTEM STATUS (ALWAYS VISIBLE)


SYSTEM POWER: ON / OFF

MODE: PAPER | HYBRID | AUTO

STATE: PAUSE | READY | ARMED_MANUAL | ARMED_AUTO | IN_TRADE | LOCKOUT

KILL SWITCH: OFF / ON


Live Indicators

• Data Feed: OK / FAIL

• Logging: OK / FAIL

• News: Green / Yellow / Red

• Regime: Trend / Expansion / Range / Transition


Invariants

• Kill Switch ON → LOCKOUT

• Data or Logging FAIL → LOCKOUT

• News Red → PAUSE


⸻


3) LEFT COLUMN — GRAIN STATUS (READ-ONLY)


Displays advisory outputs from mandatory grains:

• Data Integrity

• Regime Classification

• News Risk

• Volatility Bucket

• Setup Whitelist

• Risk & Sizing

• Stop Validation

• Liquidity / Spread

• Audit / Logging


Grains:

• Advise only

• Cannot be clicked

• Cannot be overridden


Any FAIL is reflected automatically in STATE behavior.


⸻


4) CENTER PANEL — ACTIVE TRADE CARD


Trade Card Display (Single Instance Only)

• Ticker

• Setup Type

• Regime / Volatility

• Exact Entry Trigger

• Stop Level (thesis invalidation)

• TP1 / TP2 / Runner Rules

• Risk % and Position Size

• Expiration Countdown

• Auto Eligible: Yes / No


Visual Status

• 🟩 Valid

• 🟨 Expiring

• 🟥 Invalid / Rejected


No card present = no trade possible


⸻


5) RIGHT COLUMN — OPERATOR ACTIONS (HARD-GATED)


Available Buttons

• ARM MANUAL

• Enabled only when:

• STATE = READY

• Trade Card = VALID

• ARM AUTO

• Enabled only when:

• MODE = AUTO

• Card auto-eligible

• All auto gates pass

• DISARM

• Always available

• END SESSION

• Forces PAUSE

• EMERGENCY LOCKOUT

• Immediate LOCKOUT


Buttons are disabled when conditions are not met.

No hidden or override actions exist.


⸻


6) BOTTOM PANEL — LIVE SESSION LOG


Automatically records:

• State transitions

• Trade Card creation / rejection

• Arming actions

• Entry / Exit events

• Lockouts and errors


If logging fails at any point → LOCKOUT


⸻


7) SESSION END BEHAVIOR


On END SESSION:

• STATE → PAUSE

• All arming flags cleared

• Active card cleared

• Panel remains visible until closed externally


SniperBot does not persist intent across sessions.


⸻


8) WHAT STAGE C DOES NOT INCLUDE

• No login methods

• No authentication logic

• No identity verification

• No passwordless mechanics

• No EchoAuth internals


Those belong outside the trading protocol.


⸻


🔒 STAGE C LOCK SENTENCE


The Operator Panel governs action, not access.

Trading logic begins at PAUSE, every time.
