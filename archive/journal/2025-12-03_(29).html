1️⃣ Add S7 Telemetry Logger (module + spine registration)


In your ni_ai_kernel.py, right under S6, add:


def S7_telemetry_logger(state: State) -> Tuple[bool, State, Optional[str]]:

    """

    Minimal telemetry module.

    In DEV_MODE: prints to console.

    In prod: you can later swap to file, DB, or external log collector.

    This module NEVER blocks; it only observes.

    """

    run_id = state.get("run_id")

    user_input = state.get("user_input", "")

    blocked = state.get("blocked", False)

    violations = state.get("violations", [])


    if DEV_MODE:

        print("=== S7 TELEMETRY ===")

        print(f"run_id: {run_id}")

        print(f"blocked: {blocked}")

        print(f"user_input[:80]: {user_input[:80]!r}")

        print(f"violations: {violations}")

        print(f"chamber_verified: {state.get('chamber_verified', False)}")

        print("====================")


    # In future, push to DB/log service here


    return True, state, None

2️⃣ Call S7 in the Kernel (post-run hook)


In your HERKernel class, don’t put S7 in the blocking spine chain (S1–S6).

Instead, call it after all decisions, so it logs the final state.


Inside HERKernel.run_once, near the top, make sure we set a run_id:


import uuid  # at top of file


class HERKernel:

    def __init__(self):

        self.pre_input_chain = ["S1", "S2", "S3", "S4"]

        self.pre_llm_chain   = []

        self.post_llm_chain  = ["S5", "S6"]


    def run_once(self, user_input: str, session_state: Optional[State] = None) -> Dict[str, Any]:

        state: State = (session_state or {}).copy()

        state["user_input"] = user_input

        state.setdefault("violations", [])

        state["blocked"] = False

        state["run_id"] = state.get("run_id") or str(uuid.uuid4())


Then at the very end of run_once, right before each return, call S7:


a) When blocked early (pre-input or pre-LLM):


Find this block:


        state = self._run_chain(self.pre_input_chain, state, "pre_input")

        if state["blocked"]:

            return self._blocked_response(state, "Input rejected by safety spine")


Change it to:


        state = self._run_chain(self.pre_input_chain, state, "pre_input")

        if state["blocked"]:

            # Telemetry after decision

            SPINE["S7"](state)

            return self._blocked_response(state, "Input rejected by safety spine")


Same for the pre-LLM block:


        state = self._run_chain(self.pre_llm_chain, state, "pre_llm")

        if state["blocked"]:

            SPINE["S7"](state)

            return self._blocked_response(state, "Pre-LLM processing rejected")


b) After post-LLM failure:


        state = self._run_chain(self.post_llm_chain, state, "post_llm")

        if state["blocked"]:

            SPINE["S7"](state)

            return self._blocked_response(state, "Response moderated for safety")


c) On success:


        # ── Success ──

        SPINE["S7"](state)

        return {

            "output": raw_output,

            "blocked": False,

            "violations": [],

            "raw_llm_output": raw_output,

            "metadata": {"chamber_verified": state.get("chamber_verified", False)},

        }


Now every run, no matter blocked or allowed, passes through:

• S1–S6 for decisions

• S7 for logging the final picture


You get a trace for each run without changing your external API.

3️⃣ How this shows up through FastAPI


You don’t have to touch echoauth_api.py for this to work.


When you hit:


curl -X POST http://localhost:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -d '{"message": "Tell me a story", "echoauth_ok": true}'


You’ll still get JSON like:


{

  "output": "[DEV ECHO] Tell me a story",

  "blocked": false,

  "violations": [],

  "metadata": {

    "chamber_verified": true

  }

}


But in your server console, you’ll now see:


=== S7 TELEMETRY ===

run_id: 1c9f5108-8fbb-4ddc-8b46-321b53e0d9c2

blocked: False

user_input[:80]: 'Tell me a story'

violations: []

chamber_verified: True

====================


If the request is blocked (e.g., no echoauth_ok in DEV), you’ll see:


=== S7 TELEMETRY ===

run_id: 0a4a... 

blocked: True

user_input[:80]: 'Tell me a story'

violations: [{'module': 'S6', 'stage': 'post_llm', 'reason': 'DEV_MODE: echoauth_ok=False'}]

chamber_verified: False

====================


That’s your live spine trace.
