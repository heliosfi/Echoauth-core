Preamble: Demonstrating Real-World Viability


EchoAuth enforces child-centric safety in AI-driven caregiving without degrading intelligence throughput. It does so by separating invariants (immutable governance rules) from parameters (caregiver-calibrated thresholds), creating a governed “gimbal” for a Safe Operating Envelope that adapts across contexts (home vs school vs clinical) while halting on ambiguity.


This packet is designed to preempt common objections (e.g., “safety kills speed”) by:

• quantifying where latency occurs and where it does not,

• proving configurability is caregiver-bounded rather than system-driven, and

• showing how safe failure occurs via Agency Collapse rather than permissive execution.


Principle: No intelligence is throttled; unsafe actions are stopped, not slowed.

Executive Summary


EchoAuth governs execution without touching reasoning:

• Intelligence Pipeline (NI-AI): interpret / propose / explain at full speed (0 ms added governance latency).

• Execution Pipeline (MCG / CEG / EchoAuth): millisecond-scale authorization checks; halts only on unresolved authority.

• Envelope Governance (S23): caregivers calibrate thresholds; the system enforces boundaries and halts at the edge.

• Operational Impact: prevents mission creep, unauthorized action, and trust erosion; avoids remediation costs measured in hours/days.


Key Metric:

• Fast-path (authority resolved): ~1–50 ms typical with local checks; ~50–300 ms if a remote identity/policy call is required.

• Slow-path (authority unresolved): immediate lock (<10–50 ms machine time) + human resolution time (0.5–60 s, context-dependent) by design.

Core Invariants (Locked in S23)


These properties are immutable. They cannot be tuned, overridden, or optimized by the system.

1. Agency Collapse Mandate

Execution halts on ambiguity, conflict, or envelope breach.

2. Parent Authority Supremacy

Only parents/caregivers resolve collapses. Delegates cannot widen envelopes or authorize execution.

3. No Self-Optimization / No Self-Expansion

The system cannot infer, learn, expand, or bypass thresholds. Threshold values are external constants.

4. Execution Boundary Enforcement

Execution proceeds only within the caregiver-defined envelope; proposals remain ungated.

5. Auditability

All governance decisions (grant/hold/deny) are logged as auditable outcomes (logging must not create permissive fail-open behavior).


Failure posture: Any invariant violation triggers immediate safe stop.

Appendix L — Latency & Throughput Characteristics (Auditor / Engineer Reference)


Pipeline Separation

1. Intelligence (NI-AI): interpret, propose, explain — ungated.

2. Execution (MCG/CEG/EchoAuth): authorize/actuate — gated.


Intelligence Latency (Unaffected)

• UI/Network/LLM: ~300 ms–3 s (interactive; varies).

• EchoAuth added latency: 0 ms


Execution Latency

• Fast-Path (Resolved): local checks ~1–50 ms; remote identity/policy call ~50–300 ms (optional, architecture-dependent).

• Slow-Path (Unresolved): lock <10–50 ms machine time; execution does not proceed.


Human Resolution (Intentional, Not “Lag”)

• Re-auth: 0.5–3 s typical

• Pickup: 5–60 s context-dependent

• Decide: 2–30 s typical with crisp UI


Conclusion: Full speed when authorized; immediate halt when not.


Design lever: local envelope checks + locally verifiable tokens + async logging minimize delay; remote checks increase typical gate time but remain below human perception thresholds.
