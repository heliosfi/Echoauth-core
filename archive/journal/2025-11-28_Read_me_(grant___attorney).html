📘 EchoAuth™ MVP — README


Inventor: Nicholas B. Carty

Design Philosophy: Caregiver-governed, identity-first, conflict-collapsed pediatric safety authentication for non-verbal autistic children.


⸻


Overview


EchoAuth™ is a neuro-inclusive authentication and caregiver check-in system that converts real-world non-verbal behavior and environmental observations into identity tokens before AI evaluation. A lightweight NI-AI engine evaluates cues through internal reasoning paths, collapses disagreement internally, enforces one clear output per caregiver event, and automatically flags any risk-sensitive or uncertain check-ins to a dedicated human review queue. All resolution details and competing inference threads are written to private local audit logs for legal or clinical review and are never exposed to child-facing UI surfaces.


⸻


MVP Pilot Behavior


This repository includes:

• A local SQLite database for caregiver, child, check-in, and NI-AI decision logs.

• A cue intake doctrine that preserves caregiver authority and child identity stability.

• An adaptive evaluator that produces categorical labels (Safe / Needs Attention) with confidence_score and emotion/distress_score (0.0–1.0).

• A conflict flag + manual review loop for any event meeting automatic review thresholds.


⸻


Local Proof-of-Use Demo


Run the MVP locally using:


python run_local_demo.py


This script will:

1. Initialize or recreate the local database.

2. Create a demo caregiver and child profile if none exist.

3. Evaluate a caregiver-provided cue packet through NI-AI logic.

4. Collapse internal disagreement before output.

5. Store check-in results and conflict scores privately in audit logs.

6. Print the single, final decision to the terminal.

7. Mark and queue any high-risk or low-confidence events as pending review for human oversight.


⸻


Review Queue Access (Caregiver/Clinician Only)


Pending flags can be queried via the API surface (if server is running):


GET /checkins/flagged?caregiver_id={id}

POST /checkins/{id}/review


Human reviewers may:

• Document what occurred in reviewer_notes.

• Mark the event reviewed without surfacing internal NI-AI reasoning in the child UI.


⸻


Core Technical Principles

• Identity First: Child identity tokens are created from caregiver cues prior to inference.

• One Output Only: No multi-result answers are returned to child UI.

• Internal Conflict Only: Reasoning threads compete internally; winner is enforced; others collapse.

• Audit-Safe Logging: Logs persist locally for legal/clinical verifiability; never UI-surfaced.

• Danger-Aware Heuristics: Real-world behaviors and sensory overload signals influence labels honestly.

• Human in Loop: Uncertainty or risk automatically routes to reviewer queue for caregiver or clinician resolution.
