🔥 PHASE 1 — SETUP & CALIBRATION (Days 1–5)


Objectives:

• Configure the prototype in a live environment

• Calibrate baseline physiological values

• Establish parent + child safety thresholds

• Begin low-stakes observation logging


Tasks:

1. Install the EchoAuth App / Dashboard

• Run on laptop or phone

• Ensure logging works (safety.log + chamber.log)

2. Wearable Setup on Child & (optional) Parent

• HR, EDA, temp sensors

• Connect via BLE module

3. Baseline Calibration Session (30–60 minutes)

Capture 3 emotional states:

• Green: calm playing, TV, reading

• Amber: mild excitement, transitions

• Red: mild frustration (NO forcing—use past logs or natural events)

4. Store baselines in DB

• baseline_hr

• baseline_eda

• threshold multipliers

5. Confirm HER Kernel Works with Real Signals

• classification changes (G → A → G)

• logs appear reliably

• no drops during 10-minute windows


Deliverables:

• Child-specific thresholds

• Baseline sensor stability logs

• Dashboard shows real emotional-state changes


──────────────────────────────────


🔥 PHASE 2 — DAILY LIVING OBSERVATION (Days 6–14)


Objectives:

• Validate HER kernel behavior in natural environment

• Test stress detection during real daily transitions

• Begin handoff simulations


Daily Tasks (10–20 min each):

1. Wearable Session:

Child wears sensor for 10–30 minutes during:

• morning routine

• playing

• snack transitions

• TV time

2. Record Natural Emotional Shifts

Parent marks (taps in UI or notebook):

• “Child elevated”

• “Child calm again”

3. Simulated Caregiver Handoff (1–2 per day)

Parent → you

You → parent

(Does not need actual caregiver yet)

4. Check Chamber Behavior

• Approved handoff → allowed

• Random unapproved attempt (simulated) → blocked

• Red-state attempt → auto-denied

5. Collect logs

• emotional shifts

• sensor connection events

• handoff requests

• block events


Deliverables:

• Real-world HER kernel map

• First Red-State block trace

• Documentation of system consistency

• Initial parent testimonial


──────────────────────────────────


🔥 PHASE 3 — STRESS EVENT TRACKING (Days 15–21)


Aim: Capture how EchoAuth responds when the child experiences:

• upset

• frustration

• sensory overload

• transitions they dislike


NO forced events — use only naturally occurring ones.


Tasks:

1. Wearable during predictable triggers:

• bedtime

• taking a device away

• saying “no”

• leaving the house

• stopping a preferred activity

2. HER Kernel Trace Collection:

Look for:

• delayed detection

• false Amber flags

• correct Red-state predictions

• recovery time

3. Handoff Attempt During Amber/Red

• confirm Chamber blocks it

• verify override requires human confirmation

• verify logs capture everything


Deliverables:

• 2–4 high-quality Red-state traces

• HER kernel accuracy estimate

• Chamber correct-block evidence

• Stress → recovery timeline


These are the gold-standard pieces for SBIR + Angels.


──────────────────────────────────


🔥 PHASE 4 — REAL-WORLD VALIDATION (Days 22–30)


Objectives:

• Evaluate system stability

• Collect user experience feedback

• Prove the safety-value of EchoAuth

• Prepare for formal clinical LOI


Weekly Tasks:

1. Multi-day wearable use (2–4 days)

2. Handoff with real caregiver (optional)

3. Parent logs a short note daily: “What EchoAuth helped with today.”

4. Extract logs and generate charts:

• emotion timeline

• number of Amber/Red events

• number of blocked handoffs

• override usage

5. Evaluate system uptime:

• Sensor connectivity consistency

• BLE drops

• API stability


Deliverables:

• Alpha Pilot Report (PDF)

Includes:

• emotional-state graph

• block/allow events

• parent testimony

• safety incidents prevented

• HER accuracy stats

• Chamber performance metrics

• Founder Testimonial + Child Case Study

• Material for clinical LOI (“We have validated the system at home”).
