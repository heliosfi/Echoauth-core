◦ Lifecycle: Auto-rotation every 90 days; audited access

• Key Type 2: Transport Encryption Keys (TLS 1.3 Certificates)

◦ Generated: Let’s Encrypt or Agency CA

◦ Used For: All inbound/outbound traffic (F1-F10 Flows)

◦ Lifecycle: Renewal every 90 days; revocation on compromise

• Key Type 3: Hashing Keys (SHA-256/512)

◦ Generated: Internal to EchoAuth

◦ Used For: Behavioral Pattern Hashes, RMS Profiles

◦ Lifecycle: Static but rekeyed on major updates

• Key Type 4: Session Keys (Ephemeral)

◦ Generated: Per Session (P1-P8)

◦ Used For: Temporary encryption of in-memory data

◦ Lifecycle: Destroyed at session end

Narrative Description for SSP Section: NI-AI employs a layered key management system: AES-256 for data at rest in stores, TLS 1.3 for transit across all flows, SHA-256/512 for hashing sensitive patterns, and ephemeral keys for sessions. Keys are generated and managed via FIPS-compliant KMS, with automatic rotation, access auditing, and revocation procedures to ensure confidentiality and integrity.

E. Logging Workflow (Text Version)

Purpose: Details the audit logging process, supporting AU family controls.

Text-Based Representation (Step-by-Step Flow):

1 Event Trigger: Any security-relevant action (e.g., authentication attempt, drift detection, access request) in P1-P7.

2 Log Capture: P8 Layer collects timestamp, user/session ID (pseudonymous), event type, outcome, risk score.

3 Enrichment: Adds context (e.g., IP, device type if allowed; RMS details).

4 Encryption: AES-256 applied before write.

5 Storage: Written to D3 Immutable Audit Log Store (write-once, read-many).

6 Monitoring: Real-time scans for anomalies; alerts to Admin Console.

7 Review/Export: Weekly manual reviews; exports to agency via secure API (e.g., SIEM integration).

8 Retention: 6-12 months per policy; auto-purge of old logs.

Narrative Description for SSP Section: The NI-AI Logging Workflow begins with event capture across all processes, enriching logs with minimal metadata, encrypting them, and storing in an immutable repository. Continuous monitoring detects issues, with alerts and periodic reviews ensuring accountability. Logs support incident response and can be exported to government SIEM systems without compromising privacy.
