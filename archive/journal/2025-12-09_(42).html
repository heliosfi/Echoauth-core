📘 PAGE 36 — ASD SENSORY SAFETY DESIGN


SECTION VI — Compliance, Safety & Governance


This page defines EchoAuth’s mandatory sensory-safety rules when used by autistic and neurodivergent users, especially individuals with sensory sensitivities, processing delays, or communication difficulties.


These principles must be followed by every engineer, designer, and integration partner.


────────────────────────────────────────


⭐ 36.1 — Guiding Principles


EchoAuth’s interface and runtime behavior must:


1️⃣ Reduce sensory load


2️⃣ Prevent overstimulation


3️⃣ Maintain predictability


4️⃣ Provide stable, low-stress communication pathways


This is non-negotiable.

Everything — from color, motion, timing, layout, audio cues — must support ASD comfort and stability.


────────────────────────────────────────


⭐ 36.2 — Visual Design Requirements


✔ Low-contrast color palette


Use soft blues, greens, muted neutrals — avoid high-saturation colors.


✔ Consistent grid layout


Buttons, icons, and communication symbols must remain in fixed positions.

No dynamic UI that moves or reflows unpredictably.


✔ Zero flicker


UI components must never flash, pulse, or animate rapidly.


✔ No surprise animations


Transitions must be:

• smooth

• slow

• explicitly predictable


If animation is required, it must be < 2 FPS.


✔ Symbol clarity


All icons representing needs/emotions must be:

• concrete

• visually distinct

• culturally neutral

• not abstract or metaphorical unless clinically validated


────────────────────────────────────────


⭐ 36.3 — Auditory Design Requirements


✔ No voice synthesis unless stabilized


EchoAuth does not generate emotional tone unless approved templates are used.


✔ Soft, low-frequency cues


Notification sounds (if any) must be:

• gentle

• short

• no high-pitch frequencies


✔ No surprise noises


System never emits unexpected audio.


✔ Caregiver can disable all sound output


Silent mode is fully supported.


────────────────────────────────────────


⭐ 36.4 — Timing & Interaction Predictability


✔ Fixed response timing


EchoAuth must respond within a predictable window (e.g., 300–700 ms).

This consistency reduces anxiety and cognitive strain.


✔ No long pauses


If processing takes >1.5s, system displays a neutral, stable “thinking” symbol.


✔ Step-by-step dialogs


The interaction flow must follow:

1. Input

2. Clarification

3. Confirmation

4. Action


Never skip steps.


────────────────────────────────────────


⭐ 36.5 — Environmental Sensory Constraints


EchoAuth must be usable in:

• classrooms

• therapy centers

• homes

• high-sensory environments


Therefore:


✔ Brightness auto-adaptation OFF


System must not auto-adjust brightness suddenly.


✔ Haptic feedback optional and minimal


Only gentle taps — never vibration pulses.


✔ Kiosk/Focus mode supported


Prevents accidental navigation to distracting screens.


────────────────────────────────────────


⭐ 36.6 — Communication Safety Rules


EchoAuth interprets communication in a structured, clinically aligned way:


✔ No open-ended, emotionally charged responses


✔ No anthropomorphism (“I feel…” “I think…”)


✔ No unpredictable generative content


✔ No sarcasm, metaphor, or implicit phrasing


Communication templates are:

• literal

• direct

• safety-oriented

• consistent

• predictable


────────────────────────────────────────


⭐ 36.7 — Behavioral Safety Triggers


EchoAuth automatically adjusts UI if:

• user repeatedly taps the same symbol (stress pattern)

• user shows slowing input

• user switches rapidly between options

• inactivity exceeds tolerance window


Possible automatic adjustments:

• enlarge key buttons

• reduce symbols

• simplify layout

• offer “Need Help?” prompt

• notify caregiver in clinical mode


────────────────────────────────────────
