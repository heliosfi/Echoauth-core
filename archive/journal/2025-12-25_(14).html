PAGE 21 — DAILY OPERATING CHECKLIST (CANONICAL v1.0)


Pre-Market • In-Session • Post-Session • No Drift


This page turns SniperBot from a system into a daily operating ritual.

It is the final guardrail that keeps discipline intact across time, emotion, and market conditions.


⸻


1) Pre-Market Checklist (Must Be Completed Before Open)


A. System Readiness

• ☐ Data feed healthy (no gaps / delays)

• ☐ Logging system active

• ☐ Kill switch tested

• ☐ Account equity updated

• ☐ Daily loss cap set


If any fail → DO NOT ARM


⸻


B. Market Context Classification

• ☐ Identify primary regime (Trend / Expansion / Range / Transition)

• ☐ Identify volatility bucket (~3.5% or ≥6%)

• ☐ Identify key levels (prior H/L, VWAP, major zones)

• ☐ Identify session bias (if any)


If regime ≠ Trend or Expansion → Manual only or No Trade


⸻


C. News & Event Scan

• ☐ High-impact events marked (CPI, FOMC, earnings)

• ☐ News blackout windows defined

• ☐ Options IV regime checked (if applicable)


If news = Red → NO TRADE


⸻


D. Configuration Lock

• ☐ Tickers whitelisted

• ☐ Entry types whitelisted

• ☐ Risk % locked (manual / auto)

• ☐ Auto-Mode default = OFF


Once market opens → no config changes


⸻


2) In-Session Checklist (While Market Is Live)


A. State Awareness

• ☐ Confirm current state (PAUSE / READY / ARMED)

• ☐ Confirm no LOCKOUT flags

• ☐ Confirm regime unchanged


If regime shifts → PAUSE


⸻


B. Trade Card Discipline

• ☐ Trade Card exists

• ☐ Stop & TP defined

• ☐ Card not expired

• ☐ Entry trigger exact


No card → no execution


⸻


C. Arming Rules

• ☐ Manual arming only when fully present

• ☐ Auto-Mode only if pre-market conditions still valid

• ☐ Auto-Mode max trades enforced


When unsure → stay PAUSE


⸻


D. In-Trade Conduct

• ☐ No stop widening

• ☐ No averaging down

• ☐ TP ladder respected

• ☐ Time stop enforced


Emotion detected → hands off


⸻


3) Post-Session Checklist (Mandatory)


A. Trade Review


For each trade:

• ☐ Trade Card vs execution compared

• ☐ Entry quality assessed

• ☐ Stop respected?

• ☐ TP plan respected?


⸻


B. Outcome Logging

• ☐ P&L logged

• ☐ State transitions logged

• ☐ Gate failures (if any) noted


No review → no improvement


⸻


C. System Health

• ☐ Any rule violations?

• ☐ Any manual overrides?

• ☐ Any near-misses?


If yes → adjust rules, not behavior


⸻


4) Weekly / Periodic Review (Optional but Recommended)

• ☐ Are certain setups outperforming?

• ☐ Are lockouts occurring for the right reasons?

• ☐ Is risk % appropriate for volatility?

• ☐ Is Auto-Mode being used rarely and correctly?


If Auto-Mode feels frequent → dial it back


⸻


5) Page 21 Lock


Consistency is not achieved by better trades.

It is achieved by better routines.


This checklist ensures SniperBot stays aligned with its core purpose:

disciplined execution under governance.
