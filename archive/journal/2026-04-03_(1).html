SNIPER BOT — SYSTEM CORE


1. Master Objective


The bot does not predict.


It does 4 things:

1. Classify market regime

2. Detect alignment / non-alignment

3. Allow or deny entry

4. Manage invalidation after entry


So the system is:


Regime-first, trigger-second, execution-third


Not the other way around.


⸻


2. SYSTEM LAYERS


Layer A — Macro Regime Engine


Purpose: determine whether the market is in:

• Expansion / risk-on

• Contraction / risk-off

• Transition / mixed

• Event shock / headline override


Inputs


Your core 5-chart dashboard:

• NQ / MNQ

• SMH

• NVDA

• VIX

• TLT


Optional later:

• DXY

• Oil

• Gold

• 10Y yield


Regime Logic


The engine scores each asset, then outputs one regime.


Example regime rules


if NQ_above_vwap and SMH_strength and NVDA_strength and VIX_falling and TLT_flat_or_down:

    regime = "EXPANSION"


elif NQ_below_vwap and SMH_weak and NVDA_weak and VIX_rising and TLT_rising:

    regime = "CONTRACTION"


elif mixed_signals:

    regime = "TRANSITION"


elif headline_shock and VIX_expanding_fast:

    regime = "EVENT_SHOCK"



⸻


Layer B — Structure Engine


Purpose: determine whether price structure supports entry.


This layer answers:

• trend continuation?

• breakdown?

• dead cat bounce?

• reclaim?

• failed breakout?

• range trap?


Inputs


Per chart:

• VWAP

• 9 EMA / 20 EMA

• MACD

• RSI

• volume

• candle body ratio

• prior high / low

• opening range

• key support / resistance


Output states

• BULL_CONTINUATION

• BEAR_CONTINUATION

• RANGE

• FAILED_RECLAIM

• BREAKOUT_PENDING

• BREAKDOWN_PENDING

• MEAN_REVERSION_ONLY


⸻


Layer C — Confluence Engine


Purpose: decide whether the setup is valid enough to act on.


This is the actual gate.


A trade is not taken because one thing looks good.

A trade is taken only when enough aligned conditions are true.


Confluence categories

• Full confluence

• Partial confluence

• Conflict / deny

• Isolated runner

• Headline-only move


Example


if regime == "EXPANSION" and structure == "BULL_CONTINUATION" and volume_confirmed:

    confluence = "FULL_LONG"


elif regime == "CONTRACTION" and structure == "BEAR_CONTINUATION" and vix_rising:

    confluence = "FULL_SHORT"


elif strong_asset_move and broad_market_not_aligned:

    confluence = "ISOLATED_RUNNER"


else:

    confluence = "DENY"



⸻


Layer D — Trigger Engine


Purpose: define the exact moment of execution.


This prevents early entries.


Trigger examples


For longs:

• reclaim VWAP and hold

• break prior candle high

• volume above rolling average

• MACD cross up with price confirmation

• RSI re-expanding above threshold

• successful retest of breakout level


For puts:

• lose VWAP and fail reclaim

• break opening range low

• lower high confirmed

• MACD cross down with price rejection

• VIX up while QQQ weakens

• retest failure at resistance


⸻


Layer E — Invalidation Engine


Purpose: tell the system when the setup is wrong.


This is one of the most important pieces.


The system should never enter without a defined invalidation state.


Invalidation types

• Price invalidation

• Time invalidation

• Volume invalidation

• Regime invalidation

• Correlation invalidation


Example


if long_entry:

    invalidate_if = [

        price_loses_retest_level,

        candle_closes_back_below_vwap,

        volume_dries_up_after_breakout,

        NQ_rolls_over and VIX_spikes

    ]

3. PRIMARY MARKET STATES


These are the states the whole bot should understand.


A. Expansion


Characteristics:

• NQ above VWAP

• SMH leading

• NVDA participating

• VIX fading

• TLT not catching strong bid


Bot behavior:

• favor long continuation

• allow breakout entries

• allow dip buys on reclaim


⸻


B. Contraction


Characteristics:

• NQ below VWAP

• SMH weak

• NVDA weak or failing

• VIX rising

• TLT bid


Bot behavior:

• favor puts

• reject weak longs

• allow dead-cat shorting

• prioritize failed reclaim setups


⸻


C. Transition


Characteristics:

• mixed leadership

• QQQ/NQ choppy

• VIX elevated but not exploding

• TLT inconsistent

• semis divergent


Bot behavior:

• reduce aggression

• trade only A+ setups

• no chase

• allow isolated trades only with strong structure


⸻


D. Event Shock


Characteristics:

• geopolitical news

• CPI/PCE/Fed prints

• gap risk

• oil / yields / VIX all repricing fast


Bot behavior:

• no immediate entry

• wait for post-shock structure

• treat first move as information, not signal

• shift from trend mode to reaction mode

4. CORE DECISION TREE


This is the actual machine logic.


def trade_decision(market_data, asset_data, event_data):

    regime = classify_regime(market_data, event_data)

    structure = classify_structure(asset_data)

    confluence = score_confluence(regime, structure, market_data, asset_data)


    if confluence == "DENY":

        return "NO TRADE"


    trigger = check_trigger(asset_data, market_data)


    if not trigger:

        return "WAIT"


    invalidation = define_invalidation(asset_data, market_data)

    target = define_target(asset_data, regime, structure)


    return {

        "decision": "ENTER",

        "regime": regime,

        "structure": structure,

        "confluence": confluence,

        "invalidation": invalidation,

        "target": target

    }

5. CONFLUENCE SCORE MODEL


Use a weighted system.


Suggested weights


Macro / regime

• NQ location vs VWAP = 15

• SMH trend = 10

• NVDA trend = 10

• VIX direction = 15

• TLT direction = 10


Structure

• price vs VWAP = 10

• EMA alignment = 5

• MACD confirmation = 5

• RSI trend = 5

• candle body / breakout quality = 5

• volume confirmation = 10


Total = 100


Interpretation

• 80–100 = A+ setup

• 65–79 = tradable

• 50–64 = partial / isolated only

• below 50 = no trade

6. PLAYBOOK TYPES


The system should recognize a few specific plays.


Play 1 — Bull Reclaim


Conditions:

• regime not bearish

• price reclaims VWAP

• retest holds

• volume confirms

• VIX not expanding


Action:

• long entry on hold/re-expansion


Invalidation:

• lose reclaim level


⸻


Play 2 — Failed Reclaim Put


Conditions:

• regime contraction or transition bearish

• price retakes level briefly then fails

• lower high forms

• VIX stable to rising


Action:

• enter put on breakdown


Invalidation:

• reclaim back above failure zone


⸻


Play 3 — Opening Range Breakdown


Conditions:

• weak premarket / weak internals

• lose OR low

• no reclaim

• semis weak


Action:

• short / puts


Invalidation:

• OR low reclaimed with volume


⸻


Play 4 — Isolated Runner


Conditions:

• asset-specific catalyst

• asset stronger than market

• market mixed

• volume extreme


Action:

• tradable, but marked as non-additive

• tighter invalidation

• no broad assumption


⸻


Play 5 — Dead Cat Bounce


Conditions:

• prior heavy selloff

• bounce with weak volume

• resistance failure

• VIX still elevated


Action:

• short only after rejection confirms


Invalidation:

• bounce converts into hold/reclaim

7. TIME-OF-DAY FILTERS


This matters a lot.


Phase 1 — Open Discovery


9:30–10:00

Purpose:

• no emotional chase

• detect whether gap holds or fades

• first move is not always the real move


Bot rule:

• entries only if trigger plus confirmation

• otherwise wait


Phase 2 — Morning Trend Window


10:00–11:30

Best for:

• continuation

• failed reclaim

• opening range break


Phase 3 — Midday Noise


11:30–1:30

Bot rule:

• reduce trust

• no new entries unless unusually high confluence


Phase 4 — Power Hour / Trap Hour


2:00–4:00

Bot rule:

• accept continuation only if structure remains clean

• watch for squeeze / rug-pull behavior

8. EVENT FILTER


The bot should know when not to over-trust normal signals.


High-risk event conditions

• CPI / PPI / PCE

• FOMC / Powell

• Jobs / unemployment

• major geopolitical escalation

• earnings for core leaders like NVDA


Event filter logic


if scheduled_macro_event_within_15min or unscheduled_headline_shock:

    mode = "EVENT_FILTER_ON"


When on:

• no entry before release

• wait for first reaction + second confirmation

• prioritize structure after reaction, not before

9. EXECUTION MODES


The system should not always behave the same.


Mode 1 — Aggressive


Used only when:

• regime clear

• confluence very high

• structure clean

• event risk low


Mode 2 — Standard


Used most days.


Mode 3 — Defensive


Used when:

• transition regime

• event risk elevated

• VIX high

• correlations unstable


Mode 4 — Observation Only


Used when:

• market completely dislocated

• no alignment

• headline whipping

• unclear internals


This mode is important.

The bot needs permission to say:


No trade. No edge.

11. HARD RULES


These prevent system drift.


Rule 1


No entry without regime classification.


Rule 2


No entry without invalidation.


Rule 3


One strong signal is never enough.


Rule 4


Headline moves do not override structure unless event mode is on.


Rule 5


If macro and asset conflict, classify as isolated, not aligned.


Rule 6


The bot may deny good-looking trades if confluence is incomplete.


Rule 7


No adding into partial confluence unless explicitly allowed.


Rule 8


If VIX and QQQ disagree violently, reduce trust in all continuation signals.
