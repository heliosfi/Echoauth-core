3️⃣ Structured Logging Template (Safety-Grade Logging)


A. Logging Config — logging_config.py


import logging

from logging.handlers import RotatingFileHandler

import json



class JsonFormatter(logging.Formatter):

    def format(self, record):

        payload = {

            "level": record.levelname,

            "logger": record.name,

            "message": record.getMessage(),

            "time": self.formatTime(record, self.datefmt),

        }

        if hasattr(record, "extra_data"):

            payload.update(record.extra_data)

        return json.dumps(payload)



def setup_logging():

    logger_names = ["governance", "chamber", "safety"]


    for name in logger_names:

        logger = logging.getLogger(name)

        logger.setLevel(logging.INFO)


        handler = RotatingFileHandler(

            f"{name}.log",

            maxBytes=5 * 1024 * 1024,

            backupCount=3,

        )

        handler.setFormatter(JsonFormatter())

        logger.addHandler(handler)


    # root logger

    root = logging.getLogger()

    root.setLevel(logging.INFO)


Call setup_logging() once at app startup (e.g., in main.py or FastAPI startup event).

from enum import Enum

class EmotionalState(str, Enum):

B. Interaction Flow (Text Diagram)


Sensors → HER → Spine → Chamber → UI

1. Sensors

• BLE, HR, EDA, motion

• Data goes into sensor_window object.

2. HER Kernel

• her_output = her_kernel.classify(sensor_window)

• her_output.state ∈ {GREEN, AMBER, RED}

• her_output.confidence ∈ [0, 1]

3. Governance Spine

• Build SystemContext from:

• child_id, caregiver_id, location, time

• emotional_confidence = her_output.confidence

• gov_decision = apply_governance_spine(ctx)

4. Chamber

• Interpret gov_decision.final_decision

• ALLOW/BLOCK/REQUIRE_HUMAN

• Emit events → chamber.log

5. UI

• Show current state (G/A/R)

• Show “handoff allowed/blocked”

• Show reason: “Blocked by S5 – Red-state lock”

C. Hybrid HER Integration (Optional)


You can wrap both rule-based + ML into one class:


class HybridHERKernel(BaseHERKernel):

    def __init__(self, rule_kernel: BaseHERKernel, ml_kernel: BaseHERKernel, threshold: float = 0.6):

        self.rule_kernel = rule_kernel

        self.ml_kernel = ml_kernel

        self.threshold = threshold


    def classify(self, sensor_window) -> HerOutput:

        ml_out = self.ml_kernel.classify(sensor_window)

        if ml_out.confidence >= self.threshold:

            return ml_out

        return self.rule_kernel.classify(sensor_window)


Governance spine doesn’t care which HER produced the state —

it just enforces S1–S23 on whatever state comes in.
