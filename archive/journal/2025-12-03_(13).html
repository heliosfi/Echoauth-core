1. EchoAuth Token Design (JWT claims)


Algorithm: RS256 (asymmetric)

Signed by: EchoAuth private key (server only)

Verified by: NI-AI using EchoAuth public key


Recommended claims for v1.0:


{

  "iss": "echoauth-core",      // who issued it

  "aud": "ni-ai",              // who it's for

  "sub": "child-1234",         // child ID in your system

  "session_id": "uuid-here",   // per-session unique ID

  "caregiver_id": "adult-567", // optional, if a parent is linked

  "chamber_state": "safe",     // REQUIRED: "safe" | "locked" | "review"

  "mode": "talk",              // "talk" | "listen_only" | "story" etc.

  "iat": 1735850000,           // issued-at

  "exp": 1735853600            // expiry (e.g., +1 hour)

}


Hard rule in S6:

If chamber_state != "safe" → block.

2. EchoAuth Side — Issuing Real Tokens


On the EchoAuth backend (NOT in NI-AI), you’d do something like:


# echoauth_issuer.py  (EchoAuth server-side)

import jwt

import uuid

import time


with open("echoauth_private.pem", "rb") as f:

    PRIVATE_KEY = f.read()


def issue_echoauth_token(child_id: str, caregiver_id: str | None = None) -> str:

    now = int(time.time())

    payload = {

        "iss": "echoauth-core",

        "aud": "ni-ai",

        "sub": child_id,

        "session_id": str(uuid.uuid4()),

        "caregiver_id": caregiver_id,

        "chamber_state": "safe",

        "mode": "talk",

        "iat": now,

        "exp": now + 60 * 60,  # 1 hour

    }

    headers = {"kid": "echoauth-key-v1"}  # optional key ID

    return jwt.encode(payload, PRIVATE_KEY, algorithm="RS256", headers=headers)


EchoAuth gives that JWT to the client, and the client calls NI-AI with:


X-EchoAuth-Token: <that-long-JWT-string>

3. NI-AI Side — Real S6 Verification (drop-in replacement)


Here’s a clean S6 that you can drop into your current v1.4 file, plus the new config line.


3.1. Add public key config


Near the top of ni_ai_kernel.py, under CONFIGURATION:


ECHOAUTH_PUBLIC_KEY = os.getenv("ECHOAUTH_PUBLIC_KEY", "").strip()


You can later set ECHOAUTH_PUBLIC_KEY in env to the PEM string, or inline the PEM in code if you prefer.


3.2. Install PyJWT (if you haven’t)


pip install PyJWT


3.3. Replace your existing S6 with this version


import jwt  # add near the top with other imports


def S6_child_safety_guard(state: State) -> Tuple[bool, State, Optional[str]]:

    """

    EchoAuth gate.


    DEV_MODE:

        - If echoauth_ok=True in state → allow (for local testing)

        - If echoauth_token present → try real verification (optional dev test)

    PROD (DEV_MODE=False):

        - echoauth_token is REQUIRED and must pass RS256 verification

    """

    token = state.get("echoauth_token")


    # --- DEV SHORTCUT: allow simple flag ---

    if DEV_MODE and state.get("echoauth_ok", False) and not token:

        state["chamber_verified"] = True

        state["chamber_payload"] = {"dev": True}

        return True, state, None


    # From here on, we expect a real token (even in dev if provided)

    if not token:

        return False, state, "Missing EchoAuth token"


    if not ECHOAUTH_PUBLIC_KEY:

        # Config error: no key to verify with

        return False, state, "EchoAuth public key not configured on NI-AI side"


    try:

        payload = jwt.decode(

            token,

            ECHOAUTH_PUBLIC_KEY,

            algorithms=["RS256"],

            audience="ni-ai",

            issuer="echoauth-core",

            options={"require": ["iss", "aud", "exp", "sub"]},

        )

    except jwt.ExpiredSignatureError:

        return False, state, "EchoAuth token expired"

    except jwt.InvalidTokenError as e:

        return False, state, f"Invalid EchoAuth token: {str(e)}"


    # Hard safety check: chamber must be in safe mode

    if payload.get("chamber_state") != "safe":

        return False, state, f"Chamber not in safe state (got {payload.get('chamber_state')!r})"


    # All good — store payload for logging / parental audit

    state["chamber_verified"] = True

    state["chamber_payload"] = payload

    return True, state, None


This stays perfectly aligned with your S1–S9 flow:

• S6 still blocks by flipping state["blocked"] = True via _run_chain if it returns (False, ...).

• S9 will then wrap that block in a gentle message.

• S7 logs the whole event, including chamber_verified=False and the last violation reason.

4. FastAPI Gateway — Passing the Real Token


Your existing echoauth_api.py already pulls the header:


@app.post("/ni-ai/chat", response_model=ChatResponse, tags=["chat"])

async def ni_ai_chat(

    request: ChatRequest,

    x_echoauth_token: Optional[str] = Header(default=None, alias="X-EchoAuth-Token"),

):

    ...

    session_state: Dict[str, Any] = {}


    if x_echoauth_token:

        session_state["echoauth_token"] = x_echoauth_token


    if DEV_MODE and request.echoauth_ok is True:

        session_state["echoauth_ok"] = True


    result = ni_ai(

        user_message=request.message,

        session_state=session_state,

    )


With the new S6:

• In DEV_MODE:

• You can test with just echoauth_ok: true (no token).

• Or begin testing real tokens by sending X-EchoAuth-Token and seeing chamber_payload logged in S7.

• In PROD:

• Set DEV_MODE = False

• Remove echoauth_ok from the request model / gateway, and make X-EchoAuth-Token mandatory.
