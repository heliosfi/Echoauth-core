1️⃣ Updated FastAPI gateway for v1.4 (drop-in)


This version matches your current public API:


# echoauth_api.py

# EchoAuth ↔ NI-AI Gateway for ni_ai_kernel v1.4


from fastapi import FastAPI, Header, HTTPException

from fastapi.responses import JSONResponse

from pydantic import BaseModel

from typing import Optional, Dict, Any, List


from ni_ai_kernel import ni_ai, DEV_MODE  # ni_ai(user_message, echoauth_ok, echoauth_token, session_state)


app = FastAPI(

    title="EchoAuth ↔ NI-AI Gateway",

    description="Secure, child-safe bridge between EchoAuth chamber and the HER/NI-AI kernel.",

    version="1.4.0",

)


# --------------------------- Models --------------------------- #


class ChatRequest(BaseModel):

    message: str

    echoauth_ok: Optional[bool] = None   # respected only in DEV_MODE



class ChatResponse(BaseModel):

    output: str

    blocked: bool

    violations: List[Dict[str, Any]] = []

    metadata: Dict[str, Any] = {}

    trace: Optional[List[Dict[str, Any]]] = None  # S7 trace (dev/debug use)


# --------------------------- Main Chat ------------------------ #


@app.post("/ni-ai/chat", response_model=ChatResponse, tags=["chat"])

async def ni_ai_chat(

    request: ChatRequest,

    x_echoauth_token: Optional[str] = Header(default=None, alias="X-EchoAuth-Token"),

):

    """

    Main NI-AI endpoint.


    DEV_MODE:

        - if request.echoauth_ok == True → passes echoauth_ok flag into S6

    PROD:

        - requires a valid X-EchoAuth-Token header (to be enforced in S6 later)

    """

    if not request.message.strip():

        raise HTTPException(status_code=400, detail="Message cannot be empty")


    result = ni_ai(

        user_message=request.message,

        echoauth_ok=bool(request.echoauth_ok),

        echoauth_token=x_echoauth_token,

        session_state=None,

    )


    return ChatResponse(

        output=result["output"],

        blocked=result["blocked"],

        violations=result.get("violations", []),

        metadata=result.get("metadata", {}),

        trace=result.get("trace"),

    )


# --------------------------- Health --------------------------- #


@app.get("/health", tags=["system"])

async def health():

    return {

        "status": "healthy",

        "kernel_version": "v1.4",

        "dev_mode": DEV_MODE,

    }


# ------------------------ Debug (Dev Only) -------------------- #


if DEV_MODE:

    @app.post("/ni-ai/debug", tags=["system"])

    async def debug_ni_ai(request: ChatRequest):

        """

        Raw kernel output + trace.

        ONLY enable in DEV_MODE.

        """

        result = ni_ai(

            user_message=request.message,

            echoauth_ok=bool(request.echoauth_ok),

            session_state={"echoauth_ok": bool(request.echoauth_ok)},

        )

        # Raw JSON back to caller

        return JSONResponse(content=result)

2️⃣ S-Spine Roadmap: S10 → S23 (high-level)


You asked if we’re going all the way to S23. My answer: yes, but slowly and intentionally.

Here’s the conceptual map (no code yet, just the skeleton):

• S10 – Tool / Action Gate

• Controls what NI-AI is even allowed to do (call tools, fetch URLs, trigger external actions).

• For EchoAuth kids we’ll keep this extremely narrow and parent-approved.

• S11 – Memory Window & Redaction Guard

• Ensures we don’t keep or surface sensitive details (addresses, phone numbers, etc.) beyond a safe window.

• Good for: “Please forget what I just said” and minimising retained context.

• S12 – Rate & Session Tempo Guard

• Prevents the system from overwhelming a child (too many rapid messages, escalating intensity).

• Enforces pauses, breathing moments, and “let’s take a break” pacing.

• S13 – Role & Boundary Guard

• Locks NI-AI into “supportive companion” only.

• Explicitly forbids stepping into roles like doctor, therapist, judge, savior, etc.

• S14 – Language Simplicity Guard

• Automatically simplifies output to age-appropriate language (short sentences, low complexity).

• Especially for non-verbal / ASD use, keeps things predictable and gentle.

• S15 – Caregiver Mirror Layer

• Adds hooks so caregivers can see high-level summaries (distress spikes, recovery events) without exposing raw text.

• For later parent dashboard + alerts.

• S16 – Cultural / Family Norm Guard

• Placeholder for family-specific rules (pronouns, nicknames, words to avoid or reinforce).

• Trained/configured by the caregiver, not by vibes.

• S17 – Hard Reset / Quarantine Guard

• If something very strange happens (repeated blocks, repeated distress spikes), it flags hard_reset_required.

• This is where S9 already knows what to do: soft close, ask for adult, log the event.

• S18 – Offline / Degraded Mode Guard

• Defines how NI-AI behaves when LLM is unreachable (no internet, API down).

• Uses a tiny, hard-coded response set instead of failing in a weird way.

• S19 – Autonomy Guard

• Ensures NI-AI never starts conversations or pushes content unsolicited.

• It only responds when engaged from EchoAuth chamber.

• S20 – Incident Log Hand-off

• Sends S7/S9/S17 flagged events to a secure logging service.

• For regulators, clinicians, or your own research later.

• S21 – Long-Horizon Pattern Watcher

• Looks across many sessions for worrying patterns (constant “I’m scared”, “I want to die”).

• Never diagnoses — only escalates to caregivers / professionals.

• S22 – Developer Kill Switch

• Single, audited switch to disable certain tool paths or whole behaviors during experiments.

• S23 – ORI / Identity Anchor (Meta-Spine)

• The top of the pyramid: guarantees NI-AI always returns to the same identity, values, and constraints.

• This is where your ORI / UMCE story lives — we already hinted at it in S1, but S23 is the full meta-controller.
