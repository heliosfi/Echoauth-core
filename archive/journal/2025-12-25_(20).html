PAGE 16 — SNIPERBOT STATE MACHINE (CANONICAL v1.0)


This page defines SniperBot’s finite-state machine (FSM): the exact operating states, allowed transitions, and shutdown rules. This is the “tiny car” that can drive inside the bigger vehicle—logic first, interface later.


⸻


1) Core States


S0 — PAUSE (Default / No-Trade)


Meaning: SniperBot is running, but not allowed to execute.

Allowed actions: Observe, classify, detect setups, build trade cards, alert.


Entry conditions:

• System boot

• Auto/Manual arming expired

• Any uncertainty or violation

• Post-trade cooldown


⸻


S1 — READY (Cleared to Monitor)


Meaning: Preconditions are satisfied to present opportunities.

Allowed actions: Detect whitelist setups + generate Trade Cards.


Entry conditions:

• AllPreTradeGates == PASS

• News status != Red

• Regime confidence >= threshold


⸻


S2 — ARMED_MANUAL (Human-Authorized Alerts)


Meaning: Human approves the session for signals + prompts, not execution.

Allowed actions: Send “Trade Card” + “Trigger hit” alerts.


Entry conditions:

• Human selects ARM_MANUAL

• Session within allowed hours


⸻


S3 — ARMED_AUTO (Permissioned Execution Window)


Meaning: Auto execution is permitted only under Auto-Mode rules.

Allowed actions: Execute one pre-approved trade card when trigger hits.


Hard constraints:

• Max trades/day enforced

• Risk cap enforced

• Auto expires on any gate failure


Entry conditions:

• Human selects ARM_AUTO

• AutoModeGates == PASS


⸻


S4 — IN_TRADE (Active Position Management)


Meaning: A position exists; bot manages stop/TP and logs events.

Allowed actions: Update trailing stops (if allowed), execute TP ladder, time stop.


Entry conditions:

• Order filled (or accepted if bracket immediately attached)


⸻


S5 — LOCKOUT (Safety Halt)


Meaning: Trading is disabled until reset conditions are met.

Allowed actions: Log reason, alert operator, require explicit reset.


Entry conditions (any one triggers lockout):

• Daily max loss hit

• Logging failure

• Feed/data integrity failure

• Two consecutive gate invalidations during ARMED_AUTO

• Manual kill switch pressed


⸻


2) Allowed Transitions (No Drift)


Boot / Recovery

• START → PAUSE

• PAUSE → READY (when all gates pass)


Arming

• READY → ARMED_MANUAL (human arms manual)

• READY → ARMED_AUTO (human arms auto + auto gates pass)

• ARMED_MANUAL → READY (human disarms)

• ARMED_AUTO → READY (auto expires, trade executed, or gates fail)


Trading

• ARMED_MANUAL → IN_TRADE (manual execution happens outside bot; bot begins management once position detected—optional integration)

• ARMED_AUTO → IN_TRADE (bot executes and confirms fill)


Exit / Cooldown

• IN_TRADE → PAUSE (position closed + cooldown)

• IN_TRADE → READY (only if explicitly allowed; default is PAUSE)


Safety

• ANY STATE → LOCKOUT (safety trigger)

• LOCKOUT → PAUSE (only after explicit human reset + gates re-pass)


⸻


3) Global Invariants (Always True)

1. Default state is PAUSE

2. No Trade Card → No Trade

3. RiskGate has priority over Signal

4. Stop/TP must exist before or at entry

5. If uncertainty rises → downgrade to PAUSE

6. LOCKOUT overrides everything

7. Human kill switch overrides everything


⸻


4) Shutdown & Downgrade Rules


Downgrade to PAUSE (soft stop)


Triggers:

• Regime confidence drops below threshold

• Liquidity/spreads worsen beyond limits

• Market transitions to disallowed regime

• News status turns Yellow/Red

• Outside session hours


Behavior:

• Cancel pending cards (if any)

• Do not execute new entries

• Continue observing


⸻


LOCKOUT (hard stop)


Triggers:

• Daily max loss reached

• Logging failure

• Data feed integrity failure

• Risk rule violation detected

• Operator presses KILL


Behavior:

• Disable arming

• Disable new Trade Cards

• Require explicit reset


⸻


5) Minimal Pseudocode (State Spine)


state = PAUSE


on_tick():

  if kill_switch: state = LOCKOUT

  if state == LOCKOUT: return


  update_market_context()


  if not pretrade_gates_pass():

      state = PAUSE

      return


  if state in [PAUSE, READY]:

      state = READY

      card = detect_and_build_trade_card()

      if card: notify_operator(card)


  if operator_armed_manual and state == READY:

      state = ARMED_MANUAL


  if operator_armed_auto and state == READY and auto_gates_pass():

      state = ARMED_AUTO


  if state == ARMED_AUTO and trigger_hit(card):

      execute_bracket(card)

      state = IN_TRADE


  if state == IN_TRADE:

      manage_position()

      if position_closed(): state = PAUSE
