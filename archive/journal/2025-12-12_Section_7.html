FORMAL STATE DIAGRAMS


MCG · NI-AI · EchoAuth


This block formalizes system behavior as deterministic states and transitions.

Diagrams are expressed textually so they can be rendered visually later without changing logic.


The guiding principle is:


State defines capability; transitions define safety.


⸻


1. MCG — STATE MACHINE


(Decision Discipline Layer)


States

• OBSERVE

• CONFIRM

• EXECUTE


Initial State

• OBSERVE


Allowed Transitions

• OBSERVE → CONFIRM

Condition: Repeated, consistent structure detected

• CONFIRM → EXECUTE

Condition: All confirmation rules satisfied

• CONFIRM → OBSERVE

Condition: Structure weakens or becomes inconsistent

• EXECUTE → OBSERVE

Condition: Boundary breach, timeout, structure loss, human revocation


Forbidden Transitions

• OBSERVE → EXECUTE (never allowed)

• Any → EXECUTE without passing CONFIRM

• EXECUTE → CONFIRM (no partial execution)


Invariant

• If state ≠ EXECUTE, action is prohibited.


⸻


2. ECHOAUTH — AUTHORIZATION STATE MACHINE


(Permission & Protection Layer)


States

• DENY

• ALLOW (BOUNDED)


Initial State

• DENY


Allowed Transitions

• DENY → ALLOW (BOUNDED)

Condition:

• Valid identity

• Valid role

• MCG state = EXECUTE

• Explicit authorization within bounds

• ALLOW (BOUNDED) → DENY

Condition:

• MCG exits EXECUTE

• Authorization expires

• Boundary violation

• Human revocation

• Failure detected


Forbidden Transitions

• DENY → ALLOW without MCG = EXECUTE

• ALLOW → ALLOW (no escalation)


Invariant

• Authorization is temporary and revocable.


⸻


3. NI-AI — OPERATIONAL MODE MACHINE


(Reasoning & Alignment Layer)


Modes

• PASSIVE EXPLANATION

• STRUCTURAL EXPLANATION

• EXECUTION CONTEXT EXPLANATION


Initial Mode

• PASSIVE EXPLANATION


Mode Transitions

• PASSIVE → STRUCTURAL

Condition: MCG enters CONFIRM

• STRUCTURAL → EXECUTION CONTEXT

Condition: MCG enters EXECUTE

• ANY → PASSIVE

Condition: MCG returns to OBSERVE or failure occurs


Invariant

• NI-AI never issues commands or permissions.

• NI-AI mirrors system state for human understanding only.


⸻


4. CROSS-SYSTEM STATE COORDINATION


Coordination Rules

• MCG state changes propagate outward

• EchoAuth enforces based on MCG state

• NI-AI reflects based on MCG and EchoAuth outcomes


No system can:

• Force another system’s state

• Skip a transition

• Create hidden paths


⸻


5. GLOBAL SAFETY INVARIANTS


These invariants apply across all diagrams:

1. Default Denial

Execution and authorization start denied.

2. Single-Path Progression

All execution paths pass through confirmation.

3. Immediate De-Escalation

Any failure forces return to passive states.

4. Reversibility

All execution states are temporary.

5. Auditability

Every transition is logged.
