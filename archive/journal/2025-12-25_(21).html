PAGE 1 — SYSTEM IDENTITY


SniperBot (Canonical Definition)


SniperBot is a rule-governed trade execution and risk management system.

It does not predict markets. It enforces discipline.


SniperBot exists to:

• Execute trades only when predefined structural, risk, and context conditions are satisfied

• Manage exits automatically

• Preserve capital through restraint and governance


Default state is NO TRADE.


SniperBot is human-authorized, context-aware, and bounded by governance.

PAGE 2 — CORE PRINCIPLES

1. Discipline over intelligence

2. Execution over prediction

3. Governance over autonomy

4. No-trade is a valid outcome

5. Automation handles execution, not judgment


Any system violating these principles is not SniperBot.

PAGE 3 — OPERATOR FLOW (END-TO-END)


Step 0 — Boot

• Load configuration

• Initialize governance

• Set state = PAUSE


Step 1 — Day Classification


SniperBot classifies (read-only):

• Market regime: Trend / Range / Expansion / Transition

• Volatility bucket: ~3.5% or ≥6%

• News status: Green / Yellow / Red


If regime ≠ allowed OR news = Red → remain PAUSE.


Step 2 — Setup Detection


SniperBot watches only whitelisted entries.

No discretionary scanning.


Step 3 — Trade Card Generation


When setup qualifies:

• Entry trigger

• Structure stop

• TP ladder

• Position size (risk % based)


No trade exists without a card.


Step 4 — Arm Window


Human selects:

• ARM MANUAL (alerts only)

• ARM AUTO (1 trade max)


If auto conditions fail → manual only.


Step 5 — Execution

• Entry + stop + TP placed simultaneously

• All actions logged


Step 6 — Exit Management

• TP ladder executes

• Runner optional

• Time stop enforced


Step 7 — Shutdown


Auto shuts down on:

• Daily loss hit

• Regime invalidation

• Feed issues

• Kill switch


⸻


PAGE 4 — ENTRY WHITELIST


Allowed entries:

• Break & retest (confirmed)

• VWAP flip + hold

• Key level reclaim with structure


Not allowed:

• Counter-trend

• Mean reversion

• News reaction

• Impulse entries

• “Looks strong” logic


Indicators confirm — they never trigger.


⸻


PAGE 5 — RISK MANAGEMENT ENGINE


Risk Authority


Risk is the primary authority, above signal strength.


Rules:

• Fixed % risk per trade

• Position size calculated from stop distance

• If stop too wide → trade rejected


Typical ranges:

• Manual: 0.5%–1%

• Auto: 0.25%–0.5%


Max trades/day:

• Auto: 1 (2 only if first is partial win)


⸻


PAGE 6 — STOP LOSS LOGIC


Stop loss must:

• Invalidate the thesis

• Be defined before entry

• Never widen


Optional:

• Time stop

• Break-even after TP1

• Session cutoff stop


If stop logic cannot validate → no execution.


⸻


PAGE 7 — TAKE PROFIT STRUCTURE


Mandatory ladder:

• TP1: logical structure target

• TP2: extension (ATR / measured move)

• Runner: optional trailing stop


No chasing.

No adding size.

No emotional overrides.


⸻


PAGE 8 — VOLATILITY ADAPTATION


~3.5% Range Days

• Smaller targets

• Faster exits

• Reduced trade frequency


≥6% Expansion Days

• Wider structure-valid stops

• Larger targets

• Scaling logic allowed


If volatility regime changes mid-trade → manage exits only, no new entries.


⸻


PAGE 9 — AUTO-MODE CONDITIONS (LOCKED)


Auto-Mode is allowed only if ALL are true:

• Human arms auto for the session

• Regime = pre-approved

• Entry type = whitelisted

• Risk ≤ 0.5%

• Max trades/day enforced

• Kill switch active


Auto-Mode expires:

• End of session

• On loss cap

• On regime uncertainty


Auto-Mode ≠ autonomy.
