PART I — Executive Summary (Page 1)


1. Problem Definition


As AI-enabled systems gain real-world execution affordances (e.g., medication dispense, device control, delegated care actions), safety risk shifts from cognitive error to unauthorized execution.


We define:


Execution Uncertainty — the risk that a system performs a real-world action without valid, current, and scope-bound human authority.


Existing safety work primarily addresses model cognition (interpretability, alignment, policy). Limited work addresses runtime authority-to-action binding.


⸻


2. Intervention


EchoAuth is a deterministic execution constraint layer that enforces authority verification prior to action.


It requires:

• Explicit authorization grant

• Cryptographic signature validation

• Time-bounded freshness

• Canonical payload hashing (RFC 8785)

• Scope verification

• Replay prevention

• Exclusive state locking

• Atomic commit-point nonce consumption


If any validation step fails, execution does not occur.

PART II — Appendix A


Deterministic Validation Contract


8-Step Sequential Validation


The CEG processes AuthorizationGrant in strict linear order:

1. Schema & Decision Validation

2. Freshness (issued_at_ms ≤ now_ms ≤ expires_at_ms)

3. Signature Verification

4. Replay Check (grant_id unused)

5. Payload & Context Hash Verification (JCS canonicalization)

6. Scope & Binding Verification

7. State Lock Acquisition (Exclusive)

8. Commit Point (Nonce Consumption + Side Effect Execution)


Failure at any step results in immediate exit.


⸻


Commit Point Invariant


Nonce mutation occurs only after:

• Successful validation of Steps 1–7

• Successful lock acquisition


This prevents replay and race-condition execution.
