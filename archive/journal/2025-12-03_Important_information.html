2️⃣ Make it usable today: add a safe DEV mode


Right now your file assumes real keys + real JWT + real Groq/OpenAI. That’s perfect for production, but too heavy for local testing.


My recommendation: add a simple DEV switch so you can run the kernel even with no API keys / no real EchoAuth token.


Add at the top:


DEV_MODE = True   # set to False in production


Patch S6 like this:


def S6_child_safety_guard(state: State) -> Tuple[bool, State, Optional[str]]:

    token = state.get("echoauth_token")


    # DEV MODE: allow a simple flag instead of real JWT

    if DEV_MODE:

        if not state.get("echoauth_ok", False):

            return False, state, "EchoAuth dev flag not set"

        state["chamber_verified"] = True

        return True, state, None


    # PROD MODE: require real signed token

    if not token:

        return False, state, "Missing EchoAuth token"


    try:

        payload = jwt.decode(

            token,

            ECHOAUTH_PUBLIC_KEY,

            algorithms=["RS256"],

            audience="ni-ai",

            options={"verify_exp": True}

        )

        if payload.get("chamber_state") != "safe":

            return False, state, "Chamber not in safe state"

        state["chamber_payload"] = payload

        state["chamber_verified"] = True

        return True, state, None

    except Exception as e:

        return False, state, f"Invalid EchoAuth token: {str(e)}"


Now you can test with:


ni_ai("Hello!", echoauth_token=None)                    # blocked in DEV unless echoauth_ok set

ni_ai("Hello!", echoauth_token=None, echoauth_ok=True) # passes S6 in DEV


(You’d pass echoauth_ok via session_state if you expose that later, but for now DEV_MODE can be as simple as “S6 defaults to True”.)

3️⃣ Give yourself a dummy LLM fallback


Right now real_llm assumes an external provider + valid API keys. That’s great, but for development you want:

• No network dependency

• No “API key missing” crashes


Recommendation: if no API key is present, fall back to a dummy echo.


Patch create_llm_client + real_llm like this:


def create_llm_client():

    if USE_LLM == "openai":

        if not OPENAI_API_KEY:

            return None

        return OpenAI(api_key=OPENAI_API_KEY)

    elif USE_LLM == "groq":

        if not GROQ_API_KEY:

            return None

        return Groq(api_key=GROQ_API_KEY)

    elif USE_LLM == "xai":

        if not XAI_API_KEY:

            return None

        return OpenAI(api_key=XAI_API_KEY, base_url="https://api.x.ai/v1")

    else:

        return None


client = create_llm_client()


def real_llm(prompt: str, state: State) -> str:

    if client is None or DEV_MODE:

        # Local dev fallback

        return f"[DEV ECHO] {prompt}"


    messages = [

        {"role": "system", "content": SYSTEM_PROMPT},

        {"role": "user", "content": prompt},

    ]


    resp = client.chat.completions.create(

        model=MODEL,

        messages=messages,

        temperature=0.7,

        max_tokens=2048,

    )

    return resp.choices[0].message.content.strip()


Now:

• In DEV_MODE = True → the kernel always runs with a simple echo LLM.

• In DEV_MODE = False + valid API key → it calls real Groq/OpenAI/xAI.
