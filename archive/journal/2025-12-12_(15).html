🧱 ECHOAUTH — TECHNICAL MVP STACK


Caregiver-Governed, Non-Verbal Support System


⸻


1. MVP PRINCIPLES (READ FIRST)


This MVP prioritizes:

• Reliability over novelty

• Security over speed

• Auditability over intelligence

• Human accountability over automation


No experimental tech. No speculative AI.


⸻


2. FRONTEND (CARE GIVER / PARENT)


Recommended Stack

• Web App (Responsive)

• React (or Next.js)

• HTML / CSS / TypeScript


Why

• Fast to build

• Easy onboarding for caregivers

• Works on phones, tablets, desktops

• No app store delays for pilots


Core Screens

• Secure login

• Caregiver dashboard

• Observation logging form

• History / timeline view

• User profile & role display


⸻


3. BACKEND (CORE LOGIC)


Recommended Stack

• API: Python (FastAPI) or Node.js (Express)

• Auth: JWT or session-based auth

• RBAC: Role-based access control (caregiver / parent / admin)


Responsibilities

• Authentication & authorization

• Role enforcement

• Observation storage

• Logging & audit trails

• Safety gating logic


No inference engines.

No background autonomous tasks.


⸻


4. DATABASE


Recommended

• PostgreSQL (primary)

• Optional: Redis (sessions / caching)


Data Stored

• Users (ID, role, permissions)

• Sessions

• Observations (timestamped, descriptive)

• Logs (append-only)


Data NOT Stored

• Emotional labels

• Predictions

• Psychological profiles

• Derived behavioral states


⸻


5. AUTHENTICATION & ACCESS


MVP Requirements

• Email + password or SSO

• Secure password hashing

• Session expiration

• Role-based permissions


Rules

• Every request tied to a user

• No anonymous access

• No system-initiated child interaction


⸻


6. NON-VERBAL OBSERVATION MODULE


Input Only

• Free-text observation notes

• Optional structured tags (caregiver-defined)

• Timestamp auto-generated

• Optional context fields


Explicit Constraints

• No automated interpretation

• No scoring

• No recommendations

• No alerts without caregiver action


⸻


7. LOGGING & AUDITABILITY (CRITICAL)


Log Every:

• Login

• Logout

• Observation create/edit

• Access attempt (allowed / blocked)

• Permission change


Log Fields

• User ID

• Role

• Action

• Timestamp

• Result


Logs must be:

• Immutable or append-only

• Human-readable

• Exportable


⸻


8. SAFETY GATING (MCG IMPLEMENTATION)


Simple Rule Engine


Before any action:

1. Is user authenticated?

2. Is role authorized?

3. Is action allowed in this context?

4. Is action non-clinical?


If any answer is NO → block action.


This can be implemented as middleware.


⸻


9. DEPLOYMENT (MVP SAFE)


Hosting

• AWS / GCP / Azure (basic tier)

• Dockerized services

• HTTPS only


Environment Separation

• Dev

• Staging

• Production


⸻


10. SECURITY BASELINE


Required

• HTTPS

• Encrypted passwords

• Environment variable secrets

• Basic monitoring


Not Required (for MVP)

• Advanced ML security

• Custom crypto

• Hardware auth


⸻


11. WHAT IS EXPLICITLY OUT OF SCOPE (MVP)


🚫 No AI models

🚫 No emotion detection

🚫 No behavioral prediction

🚫 No biometric claims

🚫 No Flame Self logic in backend


Flame Self = frontend UX only, optional.


⸻


12. MVP SUCCESS CRITERIA


MVP is successful if:

• Caregivers can log in securely

• Roles are enforced correctly

• Observations are logged & reviewed

• Logs are auditable

• Unsafe actions are blocked
