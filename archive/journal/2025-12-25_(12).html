1) Deployment Modes (Three, and Only Three)


MODE A — PAPER (Default First Contact)


What’s live

• State machine (Page 16)

• Gates (Page 17)

• Trade Cards (Page 18)

• Daily checklist (Page 21)


What’s disabled

• Capital execution

• Auto-Mode

• Position management tied to real orders


Purpose

• Validate discipline, not profitability

• Observe “NO TRADE” behavior

• Stress-test gates and expirations


Rules

• Minimum duration: 10 trading sessions

• No skipping days

• All Trade Cards logged


Exit criteria (all required)

• Zero rule violations

• No impulse overrides

• You trust PAUSE more than signals


⸻


MODE B — HYBRID (Small Live, Human Execute)


What’s live

• Real capital (small)

• Manual execution only

• SniperBot manages:

• Stops/TP logic (conceptually or via alerts)

• Logging

• Lockouts


What’s disabled

• Auto execution


Purpose

• Test real fills, spreads, slippage

• Calibrate emotions with money at risk

• Prove exits are respected under pressure


Rules

• Risk per trade ≤ 0.5%

• Max trades/day = 1

• No options unless liquidity is excellent


Exit criteria (all required)

• Stops respected 100%

• TP ladder respected

• Losses feel contained, boring

• No config changes mid-session


⸻


MODE C — PERMISSIONED AUTO (Rare, Narrow)


What’s live

• Auto execution for one setup

• One trade/day

• Strict Auto-Mode gates (Page 9 & 20)


What’s disabled

• Strategy switching

• Counter-trend

• News reaction

• Scaling beyond plan


Purpose

• Remove execution error

• Not increase trade frequency


Rules

• Auto must be armed daily

• Auto expires after:

• One trade

• Any gate failure

• Session end


Exit criteria

• Auto feels unnecessary most days

• Manual still preferred by default


⸻


2) Ignition Toggles (Hard Switches)


These toggles must exist and be visible:

• SYSTEM_POWER: OFF / ON

• MODE_SELECT: PAPER / HYBRID / AUTO

• ARM_MANUAL: OFF / ON

• ARM_AUTO: OFF / ON

• KILL_SWITCH: OFF / ON (always available)


Invariant

• SYSTEM_POWER = ON

• MODE_SELECT ≠ AUTO

→ Auto execution cannot occur


⸻


3) Daily Ignition Sequence (Every Session)

1. SYSTEM_POWER → ON

2. MODE_SELECT → PAPER or HYBRID

3. Complete Page 21 Pre-Market Checklist

4. Verify STATE = PAUSE

5. Observe until READY

6. ARM_MANUAL only when present

7. ARM_AUTO only if explicitly intended


If any step feels rushed → STOP


⸻


4) Auto-Mode Ignition (Exact Steps)


Auto may be armed only after:

1. MODE_SELECT = AUTO

2. Pre-market checklist complete

3. Regime = Trend or Expansion

4. No Red news

5. Risk % locked

6. Kill switch tested


Auto ignition expires automatically:

• After one execution

• On any downgrade to PAUSE

• On LOCKOUT


⸻


5) Rollback Rules (No Ego)


At any time, revert:

• AUTO → HYBRID

• HYBRID → PAPER

• PAPER → OFF


Rollback is not failure.

It is governance working.


⸻


6) Failure Handling (Expected, Not Feared)


If something goes wrong

• Trigger LOCKOUT

• End session

• Log reason

• Resume next day in PAPER


Never “fix live.”


⸻

Deployment is not about turning the system on.

It’s about proving you can turn it off.
