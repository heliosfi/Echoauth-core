SECTION IV — ENGINEERING HANDOFF (CTO / LAB-READY)


4.1 System Intent (Non-Negotiable)


EchoAuth is a caregiver-guided, non-verbal communication support system for children, including non-verbal and ASD users.


The system exists to:

• Support safe observation

• Enable caregiver learning and consistency

• Maintain accountability and traceability

• Enforce strict safety boundaries


EchoAuth does not interpret, diagnose, infer internal states, or act autonomously.


⸻


4.2 Human Role Boundary (Hard Constraint)


Child        → expresses non-verbal signals

Caregiver    → observes, interprets, decides

EchoAuth     → logs, structures, governs, safeguards


All meaning and decisions remain human-driven.


EchoAuth is infrastructure, not an authority.


⸻


4.3 MVP Technical Stack (Approved)


Frontend

• React or Next.js

• Caregiver dashboard (primary)

• Optional child-facing display (read-only, non-interactive)


Backend

• FastAPI (Python) or Express (Node.js)

• REST-only architecture

• No autonomous agents

• No model-driven decision loops


Authentication & Access

• Session or JWT-based authentication

• Role-Based Access Control (RBAC)

• Caregiver

• Parent

• Admin


Data Layer

• PostgreSQL (relational)

• Append-only audit log (separate table)


Infrastructure

• AWS / GCP / Azure (basic deployment)

• No edge inference required for MVP

• No biometric hardware dependency


⸻


4.4 Governance Layer — MCG (Implementation Definition)


MCG (Melanin Cognitive Governance) is implemented as middleware, not a model.


MCG Responsibilities:


1. Identity Anchoring

• Every action is tied to an authenticated human user

• No anonymous system actions

• No child-initiated system control


2. Interpretability Enforcement

• All data entries are human-generated

• All actions are logged and reviewable

• No hidden transformation layers


3. Safety Gating

• Explicitly blocks:

• diagnosis

• emotional inference

• behavioral prediction

• autonomous recommendations

• Explicitly allows:

• observation logging

• caregiver notes

• historical review

• exports for human review


MCG operates before any data is written or displayed.


⸻


4.5 NI-AI Scope (Engineering-Relevant Only)


NI-AI is not a model in the MVP.


NI-AI functions as:

• Design constraints

• Interaction rules

• Safety philosophy


NI-AI Enforcement Rules:


- Human-in-the-loop only

- AI may assist organization, not judgment

- No internal state claims

- No consciousness modeling


Engineers implement NI-AI through:

• UX limitations

• Feature exclusions

• Governance rules (via MCG)


⸻


4.6 Core Data Objects (Minimum Required)


User

- id

- role (caregiver / parent / admin)

- organization_id


Child_Profile

- id

- caregiver_id

- non-identifying metadata only


Observation

- id

- child_id

- caregiver_id

- timestamp

- observation_type (gesture, sound, movement, context)

- free-text notes (human-entered)


Audit_Log

- id

- user_id

- action_type

- timestamp

- allowed / blocked


No biometric inference fields permitted in MVP.


⸻


4.7 Explicit Non-Goals (Engineering Guardrails)


Engineers must not implement:


❌ Emotional analysis

❌ Diagnostic logic

❌ Predictive behavior models

❌ Autonomous child interaction

❌ Child-controlled workflows

❌ Black-box AI decisions


If a feature reduces caregiver agency, it violates scope.


⸻


4.8 MVP Build Order (Enforced)


1. Authentication + RBAC

2. Caregiver dashboard UI

3. Observation logging

4. Audit logging

5. MCG middleware enforcement

6. Export (CSV / PDF) for caregiver review


Stop at Step 6.

Anything beyond this requires governance review.


⸻


4.9 Deployment Readiness Statement


This MVP is:

• Caregiver-led

• Child-safe

• Non-diagnostic

• Fully auditable

• Grant-compliant

• Lab-deployable


EchoAuth is a safety-first communication support infrastructure, not a clinical decision system.
