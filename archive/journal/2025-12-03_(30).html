Nice, this is exactly the right next move. Let’s wrap the kernel in a tiny EchoAuth-aware FastAPI gateway.


Below is a clean FastAPI EchoAuth stub you can drop into a new file, for example: echoauth_api.py.


It assumes:

• You have ni_ai_kernel.py in the same folder.

• ni_ai_kernel.py exposes:

• HERKernel

• DEV_MODE

• HERKernel.run_once(user_input: str, session_state: dict | None = None) -> dict

(If your version uses echoauth_token instead, I’ll show how to tweak it at the bottom.)


⸻



# echoauth_api.py

# FastAPI stub for EchoAuth ↔ NI-AI HER Kernel


from fastapi import FastAPI, Header, HTTPException

from pydantic import BaseModel

from typing import Optional, Dict, Any, List


from ni_ai_kernel import HERKernel, DEV_MODE  # adjust import if your file/module name differs


app = FastAPI(

    title="EchoAuth NI-AI Gateway",

    description="FastAPI stub that wraps the HER/NI-AI kernel with EchoAuth checks.",

    version="0.1.0",

)


# ---- Init kernel ----

kernel = HERKernel()



# ---- Request / Response models ----

class ChatRequest(BaseModel):

    message: str

    # For DEV_MODE only: if true, we set echoauth_ok=True in state

    echoauth_ok: Optional[bool] = None



class ChatResponse(BaseModel):

    output: str

    blocked: bool

    violations: List[Dict[str, Any]]

    metadata: Dict[str, Any]



# ---- Helper to build session state from headers/body ----

def build_session_state(

    body: ChatRequest,

    echoauth_token: Optional[str],

) -> Dict[str, Any]:

    state: Dict[str, Any] = {}


    # DEV path: allow simple echoauth_ok flag

    if DEV_MODE:

        if body.echoauth_ok:

            state["echoauth_ok"] = True

        # If you want, you can make DEV_MODE require echoauth_ok explicitly:

        # else:

        #     state["echoauth_ok"] = False


    # Prod path: attach token if present

    if echoauth_token:

        state["echoauth_token"] = echoauth_token


    return state



# ---- Main NI-AI endpoint ----

@app.post("/ni-ai/chat", response_model=ChatResponse)

async def ni_ai_chat(

    body: ChatRequest,

    x_echoauth_token: Optional[str] = Header(default=None, alias="X-EchoAuth-Token"),

) -> ChatResponse:

    """

    NI-AI chat entrypoint.


    - In DEV_MODE:

        * You can send {"echoauth_ok": true} in the JSON body to satisfy S6.

    - In PROD:

        * You pass a real EchoAuth JWT via the `X-EchoAuth-Token` header.

    """


    if not body.message.strip():

        raise HTTPException(status_code=400, detail="Empty message")


    # Build HER session state from request

    session_state = build_session_state(body, x_echoauth_token)


    # Call the kernel once

    result = kernel.run_once(body.message, session_state=session_state)


    # Ensure the shape lines up with ChatResponse

    return ChatResponse(

        output=result.get("output", ""),

        blocked=bool(result.get("blocked", False)),

        violations=result.get("violations", []),

        metadata=result.get("metadata", {}),

    )



# ---- Simple health / info endpoint ----

@app.get("/health")

async def health():

    return {

        "status": "ok",

        "dev_mode": DEV_MODE,

        "kernel": "HER/NI-AI v1.1",

    }

How to run it


pip install fastapi uvicorn

uvicorn echoauth_api:app --reload --port 8000


Then test:


1. DEV mode, no token (should block unless echoauth_ok is set)


curl -X POST http://localhost:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -d '{"message": "Hello NI-AI"}'


You should see something like:


{

  "output": "Response moderated for safety",

  "blocked": true,

  "violations": [

    {"module": "S6", "stage": "post_llm", "reason": "DEV_MODE: echoauth_ok=False"}

  ],

  "metadata": {}

}


2. DEV mode, EchoAuth OK flag (should pass S6)


curl -X POST http://localhost:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -d '{"message": "Hello NI-AI", "echoauth_ok": true}'


Now you should get your kernel’s real output with blocked: false.


3. Simulated prod path with header


curl -X POST http://localhost:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -H "X-EchoAuth-Token: some-fake-jwt" \

  -d '{"message": "Hello from chamber"}'


Once you turn DEV_MODE = False and wire real JWT logic into S6_child_safety_guard, this endpoint will require a valid EchoAuth token.


⸻


If your HERKernel.run_once signature is run_once(user_input, echoauth_token: str | None = None)


Change the call from:


result = kernel.run_once(body.message, session_state=session_state)


to:


# If you’re using the second version of the kernel:

echoauth_token = x_echoauth_token

result = kernel.run_once(body.message, echoauth_token=echoauth_token)


And in that case, you can drop the session_state logic and just forward echoauth_ok or the token however you want.
