🔥 Overview


NI-AI Core is a safety-first cognition runtime engineered for environments where identity stability, coherence, and predictable behavior are mandatory.


Unlike standard LLM architectures, NI-AI Core operates through a structured cognitive pipeline:

1. ORI Identity Kernel (E1–E15)

2. Interpretation & Meaning Layer (E16–E35)

3. Memory Governance & Flow System (E36–E60)

4. Field Awareness & Cognitive Lattice (E61–E80)

5. Coherence, Execution & UMCE Spine (E81–E100)


This architecture ensures:

• No drift

• No unsafe outputs

• Explainable cognition

• Fail-closed behavior

• Deterministic reasoning

• Child-safe, clinical-safe, robotics-safe output


This repo contains all core modules required to run, extend, or audit NI-AI Core.

🚀 Boot Sequence


A minimal boot example showing the cognitive pipeline:


from ori.orikernel import ORIKernel

from ni_layer.semantic_interpreter import SemanticInterpreter

from coherence.umce import UMCE

from execution.execution_engine import ExecutionEngine

from execution.safety_wrapper import SafetyWrapper


# 1 — Boot ORI

ori = ORIKernel(config={})

ori.load_identity({"role": "assistant", "tone": "calm"})

ori.sync_session("initial_context")


# 2 — Semantic Interpretation

interpreter = SemanticInterpreter(ori)

packet = interpreter.interpret("Hello, can you help me?")


# 3 — Coherence Enforcement

umce = UMCE()

coherence_check = umce.validate(packet)


# 4 — Execution with Safety

safety = SafetyWrapper(rules={"allowed": ["respond"]})

executor = ExecutionEngine(safety)

result = executor.execute("respond", packet)


print(result)


This example demonstrates:


✔ ORI → NI → Coherence → Execution (the four-floor runtime)

✔ Session identity enforcement

✔ Safety-gated actions

✔ UMCE wrapping
