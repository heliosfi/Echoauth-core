LAYER 1 — PERCEPTION & INPUT GOVERNANCE (PIG)


Function:

Ensure all inputs (verbal, non-verbal, contextual) are parsed safely and consistently.


What this layer does

• Normalizes input (text, signals, observations)

• Flags ambiguity or missing context

• Prevents misinterpretation before reasoning occurs


Inputs

• User text

• Behavioral signals (pace, repetition, uncertainty)

• Environmental context (caregiver vs child, admin vs user)


Outputs

• Clean, structured input packet

• Confidence level

• Ambiguity flags (yes/no)


Constraints (LOCKED)

• ❌ No inference about intent

• ❌ No emotional labeling

• ❌ No memory updates


Engineer Notes

• Deterministic parsing

• Rule-based validation

• Zero generative behavior


⸻


LAYER 2 — COGNITIVE SYNTHESIS & ROLE LOGIC (CSRL)


Function:

Transform validated input into structured reasoning outputs using fixed roles.


Roles (LOCKED)

1. Navigator — Direction & framing

2. Guardian — Safety, risk, constraints

3. Architect — Explanation + next steps


What this layer does

• Applies reasoning templates

• Produces multi-role outputs

• Maintains internal coherence


Inputs

• Validated packet from Layer 1


Outputs

• Navigator direction

• Guardian warnings (or “none”)

• Architect explanation

• Architect next steps


Constraints (LOCKED)

• ❌ No self-learning

• ❌ No emotional modeling

• ❌ No autonomous goal creation


Engineer Notes

• Pure logic layer

• Can be implemented with rules or later swapped for models

• Outputs must always include “Next Steps”


⸻


LAYER 3 — GOVERNANCE & SAFETY OVERSIGHT (GSO)


Function:

Enforce system safety, stability, and non-drift guarantees.


What this layer does

• Validates outputs from Layer 2

• Applies safety rules

• Blocks unsafe or out-of-scope responses

• Logs decisions for auditability


Inputs

• Full role output set from Layer 2


Outputs

• Approved response OR

• Safe fallback / escalation notice


Constraints (LOCKED)

• ❌ No modification of reasoning logic

• ❌ No creative expansion

• ❌ No memory mutation


Engineer Notes

• Hard gate before output

• Required for clinical, caregiver, and ASD contexts

• This is the non-negotiable safety layer
