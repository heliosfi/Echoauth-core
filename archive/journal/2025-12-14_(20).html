4.1 Purpose of Stage 4 (DFD)


This section documents what data moves where, why it moves, and what boundaries prevent misuse.


Key constraint:

Data flow supports safety, authorization, and transparency only.

It does not support diagnosis, prediction, or intent inference as truth.


⸻


4.2 DFD Level 0 — Context Diagram (Text)


External Entities

• E1 — Child / Participant (Non-verbal supported)

• E2 — Caregiver / Parent

• E3 — Authorized Admin / Reviewer (School/Clinic/Gov)

• E4 — External Systems (Optional, integrated only by explicit agreement)

• e.g., school systems, clinic systems, identity providers, reporting endpoints


Core Process

• P0 — NI-AI Platform (MCG + EchoAuth + NI-AI)


High-Level Data Flows

• F1: Interaction Input (E1/E2 → P0)

Minimal interaction signals (text, taps, time/sequence) and optional sensory-safe inputs if enabled.

• F2: System Response / Status Output (P0 → E1/E2)

Explanations, safe prompts, “pause/observe” notices, caregiver guidance.

• F3: Policy / Configuration Updates (E3 → P0)

Approved rules, role permissions, thresholds, pilot settings.

• F4: Audit & Safety Logs (P0 → E3)

Event logs: allow/deny/pause, timestamps, system state transitions.

• F5: Optional Integration Exchange (P0 ↔ E4)

Minimal status results only (e.g., authenticated session token, compliance report), never raw child signals by default.

4.3 DFD Level 1 — Internal Data Flow (Text)


Processes (P)

• P1 — Client Interface Layer

App/web dashboard for caregivers/admins; child-facing UI supports non-verbal interaction.

• P2 — Signal Intake & Minimization

Normalizes input; strips unnecessary identifiers; assigns session ID.

• P3 — MCG Governance Check (Final Authority)

Evaluates state and policy; outputs Allow / Pause / Deny.

• P4 — EchoAuth Authorization Gate

Verifies role and permission; default deny; returns Allow / Challenge / Deny.

• P5 — NI-AI Explanation Layer (Read-Only)

Generates “why-pause / why-deny / why-no-action” explanations using applied rules.

• P6 — Audit Logging & Monitoring

Records events; writes immutable logs; triggers alerts to authorized personnel when needed.


Data Stores (D)

• D1 — Policy & Configuration Store

Governance rules, thresholds, pilot configuration, role definitions.

• D2 — Authorization & Role Store (EchoAuth)

Role bindings (caregiver/admin), device/session keys, permission states.

• D3 — Audit Log Store (Immutable)

State transitions, allow/deny/pause decisions, security events, timestamps.

• D4 — Session-Bounded Context Store (Optional)

Temporary session context summaries; expires automatically; no identity expansion.


⸻


4.4 Level 1 — Primary Flows (F)

• F1: Input Capture Flow

E1/E2 → P1 → P2

Minimal interaction input enters and is normalized.

• F2: Governance Evaluation Flow

P2 → P3 ↔ D1

MCG checks policy + state. Produces Allow/Pause/Deny.

• F3: Authorization Verification Flow

P3 → P4 ↔ D2

EchoAuth confirms the acting role (caregiver/admin). Default deny.

• F4: Explanation Flow (Read-Only)

P3/P4 → P5

NI-AI generates explanation of what happened and why.

• F5: Response Return Flow

P5 → P1 → E1/E2

Outputs are informational and sensory-safe; non-action is normal.

• F6: Logging & Audit Flow

P3/P4/P5 → P6 → D3

Logs all state changes, denials, pauses, and approvals.

• F7: Optional Admin Review Flow

D3 → E3 (via P1 dashboards)

Reviewers see audit summaries, not inference claims.

• F8: Optional External Integration Flow

P4/P6 ↔ E4

Minimal tokens/status. No raw child interaction exported by default.


⸻


4.5 Trust Boundaries (Explicit)

• B1 — Client ↔ Platform Boundary

TLS 1.3; session isolation; no privileged access from child UI.

• B2 — Governance/Authorization ↔ Data Stores

Least privilege; encryption at rest; audited access.

• B3 — Platform ↔ External Systems (Optional)

Allow-listed endpoints only; minimal required sharing; explicit agreements.


⸻


4.6 Non-Verbal Protection Clause (Data Flow)


Across all flows:

• No speech is required.

• Silence/stillness are treated as valid states.

• No “lack of response” is interpreted as intent.

• The system can remain in observe/pause indefinitely without penalty.
