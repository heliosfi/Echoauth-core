SniperBot Operator Flow v1.0


0) Boot

• Load config: tickers, max risk, allowed entry types, daily loss cap

• Set state: PAUSE / NO-TRADE


1) Classify the day


Bot outputs (read-only):

• Regime: Trend / Range / Expansion / Transition

• Volatility bucket: ~3.5% or ≥6%

• News status: Green / Yellow / Red


If Regime ≠ Allowed or News = Red → stay PAUSE.


2) Candidate setup detection


Bot watches for whitelisted setups only:

• Break & retest

• VWAP flip + hold

• Key level reclaim


When detected → it prepares a “trade card”:

• Entry price + trigger condition

• Stop location (structure invalidation)

• TP1/TP2/Runner logic

• Position size (risk % based)


3) “Arm window” (human-in-the-loop)


Bot asks:

• ARM MANUAL (alerts only)

• ARM AUTO (1 trade max) (execution permission)


Auto can only be armed if all Auto-Mode gates pass.


4) Execute (only inside an armed window)


If armed + trigger hits:

• Place order

• Place stop + TP bracket immediately

• Log everything


5) Manage exits

• TP ladder executes

• Optional trail for runner

• Time stop triggers if no follow-through


6) Shutdown conditions


Auto shuts off instantly on:

• Max daily loss

• regime change / uncertainty flag

• feed issues

• spreads widen

• manual kill switch
