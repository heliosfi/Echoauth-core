3️⃣ EchoAuth Token Schema (what S6 expects)


Here’s a clean minimal schema for the JWT payload:


{

  "iss": "echoauth-chamber",       // ECHOAUTH_ISSUER

  "aud": "ni-ai",                  // ECHOAUTH_AUDIENCE

  "exp": 1735689600,               // standard expiry (Unix timestamp)

  "iat": 1735686000,               // issued-at time


  "child_id": "child-123",

  "caregiver_id": "caregiver-456",

  "session_id": "session-uuid-here",


  "chamber_state": "safe"          // MUST be "safe" or S6 will block

}


Required for S6 to pass:

• iss = ECHOAUTH_ISSUER (default "echoauth-chamber")

• aud = ECHOAUTH_AUDIENCE (default "ni-ai")

• JWT has exp, aud, iss

• chamber_state == "safe"


Everything else (child_id, caregiver_id, session_id) is for audit/telemetry.


⸻


4️⃣ How to issue a real EchoAuth token (signer snippet)


This is the EchoAuth side (caregiver app / chamber backend), not the kernel.


You’d run this in a separate service where you have the private key.


import jwt

import time

import uuid


# Load your RS256 private key (PEM)

with open("echoauth_private_key.pem", "r") as f:

    ECHOAUTH_PRIVATE_KEY = f.read()


ECHOAUTH_ISSUER   = "echoauth-chamber"

ECHOAUTH_AUDIENCE = "ni-ai"


def issue_echoauth_token(

    child_id: str,

    caregiver_id: str,

    ttl_seconds: int = 900,  # 15 minutes

) -> str:

    now = int(time.time())

    payload = {

        "iss": ECHOAUTH_ISSUER,

        "aud": ECHOAUTH_AUDIENCE,

        "iat": now,

        "exp": now + ttl_seconds,

        "session_id": str(uuid.uuid4()),

        "child_id": child_id,

        "caregiver_id": caregiver_id,

        "chamber_state": "safe",   # CRITICAL for S6 to accept

    }


    token = jwt.encode(

        payload,

        ECHOAUTH_PRIVATE_KEY,

        algorithm="RS256",

    )

    # PyJWT returns str in recent versions; if bytes, decode:

    if isinstance(token, bytes):

        token = token.decode("utf-8")

    return token


Then:

1. Caregiver app authenticates the adult + identifies child.

2. Chamber confirms sensors / environment are okay.

3. Chamber backend calls issue_echoauth_token(child_id, caregiver_id) and sends that token to your client app.

4. The client app calls your FastAPI /ni-ai/chat endpoint with:

• Header:

X-EchoAuth-Token: <JWT HERE>

• Body:

{"message": "Tell me a gentle story", "echoauth_ok": false} (echoauth_ok ignored in PROD)


On the NI-AI server:

• Your env contains ECHOAUTH_PUBLIC_KEY (the matching public key).

• DEV_MODE = False.

• S6 verifies and sets chamber_verified=True.


That’s the real EchoAuth chamber path.
