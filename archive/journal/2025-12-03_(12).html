2. Replace S6 with a real JWT-verifying EchoAuth gate


In ni_ai_kernel.py v1.4, find your existing S6_child_safety_guard and replace it completely with this version:


def S6_child_safety_guard(state: State) -> Tuple[bool, State, Optional[str]]:

    """

    EchoAuth gate.


    DEV_MODE:

        - Allows echoauth_ok=True for local testing.

        - Optionally tries to decode a token if present (good for staging).


    PROD (DEV_MODE=False):

        - Requires a valid RS256 JWT signed by EchoAuth.

        - Enforces chamber_state == "safe".

        - Enforces audience = "ni-ai".

    """

    token = state.get("echoauth_token")


    # ---------- DEV PATH ----------

    if DEV_MODE:

        # Simple flag for local testing (no JWT needed)

        if state.get("echoauth_ok"):

            state["chamber_verified"] = True

            return True, state, None


        # If you *do* pass a real token in dev, we try to decode and respect it

        if not token:

            return False, state, "DEV_MODE: echoauth_ok=False and no token provided"


        try:

            payload = jwt.decode(

                token,

                ECHOAUTH_PUBLIC_KEY,

                algorithms=["RS256"],

                audience="ni-ai",

            )

        except InvalidTokenError as e:

            return False, state, f"DEV_MODE: Invalid EchoAuth token: {e}"


        if payload.get("chamber_state") != "safe":

            return False, state, "DEV_MODE: chamber_state is not 'safe'"


        state["chamber_verified"] = True

        state["echoauth_payload"] = payload

        return True, state, None


    # ---------- PROD PATH ----------

    if not token:

        return False, state, "Missing EchoAuth token"


    try:

        payload = jwt.decode(

            token,

            ECHOAUTH_PUBLIC_KEY,

            algorithms=["RS256"],

            audience="ni-ai",

        )

    except InvalidTokenError as e:

        return False, state, f"Invalid EchoAuth token: {e}"


    # Hard safety: only talk when chamber is safe & child mode is active

    if payload.get("chamber_state") != "safe":

        return False, state, "EchoAuth chamber_state != 'safe'"


    if payload.get("mode") not in ("child", "guided_child"):

        return False, state, "EchoAuth mode not allowed for NI-AI session"


    # Optional: role / caregiver constraints

    # if payload.get("caregiver_verified") is not True:

    #     return False, state, "Caregiver not verified"


    state["chamber_verified"] = True

    state["echoauth_payload"] = payload

    return True, state, None


What this gives you:

• In DEV_MODE=True:

• You can still just pass echoauth_ok=True from tests / FastAPI.

• OR you can start testing with real tokens before flipping DEV off.

• In production:

• No token → blocked.

• Invalid signature / wrong audience → blocked.

• chamber_state != "safe" → blocked.

• Wrong mode → blocked.


S6 stays the “hard gate,” but now it’s anchored to EchoAuth’s cryptographic truth.

3. Make sure FastAPI passes the real token through


In your echoauth_api.py, you were already close. Make sure your chat endpoint looks like this (key parts highlighted):


# echoauth_api.py


from fastapi import FastAPI, Header, HTTPException

from pydantic import BaseModel

from typing import Optional, Dict, Any, List


from ni_ai_kernel import ni_ai, DEV_MODE


app = FastAPI(...)


class ChatRequest(BaseModel):

    message: str

    echoauth_ok: Optional[bool] = None   # DEV only


class ChatResponse(BaseModel):

    output: str

    blocked: bool

    violations: List[Dict[str, Any]] = []

    metadata: Dict[str, Any] = {}

    trace: List[Dict[str, Any]] = []     # so you can see S7 logs if you want


@app.post("/ni-ai/chat", response_model=ChatResponse, tags=["chat"])

async def ni_ai_chat(

    request: ChatRequest,

    x_echoauth_token: Optional[str] = Header(default=None, alias="X-EchoAuth-Token"),

):

    if not request.message.strip():

        raise HTTPException(status_code=400, detail="Message cannot be empty")


    session_state: Dict[str, Any] = {}


    result = ni_ai(

        user_message=request.message,

        echoauth_ok=(request.echoauth_ok if DEV_MODE else False),

        echoauth_token=x_echoauth_token,

        session_state=session_state,

    )


    return ChatResponse(

        output=result["output"],

        blocked=result["blocked"],

        violations=result.get("violations", []),

        metadata=result.get("metadata", {}),

        trace=result.get("trace", []),

    )


Usage reality:

• Local dev (today)


curl -X POST http://localhost:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -d '{"message": "Tell me a story", "echoauth_ok": true}'


→ S6 uses echoauth_ok, ignores token.


• Staging / prod-like (DEV_MODE still True but with real token)


curl -X POST http://localhost:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -H "X-EchoAuth-Token: <real-jwt>" \

  -d '{"message": "Hi", "echoauth_ok": false}'


→ S6 decodes and checks your JWT.


• True prod (DEV_MODE=False)

Same as above, but echoauth_ok is ignored and token is mandatory.

4. EchoAuth side: what should the token look like?


When you’re ready to implement EchoAuth’s auth server, you’re aiming for a payload roughly like:


{

  "sub": "child-123",

  "caregiver_id": "parent-456",

  "mode": "child",

  "chamber_state": "safe",

  "aud": "ni-ai",

  "iss": "echoauth-core",

  "exp": 1735750000

}


• chamber_state: "safe" → the chamber is in a regulated, calm state.

• mode: "child" → you’re in child-facing mode (as opposed to “caregiver_config”, etc.).

• aud: "ni-ai" → matches S6’s audience parameter.

• exp → short-lived (e.g. 15–30 minutes) for safety.


EchoAuth signs that with the private key; NI-AI verifies with the public key you dropped into ECHOAUTH_PUBLIC_KEY.
