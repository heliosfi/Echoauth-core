PAGE 13 — ASD SENSORY PROFILE INTEGRATION


NI–AI Sensory Mapping • Adaptive Input • Nervous System Alignment


────────────────────────


EchoAuth must adjust not only to emotion and intent, but also to the child’s sensory profile.


The sensory system affects:

• pacing

• tone

• visuals

• sounds

• touch

• timing

• overload risk

• routine success


This page explains how NI–AI adapts in real time to ASD sensory needs.


────────────────────────


⭐ 13.1 — SENSORY PROFILE ENGINE (SPE)


Core module that understands how each child processes the world.


Profiles include:

• Hypersensitive (too much input = overwhelm)

• Hyposensitive (too little input = boredom/seekers)

• Mixed profile (common: sensitive to sound, seeking movement)


Each profile is mapped across:

• sound

• light

• touch

• motion

• social load

• cognitive load

• timing sensitivity


This shapes how EchoAuth interacts.


────────────────────────


⭐ 13.2 — SENSORY DOMAINS TRACKED


EchoAuth monitors 6 sensory domains:


1. Auditory

• volume sensitivity

• tone sensitivity

• rhythmic sensitivity

• need for silence

• need for sound-seeking (rain, hums)


2. Visual

• brightness sensitivity

• color sensitivity

• fast-motion aversion

• need for pattern-based visuals


3. Touch / Proprioception

• uncomfortable clothing

• need for pressure

• need for movement / fidgeting


4. Motion / Vestibular

• motion sickness

• need for rocking

• need for movement breaks


5. Social Load

• tolerance for back-and-forth interaction

• need for predictable phrasing

• threshold for questions


6. Cognitive Load

• step tolerance

• processing time

• overstimulation threshold


NI–AI uses these domains to modulate every interaction.


────────────────────────


⭐ 13.3 — SENSORY INPUT VARIABLES (REAL-TIME)


EchoAuth reads multiple real-time signals:

• typing speed

• response latency

• skipped steps

• short answers

• avoidance behaviors

• requests to pause

• noise level (if mic enabled)

• screen brightness

• time of day

• body movement (if allowed by parent)

• device switching


These map to sensory states like:

• visual overload

• auditory overload

• cognitive fatigue

• movement-seeking

• overstimulation

• under-stimulation

• shutdown risk


────────────────────────


⭐ 13.4 — SENSORY STATE CLASSIFICATION (SSC)


NI–AI classifies the user into one of five real-time sensory states:


STATE 1 — REGULATED

• normal responses

• calm pacing

• predictable interaction


STATE 2 — LIGHT SENSORY NOISE

• mild distraction

• slightly slower responses


STATE 3 — SENSORY UNCERTAINTY

• inconsistent input

• low affect

• subtle avoidance


STATE 4 — OVERSTIMULATION (High Risk)

• rapid device switching

• silence

• physical restlessness

• frustration indicators


STATE 5 — SHUTDOWN RISK (Critical)

• drying eyes

• complete silence

• inability to respond

• signals of overwhelm


EchoAuth never guesses — it adapts based on sensory patterns.


────────────────────────


⭐ 13.5 — NI–AI ADAPTATION RULES (BASED ON SENSORY STATE)


STATE 1 — Regulated

• normal routine speed

• full Calm Mode templates

• light visuals allowed

• no need for Safe Mode


⸻


STATE 2 — Light Sensory Noise

• reduce output length by 20%

• short questions only

• slower pacing

• soft color shift


⸻


STATE 3 — Sensory Uncertainty

• reduce complexity by 40%

• avoid new topics

• stick to routines

• grounding phrases

• pacing slows significantly


⸻


STATE 4 — Overstimulation

• activate Safe Mode

• blue dim filter

• single-step instructions

• no decisions required

• breathing routine recommended


⸻


STATE 5 — Shutdown Risk

• immediate grounding

• ultra-short phrasing (1–4 words)

• zero emotional request

• no cognitive tasks

• caregiver nudge recommended
