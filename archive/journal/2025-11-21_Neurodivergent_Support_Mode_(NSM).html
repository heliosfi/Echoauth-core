Neurodivergent Support Mode (NSM) — Core Definition


NSM = the part of NI-AI that adjusts the environment to the person, not the other way around.

It tunes pacing, sensory load, language, and timing to support autistic, ADHD, dyslexic, and other neurodivergent users.


Think of it as a “Cognitive Comfort Layer” wrapped around the Coherence Engine and EchoAuth.


⸻


NSM – 5 Core Modules


1. Sensory & Cognitive Profile Mapper (NSM-P1)


Purpose:

Builds a living profile of how the user best processes information.


Inputs:

• User preferences (if given):

• “I like slower responses”

• “Too much text overwhelms me”

• Observed interaction signals:

• Time taken to read/respond

• Scroll/exit patterns

• Repeated “can you simplify?” type behavior

• Context (device type, time of day, session length)


Outputs:

• Sensory Profile Map:

• Visual load preference (low / medium / rich)

• Text density preference (short / medium / detailed)

• Response speed tolerance

• Interrupt tolerance (how much back-and-forth is ok)


This profile feeds into all the other NSM modules.


⸻


2. Pacing & Rhythm Regulator (NSM-P2)


Purpose:

Control how fast and how heavy NI-AI responds so the user doesn’t get overwhelmed.


Controls:

• Response length:

• Short burst • Medium • Deep explanation (chosen based on profile + moment)

• Response timing:

• Slight delay to avoid “machine-gun” replies

• Option for “step-by-step” mode

• Check-in prompts:

• “Do you want more detail or is this enough?”

• “Pause here or continue?”


Outputs:

• Adjusted response timing and length that match the user’s nervous system, not just their question.


⸻


3. Complexity & Language Scaler (NSM-P3)


Purpose:

Tune how complex the explanation is — in real time.


Inputs:

• Reading speed signals

• “Can you say that simpler?” type prompts

• Repeated confusion or backtracking


Behavior:

• Simplifies vocabulary:

• Less jargon, more plain language

• Breaks content into chunks:

• Bullet points, short paragraphs

• Offers formats:

• “Want this as: A) bullet steps, B) story example, or C) diagram-style explanation?”


Outputs:

• Right-sized explanations that respect processing differences and working memory.


⸻


4. Sensory Environment & Trigger Guard (NSM-P4)


Purpose:

Prevent overload, meltdowns, or shutdowns triggered by too much input.


(This is more for UI/app layer + voice layer, but architecturally it still lives inside NSM.)


Controls:

• Visual:

• Avoid rapid flicker, harsh contrasts, or cluttered layouts

• Audio:

• Softer tones, no sudden sound spikes, option for ambient sound

• Interaction pattern:

• Avoid “multi-question stacking” when the user shows signs of strain


Signals of possible overload:

• Very long pauses suddenly

• Repeated “wait / hold / stop” style messages

• Stopping mid-thread or answering unrelatedly (cognitive slip)


Outputs:

• Calming adjustments:

• Shorter replies

• “Let’s pause here.”

• Offer grounding or recap instead of pushing new info


⸻


5. Support Bridge & Caregiver Integration (NSM-P5)


Purpose:

Let NI-AI assist the human support system (parent, caregiver, teacher, therapist) without exposing private data.


Possible Behaviors (high-level):

• Generate session summaries in simple language:

• “Today we worked on X. They responded well to Y. They seemed overwhelmed when Z happened.”

• Provide preference hints:

• “They prefer bullet steps.”

• “They do better with visual metaphors than raw numbers.”

• Suggest support actions:

• “Maybe keep instructions to 3 steps.”

• “Try giving more time after each question.”


Outputs:

• Bridge documents / notes that make it easier for humans to support the neurodivergent person, without NI-AI replacing them.

NSM – How It Sits Inside NI-AI


You can store this line exactly in your notes:


NSM sits as a “wrap-around layer” on top of the Coherence Engine, feeding parameters into:

• Emotional State Engine (how gently to respond)

• Coherence Engine (how fast to move through threads)

• EchoAuth (adjusting login/interaction flow for sensory comfort)

• Output Formatter (length, complexity, pacing)


Flow example (for your architecture notes):


User →

ESC (Emotional State Engine) →

NSM (Profile + Pacing + Complexity) →

Coherence Engine (C1–C5) →

EchoAuth (if needed) →

Response shaped by NSM rules →

User
