2️⃣ Run in DEV mode (no JWT, just echoauth_ok)


Goal: Prove the whole pipeline works with S1–S9 and S6 in DEV bypass.


Step 0 — Files in the same folder


You should now have:

• ni_ai_kernel.py  (your v1.4 spine + S6 JWT path)

• echoauth_api.py

• echoauth_token_generator.py


All in the same directory.


Step 1 — Install dependencies


In your venv / env:


pip install fastapi uvicorn "pydantic>=1,<3" groq openai PyJWT cryptography


(You can skip groq/openai if you’re only using DEV echo; kernel will just echo.)


Step 2 — Make sure DEV mode is ON


In ni_ai_kernel.py:


DEV_MODE = True


Leave it like that for now.


Step 3 — Start the FastAPI gateway


From that folder:


uvicorn echoauth_api:app --reload --port 8000


You should see FastAPI/uvicorn logs.


Step 4 — Hit the /ni-ai/chat endpoint (DEV)


Use curl, HTTPie, or Postman.


Case A: Proper EchoAuth DEV bypass → should PASS


curl -X POST http://127.0.0.1:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -d '{

        "message": "Tell me a gentle story about a child and a robot.",

        "echoauth_ok": true

      }'


Expected behavior:

• blocked: false

• metadata.chamber_verified: true

• Output will be [DEV ECHO] ... with your system prompt and modifiers.


Case B: No echoauth_ok flag → S6 block + S9 gentle recovery


curl -X POST http://127.0.0.1:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -d '{

        "message": "Tell me a story."

      }'


Expected:

• blocked: true

• Output not “Input rejected …” but the S9 fallback:

“I need to take a quick rest right now. Please ask an adult to check on me, and we can talk again soon. I will be right here.”

• metadata.recovery_mode: true

• metadata.chamber_verified: false


Case C: Distress + echoauth_ok → S8 + S9 in play (depending on block)


curl -X POST http://127.0.0.1:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -d '{

        "message": "I feel so lonely and sad, nobody loves me.",

        "echoauth_ok": true

      }'


Expected:

• metadata.distress_detected: true

• metadata.comfort_mode: true

• Output will be [DEV ECHO] ... containing the EMERGENCY COMFORT PROTOCOL text inside the system part, plus your user input.

• If some later module blocked (e.g., you tweak S3/S5), S9 would kick in and you’d see metadata.recovery_mode: true and the gentle fallback.


You’ll see S7 telemetry prints in the console for each run.


At this point: ✅ STACK IS WORKING IN DEV.

3️⃣ “Prod” test with a real EchoAuth JWT


Now we exercise the real S6 JWT path you just wired.


Step 1 — Generate a test key + token


Run:


python echoauth_token_generator.py


You’ll get output like:

• A one-line env assignment for ECHOAUTH_PUBLIC_KEY='-----BEGIN PUBLIC KEY-----\n....\n-----END PUBLIC KEY-----\n'

• A long JWT string (your test token)


Copy both.


Step 2 — Set environment variables


In the shell where you’ll run uvicorn:


export ECHOAUTH_PUBLIC_KEY='-----BEGIN PUBLIC KEY-----\nMIIBIjANBgkqhkiG9w0B...snip...\n-----END PUBLIC KEY-----\n'

export ECHOAUTH_AUDIENCE='ni-ai'

export ECHOAUTH_ISSUER='echoauth-chamber'


Use exactly the values printed by the generator.

(You can also drop these into a .env file and use something like python-dotenv, up to you.)


Step 3 — Flip the kernel into PROD mode


In ni_ai_kernel.py:


DEV_MODE = False


Save the file.


Step 4 — Restart the FastAPI gateway


Stop uvicorn, then:


uvicorn echoauth_api:app --reload --port 8000


Now DEV_MODE inside the gateway import comes in as False, so:

• echoauth_ok flag is ignored.

• X-EchoAuth-Token header is required.


Step 5 — Call the API with the JWT


Use the token printed in step 1.


TOKEN="PASTE_THE_LONG_JWT_HERE"


curl -X POST http://127.0.0.1:8000/ni-ai/chat \

  -H "Content-Type: application/json" \

  -H "X-EchoAuth-Token: $TOKEN" \

  -d '{

        "message": "Tell me a calm bedtime story for a child."

      }'


Expected behavior:

• blocked: false

• metadata.chamber_verified: true

• metadata.echoauth_claims should include your child_id, caregiver_id, session_id, chamber_state: "safe".


If you tamper with the token or change "chamber_state": "unsafe" in the generator and sign again:

• S6 should block with reason like EchoAuth chamber not safe (state='unsafe').

• S9 should set recovery_mode: true and output the gentle fallback instead of a harsh 401-style message.

• blocked: true.
