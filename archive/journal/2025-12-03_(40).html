3️⃣ Tiny next step: how to plug in a real LLM


Right now you have:


def dummy_llm(prompt: str, state: State) -> str:

    return f"Echo says: «{prompt}»"


kernel = HERKernel(dummy_llm)


To wire this around a real LLM, you ONLY change that function.


Example with a generic API (pseudo-code, you’ll adapt to your provider):


import openai  # or groq / other client


def real_llm(prompt: str, state: State) -> str:

    # you can pass state info in system messages if you want

    messages = [

        {"role": "system", "content": "You are NI-AI, identity-anchored and safety-constrained."},

        {"role": "user", "content": prompt},

    ]

    resp = openai.chat.completions.create(

        model="gpt-4.1-mini",  # or whatever you use

        messages=messages,

    )

    return resp.choices[0].message.content


kernel = HERKernel(real_llm)


Now everything is the same:


result = kernel.run_once("Hello, friend!", session_state={"echoauth_token": "valid-signed-jwt"})

print(result["output"])

print(result["blocked"])

print(result["violations"])


The only difference is:


Your HERKernel is now wrapped around a real LLM instead of dummy_llm.


The LLM becomes just an untrusted function that HER controls.
