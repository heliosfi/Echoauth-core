Appendix L — Latency & Throughput Characteristics


(Auditor / Engineer Reference)


Scope


This appendix quantifies where latency may occur in the EchoAuth system and demonstrates that governance enforcement does not meaningfully reduce intelligence throughput or execution speed under resolved authority.


⸻


Pipeline Separation


EchoAuth enforces governance only at the execution boundary.

It does not participate in model inference or reasoning.

1. Intelligence Pipeline (NI-AI) — interpret, propose, explain

2. Execution Pipeline (MCG / CEG / EchoAuth) — authorize and actuate


⸻


Intelligence Pipeline Latency (Unaffected)


Typical real-world ranges:

• UI rendering: 10–100 ms

• Network RTT: 50–300 ms (good), 300–1000 ms (congested)

• LLM inference: 300 ms – 3 s (interactive), longer for complex reasoning


EchoAuth added latency: 0 ms


The system does not gate, throttle, or delay intelligence operations.


⸻


Execution Pipeline Latency


Fast-Path (Authority Resolved, Within Envelope)

Operations:

• Role / portal validation

• Envelope checks (S7, S10, S22)

• Conflict dominance check (S8)

• Single-use execution token validation

• CEG open


Typical costs:

• Local rule checks: < 1 ms

• Cryptographic verification: 0.1–3 ms

• Async logging: < 5–20 ms

• Remote identity / policy call (if required): 50–300 ms


Total added latency:

• ~1–50 ms (local, optimized)

• ~50–300 ms (remote policy dependency)


This latency is below human perceptual thresholds.


⸻


Slow-Path (Authority Unresolved → Agency Collapse)

• CEG locks immediately: < 10–50 ms

• Execution does not proceed

• Human resolution required


This is not performance latency; it is an intentional safety stop.


⸻


Human Resolution Time (Expected and Intentional)


Sources of elapsed time:

• Biometric / PIN re-auth: 0.5–3 s (typical), 3–10 s (edge cases)

• Notification pickup: 5–60 s (context-dependent)

• Decision selection: 2–30 s


Interpretation:

Execution slows only when authority is unclear. This is a design goal.


⸻


Operational Impact


The system adds:

• milliseconds when execution is safe

• seconds only when human authority must be asserted


This prevents downstream costs measured in hours or days, including:

• unauthorized schedule drift

• clinical mission creep

• incident remediation

• trust loss and escalation overhead


⸻


Design Control Lever


Latency sensitivity depends on architecture choices:

• Local envelope checks + local token validation + async logging → near-instant gating

• Remote checks + synchronous logging → +50–300 ms


Even in worst case, delay remains negligible relative to human perception.


⸻


Conclusion


EchoAuth operates at full intelligence speed and full execution speed when authority is resolved.

Delay appears only at the human decision boundary, which is the intended safety property.

Governance Integrity: Preserved
