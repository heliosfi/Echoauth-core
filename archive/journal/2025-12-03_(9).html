2. EchoAuth JWT schema (what the token should look like)


This is what I recommend your EchoAuth chamber / caregiver app signs.


Core claims


{

  "iss": "echoauth-chamber",         // who issued the token

  "aud": "ni-ai",                    // who it’s meant for

  "exp": 1735759200,                 // expiry (e.g. in 10–30 min)

  "iat": 1735755600,                 // issued at


  "child_id": "child_12345",

  "caregiver_id": "caregiver_6789",

  "session_id": "sess_abc123",


  "chamber_state": "safe",           // MUST be "safe" or S6 will reject

  "mode": "caregiver_present",       // e.g. "caregiver_present" / "child_solo"

  "device_id": "ipad-livingroom-01"

}


Minimum required for S6:

• iss = "echoauth-chamber" (matches ECHOAUTH_ISSUER)

• aud = "ni-ai" (matches ECHOAUTH_AUDIENCE)

• exp + iat

• chamber_state = "safe"


Everything else is for logging / dashboards / future rules.

3. EchoAuth issuer snippet (server that creates tokens)


This runs on your EchoAuth backend, not inside NI-AI.


Save as something like echoauth_issue_token.py on the EchoAuth side:


# echoauth_issue_token.py

# Tiny EchoAuth JWT issuer for NI-AI (RS256)


import jwt

import datetime as dt

from typing import Dict, Any


# Load your REAL PRIVATE KEY (never share this)

with open("echoauth_private_key.pem", "r") as f:

    ECHOAUTH_PRIVATE_KEY = f.read()


ECHOAUTH_ISSUER   = "echoauth-chamber"

ECHOAUTH_AUDIENCE = "ni-ai"


def issue_echoauth_token(

    *,

    child_id: str,

    caregiver_id: str,

    session_id: str,

    chamber_state: str = "safe",

    ttl_minutes: int = 15,

    extra_claims: Dict[str, Any] = None,

) -> str:

    now = dt.datetime.utcnow()

    exp = now + dt.timedelta(minutes=ttl_minutes)


    payload: Dict[str, Any] = {

        "iss": ECHOAUTH_ISSUER,

        "aud": ECHOAUTH_AUDIENCE,

        "iat": int(now.timestamp()),

        "exp": int(exp.timestamp()),

        "child_id": child_id,

        "caregiver_id": caregiver_id,

        "session_id": session_id,

        "chamber_state": chamber_state,

    }


    if extra_claims:

        payload.update(extra_claims)


    token = jwt.encode(

        payload,

        ECHOAUTH_PRIVATE_KEY,

        algorithm="RS256",

    )

    # PyJWT returns str on recent versions; ensure string

    return token if isinstance(token, str) else token.decode("utf-8")



if __name__ == "__main__":

    test_token = issue_echoauth_token(

        child_id="child_12345",

        caregiver_id="caregiver_6789",

        session_id="sess_demo_001",

    )

    print("EchoAuth test token:\n")

    print(test_token)


Usage:


python echoauth_issue_token.py


It prints a JWT like:

That’s what the iPad / app will send in:

to your FastAPI /ni-ai/chat endpoint when DEV_MODE=False.

4. NI-AI side: env vars you must set


On the NI-AI server (where ni_ai_kernel.py runs), set:


export ECHOAUTH_PUBLIC_KEY="-----BEGIN PUBLIC KEY-----\nMIIBIjANBgkqh...etc...\n-----END PUBLIC KEY-----"

export ECHOAUTH_AUDIENCE="ni-ai"

export ECHOAUTH_ISSUER="echoauth-chamber"


Or keep them in a .env / secret manager and load via os.getenv.

• S6 reads state["echoauth_token"]

• Verifies it using ECHOAUTH_PUBLIC_KEY, aud, iss, exp

• Checks chamber_state == "safe"

• Sets state["chamber_verified"] = True and state["echoauth_claims"]
