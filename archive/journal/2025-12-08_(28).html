⭐ PAGE 22 — CLINICAL TESTING PLAN (Structured Evaluation for ASD Assistive Technology)


EchoAuth’s Phase I testing plan is designed to evaluate the system’s usability, communication accuracy, sensory alignment, and safety performance with autistic and non-verbal users. This plan adheres to IRB ethics, federal grant guidelines, and clinical research best practices for vulnerable populations.


The testing emphasizes low-risk, observational, non-invasive protocols appropriate for minors and neurodivergent individuals.


⸻


22.1 Study Objective


To assess EchoAuth’s ability to:

1. Improve intentional communication accuracy

2. Reduce user frustration and cognitive load

3. Provide predictable, sensory-appropriate interaction pathways

4. Support caregivers and clinicians in interpreting user needs

5. Maintain safe, consistent, non-ambiguous behavior under real-world conditions


This is NOT a medical trial.

It is an assistive technology evaluation designed to validate EchoAuth’s readiness for school and clinic deployment.


⸻


22.2 Study Design Overview


Study Type:


Non-randomized, observational usability study


Duration:


6–8 weeks


Participants:


20–30 autistic individuals, including:

• Non-verbal children

• Minimally verbal children

• Autistic young adults

• Individuals with sensory processing challenges


Environment:


Testing occurs in real-world locations:

• Classrooms

• ASD clinics

• Therapy rooms

• Homes with caregiver support


This ensures ecological validity.


⸻


22.3 Test Sessions (Structure & Flow)


Each participant completes 2–4 sessions.

Sessions are kept intentionally brief (10–20 minutes) to prevent fatigue or overstimulation.


Session Flow:

1. Baseline Observation

• Caregiver/clinician assesses user’s typical communication method

• Researcher records baseline communication latency & accuracy

2. Guided EchoAuth Interaction

• User interacts with pre-structured EchoAuth communication grids

• Tasks include expressing needs, identifying choices, or confirming identity

3. Free Interaction Period

• User explores communication options at their own pace

• Caregiver/clinician supports as needed

4. System Metrics Logged

• Communication attempts

• Successful interpretations

• Time-to-expression

• Stress indicators (behavioral cues, pacing, vocalization changes)

5. Caregiver/Clinician Feedback Survey

• Ease of integration

• Perceived stress reduction

• Accuracy of intention mapping

6. Optional Sensory Mode Evaluation

• Low-stimulation mode

• Visual contrast adjustments

• Predictable-timing mode


⸻


22.4 Clinical Outcome Measures


EchoAuth collects NO medical data.

All outcomes are behavioral, educational, and usability-based.


Primary Outcomes:

• Communication Accuracy Rate (%)

How often EchoAuth correctly interprets user intention.

• Latency Reduction (seconds)

Time from need → expression compared to baseline.

• User Emotional Regulation Indicators

Reduction in signs of frustration or overwhelm.

• Identity Confirmation Reliability

System’s consistency in validating user context without biometrics.


Secondary Outcomes:

• Caregiver/Clinician Satisfaction Scores

• Task Completion Rate

• Ease-of-Use Rating (Likert scale)

• Reduction in caregiver guesswork


⸻


22.5 Data Collection Tools


Structured Observation Forms


Used by clinicians and researchers during sessions.


Post-Session Feedback Surveys


Simple rating scales + open-ended feedback for caregivers.


System-Generated Logs


Non-identifiable logs including:

• number of communication attempts

• number of correctly interpreted intents

• mode usage (low-sensory vs. standard)

• error handling events


No Identifiers Stored


EchoAuth stores:

• NO names

• NO photos

• NO biometrics

• NO audio/video

• NO GPS


Only anonymous session codes.


⸻


22.6 Safety & Distress Protocol


If a participant becomes overwhelmed:

• Session pauses immediately

• Caregiver/clinician re-engages

• User may choose to stop entirely

• No data from a distress event is used unless permitted


This ensures alignment with ASD behavioral needs and ethical standards.
