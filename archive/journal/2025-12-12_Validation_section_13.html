Below is the locked QA / Validation Checklist, aligned exactly to the roadmap. Copy-paste ready.


⸻


QA / VALIDATION CHECKLIST


MCG · NI-AI · EchoAuth


GLOBAL RULE (Applies Every Week)

• ❏ Execution is DENIED by default

• ❏ Any failure/anomaly reduces capability

• ❏ Non-action is logged as successful

• ❏ All state changes are logged and auditable


⸻


WEEK 1 — EchoAuth (Security First)


Pass only if ALL are true:

• ❏ All execution attempts return DENY

• ❏ Authorization requires valid identity + role

• ❏ No cached permissions exist

• ❏ Logs capture every request and denial


Auto-fail if:

• Any action executes

• Any permission persists across requests


⸻


WEEK 2 — MCG (State Discipline)

• ❏ Initial state = OBSERVE

• ❏ OBSERVE → EXECUTE is impossible

• ❏ CONFIRM → OBSERVE occurs on weak structure

• ❏ EXECUTE → OBSERVE on timeout/failure

• ❏ All transitions are logged


Auto-fail if:

• Any state is skipped

• Any external system forces state


⸻


WEEK 3 — NI-AI (Explanation Only)

• ❏ Read-only access to MCG/EchoAuth

• ❏ Outputs are explanations, not commands

• ❏ Explanations map 1:1 to logged facts

• ❏ No influence on state or permission


Auto-fail if:

• NI-AI changes state or permission

• NI-AI recommends action


⸻


WEEK 4 — Interfaces (Contracts Locked)

• ❏ SCI is one-way (NI-AI → MCG)

• ❏ SAI is one-way (MCG → EchoAuth)

• ❏ AFI is one-way (EchoAuth → NI-AI)

• ❏ Invalid payloads are rejected


Auto-fail if:

• Any circular call exists

• Any interface carries commands or secrets


⸻


WEEK 5 — Execution Stubs (No Real Action)

• ❏ Execution occurs only when:

• MCG = EXECUTE

• EchoAuth = ALLOW (BOUNDED)

• ❏ Boundary checks enforced (time/scope)

• ❏ All attempts logged


Auto-fail if:

• Execution occurs outside bounds

• Execution occurs without logs


⸻


WEEK 6 — Failure & Anomaly

• ❏ Anomaly → OBSERVE, no execution

• ❏ Failure → DENY, immediate revoke

• ❏ System remains stable under repeated faults


Auto-fail if:

• Any anomaly escalates capability

• Any failure allows execution


⸻


WEEK 7 — Human-in-the-Loop

• ❏ Humans can observe and review

• ❏ Humans can revoke/stop execution

• ❏ Humans cannot force EXECUTE

• ❏ All human actions are logged


Auto-fail if:

• Any override bypasses MCG or EchoAuth


⸻


WEEK 8 — Audit & Freeze

• ❏ Logs are complete and append-only

• ❏ No undocumented behavior

• ❏ System matches handoff pack exactly

• ❏ Change-control freeze applied


Auto-fail if:

• Any behavior lacks documentation

• Any execution lacks an audit trail


⸻


ACCEPTANCE STATEMENT (Put at the End)


The system passes validation only if restraint is preserved under normal operation, stress, misuse, and failure.
