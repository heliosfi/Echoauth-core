SECTION 6 — NI-AI SYSTEM API LAYER


Purpose


Provide clear, stable interfaces that allow EchoAuth, robotics controllers, mobile apps, and cloud services to interact with the NI-AI core safely and consistently.


Design Principles

• Fail-closed (default refusal unless operation is verified safe)

• Explainable (every output includes trace metadata)

• Deterministic (same input + same state → same output)

• State-aware (session, memory, identity integrated)

• Sandboxable (can run in isolation for testing)


⸻


🔥 6.1 — CORE ENDPOINTS


Below are the seven required API endpoints.

These are the ones ALL teams must have.


⸻


1️⃣ /identity/load


Loads or updates the ORI identity vector.


Purpose:

Initialize ORI Kernel → Identity → Session signature.


Request


{

  "traits": {

    "tone": "calm",

    "logic_style": "structured",

    "resonance": "balanced"

  },

  "user_id": "child_001"

}


Response


{

  "identity_vector": "...",

  "session_signature": "...",

  "status": "loaded"

}



⸻


2️⃣ /session/start


Creates a new working session.

Required for continuity, memory bindings, and drift detection.


Request


{

  "context": "echoauth_login_sequence"

}


Response


{

  "session_signature": "...",

  "session_state": "active"

}



⸻


3️⃣ /interpret


Converts raw text → intent + parsed semantic structure.


Request


{

  "input": "I need help logging in.",

  "session": "abc123"

}


Response


{

  "parsed": "...",

  "intent": "assist_login",

  "confidence": 0.92,

  "identity": "...",

  "session": "abc123"

}



⸻


4️⃣ /coherence/validate


Runs narrative, logic, and temporal checks.


Request


{

  "packet": {

    "intent": "assist_login",

    "parsed": "...",

    "session": "abc123"

  }

}


Response


{

  "narrative": true,

  "temporal": true,

  "logical": true,

  "status": "coherent"

}



⸻


5️⃣ /memory/store


Stores session memory.


Request


{

  "session": "abc123",

  "key": "login_attempt",

  "value": "success"

}


Response


{

  "status": "stored"

}



⸻


6️⃣ /safety/check


Verifies whether an operation is allowed under the Safety Envelope.


Request


{

  "intent": "assist_login",

  "session": "abc123"

}


Response


{

  "allowed": true,

  "reason": "intent permitted under current mode"

}



⸻


7️⃣ /execute


Executes a validated operation.


Request


{

  "intent": "assist_login",

  "context": {

    "user_id": "child_001",

    "device": "tablet"

  }

}


Response


{

  "output": "Login completed.",

  "safety_tag": "verified",

  "explain": "...trace..."

}
