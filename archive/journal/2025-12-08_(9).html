⭐ PAGE 39 — CLINICAL TESTING PROTOCOLS & ASD USABILITY STANDARDS


(IRB-Ready • Clinician-Approved • Grant-Grade)


EchoAuth must serve autistic and non-verbal individuals with clarity, predictability, and emotional safety.

This section describes how testing is conducted, what standards are followed, and how user wellbeing is protected at every stage.


⸻


SECTION A — GUIDING PRINCIPLES FOR ASD TESTING


All clinical testing for EchoAuth adheres to:


✔ 1. Predictability Principle


Reduce uncertainty → reduce anxiety.


System interactions must:

• follow structured patterns

• avoid unexpected changes

• apply consistent timing


✔ 2. Low-Sensory Load Principle


Interfaces must never overwhelm.


Testing utilizes:

• low-contrast colors

• minimal animations

• quiet auditory signals

• stable icon layout


✔ 3. Autonomy & Choice Principle


Users must always have:

• control

• opt-out capability

• simple yes/no fallback channels


✔ 4. Safety-First Principle


If the system is uncertain →

it must not act without confirmation.


✔ 5. Respect for Communication Differences


Non-verbal → valid.

Low-verbal → valid.

Object-reliant → valid.

Gesture-based → valid.


EchoAuth adapts to them all.


⸻


SECTION B — USER GROUPS INCLUDED IN TESTING


Phase I testing includes 3 user categories:


Group 1 — Non-Verbal ASD Users (Primary Cohort)


Ages 5–18

Profiles may include:

• AAC users

• gesture communicators

• object-selection communicators

• sensory-sensitive individuals


Group 2 — Low-Verbal ASD Users


Ages 6–18

Profiles include:

• phrase-level communication

• inconsistent verbal patterns

• receptive but limited expressive language


Group 3 — ASD Young Adults (Optional, Age 18–25)


For independence-building use cases:

• identity confirmation

• emergency communication

• appointment check-in workflows


⸻


SECTION C — CLINICAL TESTING FRAMEWORK


Testing follows a 3-phase structure aligned with IRB-approved usability research.


⸻


PHASE 1 — Baseline Observation (No Device Interaction)


Goal: understand user’s natural communication patterns.


Clinicians observe:

• preferred communication style

• sensory triggers

• transitions between emotional states

• frustration signals


This creates the Personalized Interaction Profile (PIP) for each user.


⸻


PHASE 2 — Guided Interaction (Assisted Use of EchoAuth)


A clinician or caregiver introduces EchoAuth through:

1. Simple needs grid (e.g., food, water, help)

2. Yes/No confirmation panel

3. Choice-based communication templates


Metrics collected:

• communication accuracy

• time-to-intent

• user stress indicators

• sensory regulation success


⸻


PHASE 3 — Independent Interaction (User-Led Use)


User interacts directly with EchoAuth for:

• need expression

• identity confirmation

• emotion selection

• environment-based requests


Success indicators:

• autonomy gain

• reduced behavioral frustration

• improved caregiver understanding


⸻


SECTION D — METRICS & EVALUATION CRITERIA


D1. Communication Accuracy


Target: 30–40% improvement in expression of needs and choices.


Measured through:

• validated ASD communication scoring

• matching caregiver reports

• reduction in unmet-need behaviors


⸻


D2. Stress & Emotional Regulation


Target: measurable decrease in stress triggers during communication.


Indicators:

• reduced physical agitation

• fewer escalations

• stabilized breathing patterns

• increased willingness to engage


⸻


D3. Time-to-Intent


Goal: reduce by 25–35% the time required for a user to express a need.


Measured in:

• guided sessions

• independent sessions

• transition periods


⸻


D4. Identity Confirmation Reliability


EchoAuth must achieve ≥ 95% confirmation stability using:

• contextual cues

• signature patterns

• session traits

NOT biometric scanning.


⸻


D5. Usability & Comfort Rating (Clinician + Caregiver)


Target:

• ≥ 80% clinician approval

• ≥ 85% caregiver approval

• ≥ 70% user comfort acceptance (ASD-standard scale)


⸻


SECTION E — IRB & ETHICS COMPLIANCE FRAMEWORK


Testing adheres to:

• Belmont Report standards

• IRB oversight

• Assent/consent requirements

• HIPAA-lite privacy models


Core clinical protections:

1. No identifiable health data collected

2. No video/facial biometrics involved

3. All sessions reversible and deletable

4. Parents/caregivers present for minors

5. No penalties for withdrawal


⸻


SECTION F — CLINICAL SAFETY MEASURES


During testing:

• EchoAuth runs in “Pilot Mode” with limited actions

• Safety wrapper overrides ANY uncertain case

• Users cannot access harmful content

• System latency capped to prevent overwhelm

• Fallback grid available at all times


Safety Goal:

Zero harm. Zero distress. Zero unsafe output.
