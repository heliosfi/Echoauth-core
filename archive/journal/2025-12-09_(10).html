PAGE 45 — Hardware / Software Requirements


This ensures labs, schools, clinics, and funders know what is needed to deploy EchoAuth reliably.


⸻


Minimum Device Requirements

• iOS / Android tablet, 8–12 inches

• RAM: 4GB+

• Storage: 64GB+

• OS security patches active

• Touchscreen mandatory


⸻


Optional Assistive Hardware

• Adaptive switches

• Low-force input buttons

• Eye-tracking interfaces (future integration)


All external devices funnel through the NI abstraction layer.


⸻


Software Requirements

• Local storage encryption (AES-256)

• Sandboxed runtime environment

• Role-based access control enabled

• Safety policy file loaded at boot


⸻


Deployment Requirements

• Local-only: No internet needed

• Hybrid: Secure endpoint via TLS 1.3

• Enterprise: SAML/OIDC integration

PAGE 46 — Test Cases (ASD-Centered Validation)


These cases are what pilot partners, compliance officers, and researchers expect.


⸻


Test Case Group 1 — Sensory Stability

• Confirm predictable screen transitions

• Validate low-contrast visuals

• Ensure no flashing or animated elements


Pass Criteria:

✓ No sensory triggers across 20 users


⸻


Test Case Group 2 — Communication Accuracy

• User selects grid item → NI correctly interprets intent

• Special attention to ambiguous inputs

• Check UMCE rejection for contradictory signals


Pass Criteria:

✓ ≥ 80% accuracy in intention decoding


⸻


Test Case Group 3 — Safety & Override

• Test caregiver override action

• Confirm no action bypasses UMCE

• Validate fail-closed behavior on errors


Pass Criteria:

✓ All overrides logged

✓ No unsafe action executed


⸻


Test Case Group 4 — Compliance Checks

• FERPA mode: student PII isolation

• HIPAA mode: PHI encryption + access control

• COPPA mode: parental approval for any sync


Pass Criteria:

✓ Complete regulatory alignment


⸻


Test Case Group 5 — Real-World Scenarios

• User unable to express need → grid path emerges

• Sudden stress response → Calm Mode triggers

• Mis-tap or repeated input → stable recovery


Pass Criteria:

✓ System predictable under stress

✓ No breakdown in stability
