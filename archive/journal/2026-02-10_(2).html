Squeeze definition (practical)


Compute BB(20, 2). Then:

• bb_width = (upper - lower) / middle  (normalized; stable across price levels)

• A bar is “squeezed” if bb_width is in the bottom X% of the last N bars, OR below a fixed threshold.


Recommended settings for TSLA 30m

• BB_LEN = 20

• BB_STD = 2

• SQUEEZE_LOOKBACK = 80  (~40 hours of 30m bars)

• SQUEEZE_PCTL = 0.20  (lowest 20% width = squeeze regime)

• Optional: require squeeze to persist MIN_SQUEEZE_BARS = 3–6


⸻


Code: add BB width + squeeze flag to your df


def add_bollinger_width(df, length=20, n_std=2):

    mid = df["close"].rolling(length).mean()

    sd  = df["close"].rolling(length).std(ddof=0)

    upper = mid + n_std * sd

    lower = mid - n_std * sd


    # Normalized BB width

    df["bb_mid"] = mid

    df["bb_upper"] = upper

    df["bb_lower"] = lower

    df["bb_width"] = (upper - lower) / mid


    return df


def add_squeeze_flag(df, lookback=80, pctl=0.20, min_bars=3):

    # Rolling percentile threshold

    # (Using quantile on rolling window)

    df["bbw_thresh"] = df["bb_width"].rolling(lookback).quantile(pctl)


    # Squeeze if width <= threshold

    df["is_squeeze_raw"] = df["bb_width"] <= df["bbw_thresh"]


    # Require squeeze persistence (optional)

    df["squeeze_run"] = df["is_squeeze_raw"].astype(int).groupby(

        (df["is_squeeze_raw"] != df["is_squeeze_raw"].shift()).cumsum()

    ).cumsum()


    df["is_squeeze"] = df["is_squeeze_raw"] & (df["squeeze_run"] >= min_bars)

    return df


Usage (after you build df with OHLCV + indicators):


df = add_bollinger_width(df, length=20, n_std=2)

df = add_squeeze_flag(df, lookback=80, pctl=0.20, min_bars=4)

Integrate into your signal gate (one-liner)


You want the confirm candle to only count if it occurs during (or immediately after) a squeeze.


Two common variants:


Variant A (strict): confirm must be during squeeze


if not curr["is_squeeze"]:

    return False


Variant B (better): confirm must occur within K bars after squeeze ends


This catches the “release” bar.


SQUEEZE_RELEASE_BARS = 3  # allow breakout up to 3 bars after squeeze


recent_squeeze = df["is_squeeze"].iloc[i-SQUEEZE_RELEASE_BARS:i+1].any()

if not recent_squeeze:

    return False


Drop this into bullish_confirm and bearish_confirm right after the time gate and before MACD.


Example:


def bullish_confirm(curr, prev, last3, hist, i, atr_val, df):

    # Squeeze gate

    recent_squeeze = df["is_squeeze"].iloc[max(0, i-3):i+1].any()

    if not recent_squeeze:

        return False


    return (curr["close"] > RESIST

            and not is_doji_like(curr)

            and bullish_body_expands(curr, prev)

            and volume_expanding(curr, last3)

            and confirm_candle_not_too_big(curr, atr_val)

            and (not USE_MACD or macd_green_expanding(hist, i)))


Same pattern for bearish.

Why this improves TSLA performance

• TSLA 30m prints a lot of fake breaks during high BB width (already volatile).

• Breakouts that follow compression have higher probability of follow-through (especially in your allowed windows: Open Drive + Power Hour).

• The squeeze filter reduces trades, but improves signal quality.
