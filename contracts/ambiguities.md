# Blocking Issues

None at the contract type layer.

Nested business schemas for `payload`, `context`, `evidence`, `scope`, `rules`,
`facts`, `credential_set`, `context_constraints`, and `effective_scope` are not
generated because the existing specifications intentionally define those fields
as domain-specific canonical JSON objects. This is a documented implementation
exception, not an open contract ambiguity.
