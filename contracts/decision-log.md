# Decision Log

| Decision ID | Date | Decision | Evidence | Contract Impact |
| --- | --- | --- | --- | --- |
| ADR-0001 | 2026-06-18 | Resolve generic object fields as `CanonicalJsonObject`. | `specs/echoauth-spec.md`, `specs/event-bus.md`, `specs/audit-record.md`, `specs/identity-resolution.md`, `specs/authority-resolution.md`, `specs/delegation-validation.md`, `specs/policy-evaluation.md`, `specs/invariant-validator.md` repeatedly require canonical serialization, deterministic facts, and hash binding for object fields. | Added `CanonicalJsonObject` to `schemas/common.schema.json` and `api/openapi.yaml`; updated JSON Schema, OpenAPI, protobuf comments, event schema, service contracts, integration contracts, and database schema comments. |
| ADR-0002 | 2026-06-18 | Resolve `credential_refs` and `role_refs` as `StringReferenceList`. | `specs/identity-model.md` defines both fields as lists of credential/verifier or role references. | Added `StringReferenceList` to shared schemas and canonical data contracts. |
| ADR-0003 | 2026-06-18 | Resolve `validation_errors` as `ValidationErrorList`. | `specs/runtime-envelope.md` defines `validation_errors` as a list of failed validation checks. | Updated OpenAPI `RuntimeEnvelopeResult.validation_errors`; added shared schema and service/integration data contract. |
| ADR-0004 | 2026-06-18 | Preserve protobuf `google.protobuf.Struct` as the wire representation for canonical JSON object fields. | Existing protobuf contract already used `Struct`; changing to a wrapper message would alter field shape without specification authority. | Added canonical-object comments to `contracts/protobuf/echoauth.proto`; no wire-level behavior added. |
| ADR-0005 | 2026-06-18 | Database `*_json` columns store canonical JSON text; no engine-specific JSON validation is mandated. | Specifications require persistence and deterministic hashing but do not name a database engine or JSON validation function. | Added database contract header explaining canonical JSON text encoding. |
| ADR-0006 | 2026-06-18 | Do not generate nested business schemas for canonical object fields. | Existing specifications define fields as domain-specific facts, evidence, scopes, rules, or payloads and do not specify nested keys. | Marked nested schemas as documented implementation exceptions in `contracts/ambiguities.md` and readiness artifacts. |
| ADR-0007 | 2026-06-19 | Reconcile Recovery vocabulary around `EXECUTION_BLOCKED` and `HALTED` as the only entry states; retain `HOLD` as a governance outcome represented by a proposal-only `EXECUTION_BLOCKED` artifact; deprecate `recovery_allowed`; and define a separate Recovery Review Protocol without adding a general Review outcome. | `specs/runtime-state-machine.md`; `specs/runtime-halt-model.md`; `specs/runtime-recovery.md`; Sprint 2L and Sprint 2O evidence; Recovery governance review. | Added JSON Schema Recovery contracts, renamed `authority_decision_id` to `authority_resolution_id`, removed `approved_for_revalidation`, registered Recovery validation, and prohibited authorization, execution, orchestration, external integration, and state mutation. |
